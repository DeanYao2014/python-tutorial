# Python 3.12+ 教程 — 教学计划

> 定位：面向有基础的开发者，深入 Python 内部机制 + 现代用法 + 设计思想。不是参考手册。

---

## 第一部分：基础——重新认识 Python

**目标**：纠正常见误解，建立准确的 Python 心智模型。假设读者已经会写循环和函数，重点是"原来是这样"。

| 章节 | 主题 | 核心要点 | 官方文档引用 |
|------|------|---------|-------------|
| 1.1 | Python 对象模型 | 一切皆对象、`id()`/`is`/`==` 的区别、引用计数与 GC 概览、可变 vs 不可变对象的本质 | [Data Model](https://docs.python.org/3/reference/datamodel.html) |
| 1.2 | 名字与作用域 | LEGB 规则、`global`/`nonlocal`、闭包捕获机制、`del` 的真正含义 | [Naming and Binding](https://docs.python.org/3/reference/executionmodel.html) |
| 1.3 | 容器深度解析 | list 底层（动态数组）、dict 底层（哈希表，3.6+ 紧凑实现）、tuple 不只是不可变 list、切片对象 `slice`、`*` 解包的本质 | [Built-in Types](https://docs.python.org/3/library/stdtypes.html) |
| 1.4 | 字符串与字节 | Unicode 内幕（code point vs 编码）、f-string 进阶（3.12 嵌入表达式增强）、`bytes`/`bytearray`/`memoryview` | [Text Sequence Type](https://docs.python.org/3/library/stdtypes.html#text-sequence-type-str), [Bytes](https://docs.python.org/3/library/stdtypes.html#bytes-objects) |
| 1.5 | 模式匹配 | `match case`（3.10+）：解构、守卫、序列/映射/类模式、与 `isinstance` 链的对比 | [match Statements](https://docs.python.org/3/reference/compound_stmts.html#match) |
| 1.6 | 函数是一等公民 | 函数对象属性、`*args`/`**kwargs` 原理、参数默认值的绑定时机、`/` 仅位置参数（3.8+） | [Function Definitions](https://docs.python.org/3/reference/compound_stmts.html#function-definitions) |
| 1.7 | 异常体系 | 异常即对象、`__traceback__`、`except*`（3.11+ ExceptionGroup）、`add_note()`（3.11+）、异常链 `raise ... from` | [Exceptions](https://docs.python.org/3/tutorial/errors.html), [Built-in Exceptions](https://docs.python.org/3/library/exceptions.html) |
| 1.8 | 模块系统详解 | `import` 的完整流程（finder → loader → module object）、`sys.modules` 缓存、相对/绝对导入的坑、`__import__` 与 `importlib` | [Import System](https://docs.python.org/3/reference/import.html) |

**第一部分思考题**：`a = 256; b = 256; a is b` 为 True，但 `a = 257; b = 257; a is b` 呢？为什么？

## 第二部分：面向对象——Python 的 OOP 哲学

**目标**：理解 Python 对象模型的全部关键机制，能设计出地道的 Python 类。

| 章节 | 主题 | 核心要点 | 官方文档引用 |
|------|------|---------|-------------|
| 2.1 | 类与元类的基础 | `type` 是元类、`__init__` vs `__new__`、`__call__` 让实例可调用、描述符初识（property 即描述符） | [Data Model: Special Method Names](https://docs.python.org/3/reference/datamodel.html#special-method-names) |
| 2.2 | 属性控制 | property、`__getattr__`/`__getattribute__`/`__setattr__`/`__delattr__`、`__slots__` 的内存优化原理 | [Customizing Attribute Access](https://docs.python.org/3/reference/datamodel.html#customizing-attribute-access) |
| 2.3 | 魔术方法全景 | `__repr__`/`__str__`、`__eq__`/`__hash__`（可变对象不该实现 `__hash__`）、`__bool__`、`__len__`、运算符重载 | [Special Method Names](https://docs.python.org/3/reference/datamodel.html#special-method-names) |
| 2.4 | 可迭代与可调用对象 | `__iter__`/`__next__`（iterator protocol）、`__getitem__` 也能迭代？`__call__`、实现一个"类文件"对象 | [Container Types](https://docs.python.org/3/reference/datamodel.html#emulating-container-types) |
| 2.5 | 继承与 MRO | C3 线性化算法、super() 的工作原理（不只是"调用父类"）、Mixin 模式、多继承的正确姿势 | [Data Model: `__mro__`](https://docs.python.org/3/reference/datamodel.html#the-standard-type-hierarchy) |
| 2.6 | 数据类：告别样板代码 | `@dataclass`（字段、`field`、`__post_init__`）、`slots=True`（3.10+）、keyword-only 字段（3.10+）、`KW_ONLY`（3.10+）、与 `TypedDict`/`NamedTuple` 的取舍 | [dataclasses](https://docs.python.org/3/library/dataclasses.html) |
| 2.7 | 类型系统进阶 | `TypeVar`/`ParamSpec`/`TypeVarTuple`、`Protocol`（结构化子类型）、`Self`（3.11+）、`@overload`、`Literal`、`Final`、`TypedDict` | [typing](https://docs.python.org/3/library/typing.html) |
| 2.8 | 元类实战 | `__init_subclass__`（实际比元类更常用）、`__set_name__`、元类的典型场景（ORM、注册表模式）、什么时候不该用元类 | [Metaclasses](https://docs.python.org/3/reference/datamodel.html#metaclasses) |

**第二部分思考题**：用元类实现一个自动注册子类的"插件系统"。`@dataclass`、`TypedDict`、`NamedTuple` 分别适合什么场景？

## 第三部分：高级主题——Python 的核心能力

**目标**：这是教程的**重点部分**。深入 Python 最强大的语言特性，理解背后的设计思想。

| 章节 | 主题 | 核心要点 | 官方文档引用 |
|------|------|---------|-------------|
| 3.1 | 迭代器与生成器 | Iterator Protocol（`__iter__`/`__next__`）、生成器函数 vs 生成器表达式、`yield from` 的委托语义、`send()`/`throw()`/`close()` 双向通信 | [Iterator Types](https://docs.python.org/3/library/stdtypes.html#iterator-types), [Generator Types](https://docs.python.org/3/library/stdtypes.html#generator-types) |
| 3.2 | 装饰器完全指南 | 闭包是基础、`functools.wraps` 为什么必须加、参数化装饰器（三层函数）、`functools.partial` vs 装饰器、类装饰器 vs `__init_subclass__` | [Compound Statements: Function Definitions](https://docs.python.org/3/reference/compound_stmts.html#function-definitions) (decorator syntax) |
| 3.3 | 上下文管理器深度 | `with` 的底层协议、`contextlib.contextmanager` 用生成器简化、`contextlib.ExitStack` 动态管理多个资源、异步上下文管理器 `__aenter__`/`__aexit__` | [With Statement](https://docs.python.org/3/reference/compound_stmts.html#with), [contextlib](https://docs.python.org/3/library/contextlib.html) |
| 3.4 | 描述符协议 | `__get__`/`__set__`/`__delete__`/`__set_name__`、数据描述符 vs 非数据描述符、查找链（`__dict__` → 描述符 → `__getattr__`）、property/staticmethod/classmethod 都是描述符 | [Descriptor Protocol](https://docs.python.org/3/reference/datamodel.html#implementing-descriptors) |
| 3.5 | 并发：线程与 GIL | GIL 为何存在、GIL 在 3.12/3.13 的变化（PEP 703 进展）、`threading` 何时有用、`concurrent.futures` 统一接口、`queue.Queue` 线程通信 | [threading](https://docs.python.org/3/library/threading.html), [Global Interpreter Lock](https://docs.python.org/3/c-api/init.html#thread-state-and-the-global-interpreter-lock) |
| 3.6 | 并发：asyncio 深入 | 事件循环本质、`async/await` 是协程不是线程、Task vs Future、`asyncio.gather` vs `asyncio.TaskGroup`（3.11+）、`asyncio.to_thread`（3.9+）、常见陷阱（阻塞事件循环） | [asyncio](https://docs.python.org/3/library/asyncio.html), [Coroutines and Tasks](https://docs.python.org/3/library/asyncio-task.html) |
| 3.7 | 内存管理与性能 | 引用计数 + 循环检测 GC、`weakref`、`__slots__` 的内存账本、`sys.getsizeof`、`tracemalloc` 分析内存泄漏、`dis` 模块看字节码 | [gc](https://docs.python.org/3/library/gc.html), [weakref](https://docs.python.org/3/library/weakref.html) |
| 3.8 | 元编程 | `exec`/`eval`/`compile`、`type()` 的三参数形式动态创建类、`__prepare__`、`inspect` 模块探查对象、AST 操作简介 | [inspect](https://docs.python.org/3/library/inspect.html), [ast](https://docs.python.org/3/library/ast.html) |

**第三部分思考题**：装饰器和 AOP（面向切面编程）的异同？如何用生成器实现一个简单的"管道"（pipeline）？GIL 在 IO 密集和 CPU 密集场景的表现有何不同？

## 第四部分：库与生态——理解设计，不止于用法

**目标**：不讲 API 大全。每个库回答：解决什么问题、核心设计思想、一个简化实现。

### 4.1 数据处理与数值计算

| 库 | 核心讲解点 | 简化实现 |
|----|-----------|---------|
| **NumPy** | ndarray 的内存布局（连续存储 vs Python list 的指针数组）、广播（Broadcasting）的规则推演、向量化与 ufunc | 用 `ctypes` 实现一个简化版一维 ndarray，展示连续内存 + stride 访问 |
| **Pandas** | DataFrame 的列式存储思想、链式操作与方法链、groupby-apply 的 split-apply-combine 模式、`SettingWithCopyWarning` 的根源 | 用字典 + NumPy 实现一个 50 行的 DataFrame 原型，展示列式选择和广播赋值 |
| **Polars**（选讲） | 惰性求值、查询优化、Arrow 内存格式、与 Pandas 的对比 | — |

### 4.2 Web 框架——从简化实现到生产用法

| 库 | 核心讲解点 | 简化实现 |
|----|-----------|---------|
| **Flask 的本质**（教学用） | WSGI 协议、路由注册（装饰器模式）、请求上下文（`threading.local` 魔法）、中间件链——**理解这些思想后，日常开发直接用 FastAPI** | 写一个 60 行的 "MiniFlask"：WSGI 入口 + 路由表 + `@app.route` 装饰器 + 请求分发 |
| **FastAPI** | ASGI 协议、类型注解驱动参数校验（Pydantic 集成）、依赖注入（Depends）的实现原理、`BackgroundTasks` | 写一个 80 行的 "MiniFastAPI"：路径装饰器 + 类型反射提取参数 + Depends 注入链 |
| **SQLAlchemy** | Unit of Work 模式、Identity Map、Session 与事务边界、`relationship` 的 lazy/eager loading | 写一个 80 行的 "MiniORM"：基于描述符的字段定义 + 元类收集字段 + 简化的 Query Builder |

### 4.3 测试与质量
