# 1.3 容器深度解析

## 要点

1. list= 连续内存的指针数组, 连续, 有序, 按位置紧挨着, 有额外的超额分配

## list 底层：动态数组（Dynamic Array）

Python 的 `list` 在 C 层面是一个**动态数组**——连续内存中存放的是**指向每个元素的指针**，而非元素本身。

```python-run
import sys

lst = []
print(f"空 list 占用: {sys.getsizeof(lst)} 字节")

for i in range(10):
    lst.append(i)
    print(f"元素数={i+1:>2}, getsizeof={sys.getsizeof(lst):>3} 字节")
```

**关键**：`list` 不会每 `append` 一次就 `realloc` ——它采用**超额分配（Over-allocation）**：

```
新容量 ≈ 新长度 + (新长度 >> 3) + (3 if 新长度 < 9 else 6)
```

`append` 的**均摊时间复杂度是 O(1)**，不是 O(n)。

| 操作 | 时间复杂度 | 原因 |
|------|-----------|------|
| `lst[i]` | O(1) | 指针数组，随机访问 |
| `lst.append(x)` | O(1) 均摊 | 超额分配，偶尔搬数据 |
| `lst.insert(0, x)` | O(n) | 所有元素右移一位 |
| `lst.pop()` | O(1) | 只删尾部 |
| `lst.pop(0)` | O(n) | 所有元素左移一位 |
| `x in lst` | O(n) | 线性扫描 |

> `list` 存储的是**指针数组**，元素本身分散在堆上。这跟 C 数组（元素紧挨存放）有本质区别。

## dict 底层：哈希表（Hash Table）

`dict` 的底层是哈希表，分三步：

1. 对 key 调用 `hash(key)` 得到哈希值
2. `index = hash(key) & mask` 映射为内部索引
3. 冲突用**开放寻址法（Open Addressing）**探测下一个位置

```python-run
print(f"hash(42): {hash(42)}")
print(f"hash('hello'): {hash('hello')}")
print(f"hash((1, 2)): {hash((1, 2))}")
try:
    hash([1, 2, 3])  # list 可变 → 不可哈希
except TypeError as e:
    print(f"不可哈希: {e}")
```

> 可变对象不可哈希的原因：如果 key 内容变了，`hash(key)` 就变了，dict 在原来位置找不到它。

### Python 3.6+ 紧凑实现（Compact Dict）

Python 3.6 将 dict 的**索引表**和**数据表**分离：

```
旧实现（3.5 之前）：索引表 = [entry0, entry1, ...]  每个 entry 含 hash/key/value，空洞多

新实现（3.6+）：
  索引表 = [0, -1, 2, 1, ...]    ← 只存 entry 编号（稀疏，每项 1 字节）
  entries = [[hash, key, value], ...] ← 紧密排列，无空洞
```

结果是 Python 3.6+ 的 dict **节省 20-25% 内存**，并且天然保持插入顺序（Python 3.7+ 正式保证）。

```python-run
d = {"c": 3, "a": 1, "b": 2}
print(list(d.keys()))  # ['c', 'a', 'b'] — 按插入顺序
```

## set 与 dict 的关系

`set` 就是**没有 value 的 dict**——内部结构完全一样，`1 in s` 同样是 O(1) 均摊。

> `set` = 只有 key 的 dict；`frozenset` = 只有 key 的不可变 dict。

## tuple 的本质：不只是"不可变 list"

`tuple` 的意义不是"不能修改"，而是**结构语义（Structural Semantics）**——每个位置有固定含义：

```python-run
# tuple 表示"有结构的数据"——坐标、函数多返回值、数据库行
point = (3, 4)             # 0 号位 = x, 1 号位 = y
person = ("Dean", 1990)    # 0 号位 = 名字, 1 号位 = 出生年份
name, year = person        # 结构解包
print(f"{name}, {year}")
```

| | list | tuple |
|--|------|-------|
| 语义 | 同质序列（都是用户、都是分数） | 异质组合（名字+年龄+性别） |
| 长度 | 运行时增减 | 创建时确定 |
| 典型用途 | 循环收集结果、消息队列 | 函数多返回值、dict key、坐标 |

`namedtuple` 让元素有名字，保持 tuple 的不可变性和轻量：

```python-run
from collections import namedtuple
Point = namedtuple("Point", ["x", "y", "z"])
p = Point(1, 2, 3)
print(f"{p.x}, {p[1]}, isinstance tuple: {isinstance(p, tuple)}")
```

## 切片对象：`slice` 是一个真实的类型

`obj[1:5:2]` 被解释器翻译成 `obj[slice(1, 5, 2)]`。`slice` 是可以命名和复用的对象：

```python-run
lst = list(range(10))
print(lst[2:7:2])               # [2, 4, 6]
print(lst[slice(2, 7, 2)])      # 等价
print(lst[slice(None, None, 2)]) # [0, 2, 4, 6, 8] 隔一个取一个
print(slice(2, 100, 3).indices(10))  # (2, 10, 3) 规范化越界
```

> 自定义容器 `__getitem__` 需要处理三种 key：`int`、`slice`、`tuple`（多维索引）。

## `*` 解包的本质：可迭代对象展开

```python-run
# 左侧：捕获剩余元素
a, *b, c = range(5)
print(f"a={a}, b={b}, c={c}")  # a=0, b=[1,2,3], c=4

# 右侧：展开可迭代对象
print([*[1, 2], *[3, 4]])      # [1, 2, 3, 4]

# 函数调用
print(*[1, 2, 3])              # 等价 print(1, 2, 3)

# ** 展开映射（mapping）
d = {"a": 1, "b": 2}
print({**d, "c": 3})           # {'a': 1, 'b': 2, 'c': 3}
```

> `*` 本质上调用的是 `__iter__`，逐个 yield。任何可迭代对象都能被展开。

## 列表推导式 vs 生成器表达式

```python-run
# 列表推导式——立即求值，产生 list
squares_l = [x**2 for x in range(5)]
print(f"list: {squares_l}, type: {type(squares_l)}")

# 生成器表达式——惰性求值，产生 generator
squares_g = (x**2 for x in range(5))
print(f"generator: {squares_g}, type: {type(squares_g)}")
```

| | `[x for x in ...]` | `(x for x in ...)` |
|--|------|-----|
| 求值 | 立即、一次性 | 惰性、逐个 |
| 返回 | `list` | `generator` |
| 内存 | 全部在内存 | 只在迭代时产生一个 |
| 重复遍历 | 可以 | **不可以** |
| 场景 | 结果集小、需多次访问 | 数据量大、只遍历一次 |

## 容器复制：浅拷贝 vs 深拷贝

```python-run
import copy

original = [[1, 2], [3, 4]]

# 浅拷贝——只复制外层容器，内层元素共享
shallow = original[:]         # 或 .copy() 或 copy.copy()
shallow[0][0] = 99
print(f"original: {original}")  # [[99, 2], [3, 4]] —— 被影响了！
print(f"同一内层: {original[0] is shallow[0]}")  # True

# 深拷贝——递归复制所有层
original2 = [[1, 2], [3, 4]]
deep = copy.deepcopy(original2)
deep[0][0] = 99
print(f"\noriginal2: {original2}")   # [[1, 2], [3, 4]] —— 隔离
print(f"同一内层: {original2[0] is deep[0]}")    # False
```

| 方式 | 等价 | 嵌套可变对象 |
|------|------|-------------|
| `lst[:]` | `.copy()` / `copy.copy()` | 共享——修改会互相影响 |
| `[row[:] for row in lst]` | 手动逐层 | 视推导式深度 |
| `copy.deepcopy(lst)` | — | 完全隔离 |

## 经典陷阱：`[[]] * 5`

```python-run
# 陷阱：* 复制的是引用
grid = [[]] * 5
grid[0].append(1)
print(grid)  # [[1], [1], [1], [1], [1]] —— 全变了！
print(f"同一个对象: {grid[0] is grid[1]}")  # True

# 正确：用列表推导式，每次迭代创建新对象
grid = [[] for _ in range(5)]
grid[0].append(1)
print(grid)  # [[1], [], [], [], []]
```

> `[[]] * 5` 把**同一个空 list 的引用**复制 5 份。内部元素是可变对象时，永远用推导式或 `copy`，别用 `*` 复制。

## 总结：容器的三个本质问题

1. **是什么**：`list` = 动态指针数组，`dict` = 哈希表（3.6+ 紧凑），`set` = 没有 value 的 dict，`tuple` = 有结构语义的不可变序列
2. **为什么需要**：不同访问模式选不同容器——随机访问选 list/dict，有序不重复选 set，结构化数据选 tuple
3. **怎么用对**：`append` 是均摊 O(1)；嵌套可变对象小心浅拷贝；`*` 复合可变对象用推导式不用乘法

## 本章小结

- `list` 底层是动态指针数组，超额分配使 `append` 均摊 O(1)；`insert(0, x)` 需移动所有元素，是 O(n)
- `dict` 底层是哈希表，Python 3.6+ 紧凑实现节省 20-25% 内存且保持插入顺序（3.7+ 正式保证）
- `set` 是没有 value 的 dict；`tuple` 不是"不可变 list"，而是有结构语义的异质组合
- `slice` 是一个真实的类型对象，`obj[a:b:c]` 等价于 `obj[slice(a, b, c)]`
- 浅拷贝只复制外层容器（内层共享），深拷贝递归复制所有层；`[[]] * 5` 复制的是同一个空列表的引用

## 思考题

1. 为什么 `lst.insert(0, x)` 比 `lst.append(x)` 慢得多？从动态数组底层搬迁成本解释。
:::tip 答案
`insert(0, x)` 必须把列表里所有的已存在的元素全部向后挪动一位, 时间复杂度`O(n)`

`append(x)` 直接放在最后,不用挪动任何元素, 事件复杂度`O(1)`
:::

:::tip 答案
**容器的可哈希性 = 里面所有元素都必须可哈希**;

tuple不可变,但是list是可变的;
