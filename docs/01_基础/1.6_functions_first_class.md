# 1.6 函数是一等公民

## 函数即对象：揭开 `def` 的真面目

`def` 不是"声明一个函数"，而是**执行一段代码、创建一个函数对象、然后把这个对象绑定到一个名字上**。

```python-run
def greet(name):
    """向某人问好。"""
    return f"Hello, {name}!"

print(type(greet))                    # <class 'function'>
print(greet.__name__)                 # greet
print(greet.__doc__)                  # 向某人问好。
print(greet("World"))                 # 正常调用
```

既然是对象，它就有属性。下面把函数从里到外扒开看：

```python-run
def add(a: int, b: int = 0) -> int:
    """返回两数之和。"""
    return a + b

# 函数"皮肤" —— 元信息
print("名字:", add.__name__)            # add
print("文档:", add.__doc__)             # 返回两数之和。
print("模块:", add.__module__)          # __main__ (当前交互环境)

# 函数"骨架" —— 代码结构的只读映射
print("局部变量名:", add.__code__.co_varnames)  # ('a', 'b')
print("参数数量:", add.__code__.co_argcount)     # 2

# 函数"脂肪" —— 运行时可修改的东西
print("默认值:", add.__defaults__)      # (0,)
print("类型注解:", add.__annotations__) # {'a': int, 'b': int, 'return': int}
```

> **关键洞察**：`__code__` 是 code 对象（编译后的字节码快照），**不可变**；`__defaults__` 和 `__annotations__` 是普通 Python 对象，**可以运行时修改**——这为后面讲默认参数陷阱埋下了伏笔。

## 函数的魔术属性：function 对象拆解

上一节只是浅览了函数对象的几个属性。下面是完整的解剖图——把 `function` 对象从内到外拆开看。

### 完整属性清单

```python-run
def counter(base: int, *, step: int = 1) -> int:
    """每次返回 base + step * n。"""
    n = 0
    def inner():
        nonlocal n
        n += 1
        return base + step * n
    return inner

c = counter(10, step=5)
c(); c()  # 调用两次

# ── 元信息 ──
print("__name__:     ", c.__name__)       # inner — 函数定义时的名字,这里是inner 没有使用functools.wrap包裹
print("__qualname__: ", c.__qualname__)   # counter.<locals>.inner — 完整限定名
print("__doc__:      ", c.__doc__)        # None — inner 没有写文档字符串
print("__module__:   ", c.__module__)     # __main__

# ── 代码结构 ──
print("__code__.co_varnames:", c.__code__.co_varnames)  # ()
print("__code__.co_freevars:", c.__code__.co_freevars)  # 自由变量,也就闭包捕获的变量 ('base','n','step')

# ── 闭包数据 ──
for i, cell in enumerate(c.__closure__):
    print(f"__closure__[{i}]: {c.__code__.co_freevars[i]} = {cell.cell_contents}")

# ── 运行时可变状态 ──
print("__defaults__:   ", c.__defaults__)    # None — inner 没有位置默认参数
print("__kwdefaults__: ", c.__kwdefaults__)  # None — inner 没有关键字默认参数
print("__globals__:    ", type(c.__globals__))  # <class 'dict'> — 模块全局命名空间

```

### 属性分类速查

| 属性 | 类型 | 读写？ | 含义 |
|------|------|--------|------|
| `__name__` | str | 可写 | 函数定义时的名字 |
| `__qualname__` | str | 可写 | 完整限定名（含类和闭包层级，PEP 3155） |
| `__doc__` | str \| None | 可写 | 文档字符串 |
| `__module__` | str | 可写 | 函数定义所在模块名 |
| `__code__` | code | 只读* | 编译后的字节码对象 |
| `__globals__` | dict | 只读 | 函数全局命名空间的引用 |
| `__closure__` | tuple[cell] \| None | 只读 | 闭包捕获的自由变量，普通函数为 `None` |
| `__defaults__` | tuple \| None | 可写 | 位置参数默认值 |
| `__kwdefaults__` | dict \| None | 可写 | keyword-only 参数默认值 |
| `__annotations__` | dict \| None | 可写 | 类型注解 |
| `__dict__` | dict | 可写 | **函数自己的属性字典**——可以挂任何东西 |
| `__wrapped__` | object \| None | 可写 | 被 `@wraps` 装饰时指向原函数 |

> **`__globals__` 是关键**：每个函数对象持有一个指向它定义所在模块的 `globals()` 字典的引用。这就是为什么嵌套函数能访问模块级全局变量——不是走 LEGB 查找，而是直接走 `__globals__['name']`。

### 函数也可以有自己的属性

因为函数是对象，所以**可以为函数动态添加属性**：

```python-run
def process(data):
    process.call_count += 1   # 函数自己的属性！
    return [x * 2 for x in data]

process.call_count = 0        # 挂了一个属性在 process.__dict__ 上

process([1, 2, 3])
process([4, 5])
print(f"process 被调用了 {process.call_count} 次")  # 2
print(process.__dict__)       # {'call_count': 2}
```

> 这比定义类再用 `self.count` 轻量得多。Django 的信号系统、Flask 的路由注册都利用了函数属性的灵活性。但对于需要多个状态变量或会被多处修改的复杂场景，类装饰器比函数属性更清楚。

## 函数的魔术方法：`__call__` 和 `__get__`

上面讲的都是**属性**。下面两个是**方法**——定义了函数作为对象的核心行为。

### `__call__`：函数为什么能"被调用"

`()` 操作符其实就是调用 `type(obj).__call__(obj, *args, **kwargs)`。函数之所以能 `f(x)`，是因为 `function` 类型实现了 `__call__`：

```python-run
def add(a, b):
    return a + b

# 这两种写法等价：
print(add(1, 2))                       # 3
print(type(add).__call__(add, 1, 2))   # 3 — 这就是 () 的底层实现
```

`function.__call__` 内部由 CPython 的 `PyFunction_Type` 的 `tp_call` 槽位实现，逻辑是：创建新的栈帧 → 用参数填充 `f_locals` → 执行 `__code__` 字节码 → 返回结果或抛出异常。

> 任何实现了 `__call__` 的类都能像函数一样被调用——这就是"可调用对象（Callable）"的完整定义。函数只是 `__call__` 的一个特例。

### `__get__`：`self` 为什么能自动绑定

**函数是描述符（Descriptor）**。这是 Python 对象模型中最容易被忽略的设计——但搞懂了它，`self` 的来源就一清二楚了。

```python-run
class Dog:
    def bark(self):
        return f"汪汪！我是 {self.name}"

# 1. 通过类访问 → 获取原始 function 对象
print(type(Dog.bark))  # <class 'function'>

# 2. 通过实例访问 → 获取 bound method
d = Dog()
d.name = "旺财"
print(type(d.bark))    # <class 'method'>

# 3. 手动触发 __get__，和实例访问等价
bound = Dog.bark.__get__(d, Dog)
print(type(bound))     # <class 'method'>
print(bound())         # 汪汪！我是 旺财
```

**发生了什么？**

```
d.bark                          ① Python 发现 bark 是函数，有 __get__
  → Dog.bark.__get__(d, Dog)    ② 调用函数的描述符协议
    → MethodType(bark, d)       ③ 返回一个绑定方法对象
      → 调用时自动把 d 作为第一个参数传入
```

这就是 `self` 的来源：**不是语言特殊处理，而是函数的 `__get__` 描述符协议自动完成的**。当你写 `d.bark()`，Python 的后台等价于 `Dog.bark.__get__(d, Dog)()`。

```python-run
# @staticmethod 和 @classmethod 之所以行为不同，正是通过 __get__ 控制的
class Demo:
    def regular(self):   return "实例方法"
    @classmethod
    def cm(cls):         return "类方法"
    @staticmethod
    def sm():            return "静态方法"

# 三者的 __get__ 行为完全不同
d = Demo()
print(type(Demo.regular.__get__(d, Demo)))  # <class 'method'> — 绑定实例
print(type(Demo.cm.__get__(d, Demo)))       # <class 'method'> — 绑定类
print(type(Demo.sm.__get__(d, Demo)))       # <class 'function'> — 原样返回

# 手动模拟三者的访问
print(Demo.regular.__get__(d, Demo)())      # 实例方法：传入 d
print(Demo.cm.__get__(d, Demo)())           # 类方法：传入 Demo
print(Demo.sm.__get__(d, Demo)())           # 静态方法：什么都不传入
```

> **一句话总结**：`function` 类型实现了 `__get__`，让函数成为非数据描述符。`@classmethod` 和 `@staticmethod` 替换了函数的 `__get__` 行为——它们不是"语言特性"，而是描述符协议的不同实现。

### 从函数到属性：链接到描述符

函数的 `__get__` 是描述符协议的入口。理解了这一点，就能自然延伸到 `@property`（数据描述符）、`__slots__`（描述符）、ORM 的字段定义——它们用的都是同一个协议。详见 [描述符协议（3.4）](../03_高级主题/3.4_描述符协议)。

## 一等公民到底是什么意思？

**一等公民（First-Class Citizen）** 这个概念中文翻译容易让人觉得抽象。一句话：一个编程实体如果满足三个条件，它就是一等公民：

| 条件 | 函数能做到吗？ | 证据 |
|------|---------------|------|
| 可以赋值给变量 | 可以 | `f = len` |
| 可以作为参数传递 | 可以 | `sorted(data, key=len)` |
| 可以作为返回值 | 可以 | 闭包、装饰器 |

```python-run
# 条件 1：赋值给变量
f = print
f("我被另一个名字调用了!")

# 条件 2：作为参数传递
def apply(func, value):
    return func(value)

print(apply(str.upper, "hello"))      # HELLO

# 条件 3：作为返回值（生产新函数）
def make_multiplier(n):
    return lambda x: x * n

double = make_multiplier(2)
print(double(21))                     # 42
```

> 在 Python 中，函数和整数、字符串一样，是**普通的运行时对象**——这就是"一等公民"的全部含义。也正是因为函数是对象，才有了装饰器、闭包、回调这些高层抽象。

## `*args` 和 `**kwargs`：名字背后的原理

很多人会用 `*args` / `**kwargs`，但说不清楚底层到底发生了什么。本质是两步：**定义侧打包** + **调用侧解包**。

### 定义侧：`*` 把多余的位置参数"吞"成元组，`**` 把多余的关键字参数"吞"成字典

```python-run
def show(name, *args, **kwargs):
    print(f"name: {name!r}")
    print(f"args: {args!r}")
    print(f"kwargs: {kwargs!r}")

show("Dean", 1, 2, 3, city="NYC", score=100)
# name: 'Dean'
# args: (1, 2, 3)
# kwargs: {'city': 'NYC', 'score': 100}
```

### 调用侧：`*` 把可迭代对象"炸开"为位置参数，`**` 把映射"炸开"为关键字参数

```python-run
def connect(host, port, timeout=30):
    print(f"连接 {host}:{port}, 超时 {timeout}s")

# * 解包任何可迭代对象（列表、元组、生成器...）
config = ("db.example.com", 5432)
connect(*config)

# ** 解包任何映射对象
opts = {"host": "db.example.com", "port": 5432, "timeout": 10}
connect(**opts)
```

> 名字 `args` / `kwargs` 只是**约定**，不是语法要求。`*whatever` / `**whatever` 都行，但遵守约定能让任何 Python 程序员一眼看懂意图。

## 参数顺序：五种参数，一个规则

Python 3 的函数签名中，参数分五类，**必须按以下顺序出现**：

| 种类 | 语法 | 调用时传入方式 |
|------|------|---------------|
| positional-only | `f(a, b, /)` | 只能按位置 |
| positional-or-keyword | `f(a, b)` | 位置或关键字均可（默认） |
| var-positional | `f(*args)` | 攒成元组 |
| keyword-only | `f(*, a, b)` | 只能按关键字 |
| var-keyword | `f(**kwargs)` | 攒成字典 |

### 用 `/` 强制位置传参（Python 3.8+）

`/` 左边的参数不能以关键字方式传入：

```python-run
def describe(obj, /, *, detail=False):
    """obj 必须按位置传，detail 必须按关键字传。"""
    base = f"<{type(obj).__name__}>"
    return f"{base} {obj}" if detail else base

print(describe(42))                    # <int>
print(describe(42, detail=True))       # <int> 42
# describe(obj=42)                     # ❌ TypeError: 不允许关键字传参
# describe(42, True)                   # ❌ TypeError: detail 必须用关键字
```

为什么需要 `/`？典型场景：`len`、`str.startswith` 这类参数名本身就无意义的名字，没必要暴露给调用者去写关键字。

### 用 `*` 强制关键字传参

`*` **之后**的所有参数都是 keyword-only：

```python-run
def load_config(path, *, encoding="utf-8", timeout=30):
    """path 可以位置传，其余必须用关键字。"""
    print(f"加载 {path}: {encoding}, {timeout}s")

load_config("/etc/app.json", timeout=10, encoding="gbk")
# load_config("/etc/app.json", "gbk", 10)  # ❌ TypeError
```

强制关键字传参让调用侧代码可读性更高——看到参数名就知道每个值的含义，这是 `open(file, mode, buffering, encoding, ...)` 真正需要的设计。

### 完整签名实战

```python-run
def full_demo(a, b, /, c, d, *args, e, f, **kwargs):
    """a, b → positional-only    (Python 3.8+ 用 / 标记)
       c, d → positional-or-keyword
       *args → 吞掉所有剩余位置参数（不包括 keyword-only）
       e, f → keyword-only       (在 *args 之后)
       **kwargs → 吞掉所有剩余关键字参数
    """
    print(f"pos-only:   a={a}, b={b}")
    print(f"pos-or-kw:  c={c}, d={d}")
    print(f"var-pos:    args={args}")
    print(f"kw-only:    e={e}, f={f}")
    print(f"var-kw:     kwargs={kwargs}")

full_demo(1, 2, 3, 4, 5, 6, 7, e=8, f=9, x=10, y=11)
```

## 默认参数陷阱：Python 面试第一题

```python-run
def add_to_list(item, target=[]):
    target.append(item)
    return target

print(add_to_list(1))  # [1]
print(add_to_list(2))  # [1, 2]  ← ？？？
print(add_to_list(3))  # [1, 2, 3]
```

**根因**：`def` 语句被执行时，默认值表达式**立即求值一次**，产生的对象存在 `func.__defaults__` 里。每次调用如果没传参，复用的是**同一个对象**。

验证：

```python-run
def bad(lst=[]):
    lst.append(1)
    return lst

print(bad.__defaults__)  # ([],)
bad()
print(bad.__defaults__)  # ([1],)  ← 默认值对象被原地修改了！
```

### 正确模式：`None` 哨兵 + 函数体内初始化

```python-run
def add_to_list(item, target=None):
    if target is None:
        target = []            # 每次调用单独创建一个新列表
    target.append(item)
    return target

print(add_to_list(1))  # [1]
print(add_to_list(2))  # [2]  ← 符合预期
```

> **规则**：只要默认值是可变对象（`list`、`dict`、`set`、自定义类实例），一律用 `None` 作为哨兵，在函数体里初始化。这条规则回避了 90% 的默认参数 bug。

## `inspect.signature()`：用代码读代码

不想手动解析 `__code__` 和 `__defaults__`？标准库 `inspect` 提供人类可读的函数签名内省：

```python-run
import inspect

def fetch(url, /, *, method="GET", timeout=30, headers=None):
    """发起 HTTP 请求。"""
    pass

sig = inspect.signature(fetch)
print(sig)  # (url, /, *, method='GET', timeout=30, headers=None)

for name, param in sig.parameters.items():
    kind = param.kind.name          # POSITIONAL_ONLY, KEYWORD_ONLY, ...
    default = param.default
    default_str = f" = {default!r}" if default is not inspect.Parameter.empty else ""
    print(f"  {name}: {kind}{default_str}")
```

输出一目了然——`/` 和 `*` 都在，参数的类型标记清清楚楚。这在动态框架（FastAPI、click）中被大量用于自动生成 CLI 接口和参数校验。

## `functools.partial`：锁定一部分参数

**部分应用（Partial Application）** 的意思是：给定一个函数和一部分参数，返回一个"已经把那些参数填好了"的新函数：

```python-run
from functools import partial

def log(level, module, message):
    print(f"[{level}] {module}: {message}")

info = partial(log, "INFO")
sql_info = partial(info, "database")

sql_info("连接池已初始化")
# [INFO] database: 连接池已初始化

# inspect 查签名：可以看到 fixed 参数
print(inspect.signature(sql_info))  # (message)
```

> `partial` 的底层用闭包存储被绑定的参数，但直接用 `partial` 比手写闭包更具声明性、更可内省。

## 类型注解与 `Callable`

函数既然是对象，就可以用类型标注它的"形状"——接受什么参数、返回什么值：

```python-run
from collections.abc import Callable

# 定义一个"注册器"，要求回调必须是 Callable 类型
def register_handler(name: str, handler: Callable[[int, str], bool]) -> None:
    """handler 接受 (int, str)，返回 bool。"""
    ...

# 满足签名的函数
def my_handler(code: int, msg: str) -> bool:
    return code >= 200

register_handler("check", my_handler)  # 类型检查通过
```

`Callable[[args...], return_type]` 的语法：`[[参数1类型, 参数2类型], 返回值类型]`。

> `Callable` 是**结构化类型（Structural Type）** / 鸭子类型的一种体现——只要一个对象能用 `()` 调用且签名匹配，它就满足 `Callable`。类实现了 `__call__` 也算。

## 本章小结

- `def` 不是"声明函数"，而是执行代码、创建函数对象、绑定到名字——函数和整数一样是普通运行时对象，可以挂任意属性
- 函数对象有丰富的魔术属性：`__code__`（字节码）、`__globals__`（全局命名空间引用）、`__closure__`（闭包变量）、`__qualname__`（完整限定名）
- `__call__` 是函数可被调用的根源——任何实现 `__call__` 的类都能像函数一样使用
- `__get__` 让函数成为描述符——`d.bark()` 等价于 `Dog.bark.__get__(d, Dog)()`，`self` 就是这样自动绑定的
- `@staticmethod` 和 `@classmethod` 本质上是通过替换 `__get__` 的行为来改变方法绑定方式
