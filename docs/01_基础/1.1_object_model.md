# 1.1 Python 对象模型

## 知识点
1. 一切皆对象, 所有对象都存储在堆上,每一个对象都有独立的身份和类型
2. 小整数和简单字符串内存驻留
3. 可变不可变对象, 不可变对象修改意味着创建新对象
4. 参数的参数一般不要使用可变参数,使用None标记每次调用时初始化

## 一切皆对象

**一切都是对象**（Everything is an Object）。整数、字符串、函数、类、模块、甚至 `type` 自身——统统是对象，存储在堆上，拥有独立的身份和类型。
+ 函数是对象: 所以它可以被赋值给变量, 也可以作为参数传给另一个函数, 也可以作为返回值.这是装饰器能存在的根本前提.
+ 类是对象: 类本身是`type` 这个元类创建的实例. 所以类也可以被当做参数传递(比如装饰器类), 或者在运行时动态创建

**所有对象都存储在堆上**,变量只时一个便利贴.
> 这是Python和C/C++等语言最大的不同, python的本质就是贴标签, 而不是拷贝数据

+ 当你执行a=100时, Python在堆内存Heap中开辟一块空间,把整数`100`放进去, 然后再`stack` 创建名为`a` 的标签,贴在这个100对象上
+ 当你指向 `b = a` , python不会复制`100`,而是直接把`b` 这个标签 也贴在`100` 这个对象上

**每一个对象都有独立的身份和类型** . 通过`id` 和`type` 
+ 身份: 通过id()查看, 其实就是内存中的地址.
+ 类型type: 通过type()查看, 查看类型



## `id()`、`is`、`==` 的区别

这三个东西回答不同的问题：

| | 问题 | 比较什么 |
|--|------|---------|
| `id(obj)` | 这个对象"住在哪"？ | 返回对象在**当前生命周期内**的唯一整数标识（CPython 中是内存地址） |
| `a is b` | a 和 b 是**同一个**对象吗？ | 比较 `id(a) == id(b)` ——**身份相等** |
| `a == b` | a 和 b 的**值**相等吗？ | 调用 `a.__eq__(b)` ——**值相等** |

```python-run
a = [1, 2, 3]
b = [1, 2, 3]       # 值相同的另一个 list
c = a               # c 和 a 指向同一个对象

print(f"a is b: {a is b}")     # False — 不同对象
print(f"a == b: {a == b}")     # True  — 值相同
print(f"a is c: {a is c}")     # True  — 同一个对象
print(f"id(a): {id(a)}, id(b): {id(b)}")
```

> **核心规则**：和 `None` 比较永远用 `is`，因为 `None` 是单例。其他情况默认用 `==`——除非你**真的**关心两个名字是否指向同一个对象。

## 小整数缓存与字符串驻留

为了提高性能，CPython 会**预创建并缓存**一些常用的不可变对象。

### 小整数缓存（Small Integer Caching）

CPython 启动时就把 **-5 到 256** 的整数对象创建好了。任何用到这些值的变量都指向**同一个对象**：

```python-run
a = 256
b = 256
print(f"256 is 256: {a is b}")  # True — 指向缓存的同一个对象

a = 257
b = 257
print(f"257 is 257: {a is b}")  # False — 每次都创建新对象（交互式环境）

# 注意：同一行 / 同一函数内，编译器可能会优化字面量
def test():
    x = 257
    y = 257
    return x is y

print(f"同一函数内 257 is 257: {test()}")  # 可能为 True（编译器优化）
```

> 这个缓存范围是 **CPython 实现细节**，不要在生产代码中依赖 `is` 比较整数值。Jython、PyPy 的行为可能不同。

### 字符串驻留（String Interning）

类似地，看起来像标识符的字符串会被"驻留"——相同内容的字符串共享同一个对象：

```python-run
a = "hello"
b = "hello"
print(f"'hello' is 'hello': {a is b}")  # True — 自动驻留

# 包含特殊字符的字符串不会自动驻留
c = "hello world!"
d = "hello world!"
print(f"'hello world!' is 'hello world!': {c is d}")  # 可能为 False

# 显式驻留
import sys
e = sys.intern("hello world!")
f = sys.intern("hello world!")
print(f"intern 后: {e is f}")  # True
```

## 可变 vs 不可变：Python 对象最核心的分界线

Python 的类型分为两派：

| | 可变（Mutable） | 不可变（Immutable） |
|--|---------------|-------------------|
| **能原地修改吗？** | 能——对象还是同一个，内容变了 | 不能——"修改"意味着创建新对象 |
| **`__hash__` 呢？** | 不能哈希（`unhashable`） | 可以哈希 |
| **能当 dict 的 key 吗？** | 不能 | 能 |
| **典型代表** | `list`, `dict`, `set`, `bytearray` | `int`, `float`, `str`, `tuple`, `frozenset`, `bytes` |

### "修改"不可变对象 = 创建新对象

```python-run
s = "hello"
print(f"修改前: id(s) = {id(s)}")

s = s + " world"    # 看起来是"修改"，实际是创建了新 str
print(f"修改后: id(s) = {id(s)}")  # id 变了！这是另一个对象

# 对比：list 是原地修改
lst = [1, 2, 3]
print(f"修改前: id(lst) = {id(lst)}")

lst.append(4)       # 原地修改，对象同一
print(f"修改后: id(lst) = {id(lst)}")  # id 不变！
```

> 这个区别直接影响性能：对字符串做大量的 `+` 拼接会不断创建新对象（O(n^2)内存分配），而 `list.append` 是 O(1) 摊销。字符串拼接多了要用 `''.join()`。

### tuple 的"不可变"只是浅层的

```python-run
t = (1, 2, [3, 4])      # tuple 本身不可变，但它包含的 list 可变
print(f"修改前: {t}")

t[2].append(5)           # 修改的是 tuple 内部的 list，不改变 tuple 本身
print(f"修改后: {t}")    # (1, 2, [3, 4, 5]) — tuple 的"元素"没变

# t[0] = 10  # TypeError! tuple 的元素引用不能被重新绑定
```

> tuple 的不可变指的是**元素引用的集合**不可变——你不能把 `t[0]` 从指向 1 改成指向 10。但如果某个元素本身是可变对象，那个可变对象的内部状态是可以改的。

## 可变默认参数：经典陷阱

```python-run
def add_item(item, lst=[]):     # 默认参数在函数定义时只求值一次！
    lst.append(item)
    return lst

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2] —— 咦？默认值不是应该每次都是空列表吗？
print(add_item(3))  # [1, 2, 3] —— 越积越多！
```

> **根源**：函数的默认参数在 `def` 语句执行时**只求值一次**。`lst=[]` 创建的那个空列表是一个**固定的可变对象**，所有调用共享它。`def` 不是"每次调用时赋值"，而是"定义时绑定"。

**正确写法**——用 `None` 当哨兵值（Sentinel Value）：

```python-run
def add_item(item, lst=None):
    if lst is None:
        lst = []          # 每次调用都创建新的空列表
    lst.append(item)
    return lst

print(add_item(1))  # [1]
print(add_item(2))  # [2] —— 正确！
print(add_item(3))  # [3]
```

## 本章小结

- 一切皆对象：整数、函数、类、type 自身都是对象，存储在堆上，拥有 id、type 和 __dict__
- `is` 比较身份（id），`==` 比较值（`__eq__`），和 `None` 比较永远用 `is`
- 引用计数是 Python 主要的内存回收机制，`del` 删除的是名字不是对象；分代 GC 处理循环引用
- 可变对象（list、dict、set）原地修改、不可哈希；不可变对象（int、str、tuple）的"修改"实际是创建新对象
- 函数默认参数在 `def` 时只求值一次——可变默认值用 `None` 哨兵模式避免共享陷阱
