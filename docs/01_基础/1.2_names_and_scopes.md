# 1.2 名字与作用域

## `=` 不是赋值，是名字绑定

Python 中 `=` 的真正含义是**名字绑定（Name Binding）**：在命名空间中创建一个名字，把它指向某个对象。

```python-run
x = [1, 2, 3]
y = x              # 给同一个对象贴第二个标签
y.append(4)
print(x)           # [1, 2, 3, 4] —— x 和 y 贴的是同一个对象
```

> `=` 从不复制对象。它只是在当前作用域里建立"名字 → 对象"的映射。

```python-run
a = "hello"
b = a              # b 指向同一个字符串
a = "world"        # a 被重新绑定到新字符串，b 不受影响
print(a, b)        # world hello
```

## LEGB 规则：Python 去哪找名字

当 Python 遇到一个名字时，按严格顺序在四个作用域（Scope）中查找：

**Python 有两套完全独立的命名空间：**
1. 直接变量名访问：遵循 LEGB 规则，编译时静态确定作用域，和对象无关。
2. 对象.属性访问：走 __getattribute__，是动态属性查找，属于对象系统。


| 作用域 | 英文 | 含义 |
|--------|------|------|
| **L**ocal | 局部 | 函数内部 |
| **E**nclosing | 闭包外层 | 外层函数的局部作用域 |
| **G**lobal | 全局 | 模块级别 |
| **B**uilt-in | 内置 | `print`、`len`、`range` 等 |

> 规则：**从内向外，找到就停**。内层同名变量会"遮蔽（Shadowing）"外层。

```python-run
len = "global_len"          # G: 覆盖了内置的 len（仅演示用，别这样写）

def outer():
    len = "enclosing_len"   # E
    def inner():
        len = "local_len"   # L
        print(len)
    inner()

outer()
del len                     # 恢复内置 len
```

逐层注释掉 `len` 的定义，感受 LEGB 的查找过程。

## `global` —— 越过局部作用域

函数内**读取**全局变量不用声明，但**赋值**会创建同名局部变量：

```python-run
count = 0

def increment():
    global count     # 告诉 Python：下面用的是全局的 count
    count += 1

increment()
print(count)         # 1
```

去掉 `global count` 会怎样？

```python-run
count = 0

def increment():
    # count += 1  # UnboundLocalError! 赋值使 count 成为局部变量，但右侧未定义
    pass
```

> **规则**：函数内只要对某名字做了绑定操作(比如在某个函数里给某个变量赋值,不管出现在第几行, 他都是局部变量, 除非通过关键字`global`,`nonlocal`排除)（`=` 左侧、`import`、`for` 循环变量等），编译阶段 Python 就把该名字标记为局部变量，不再向外查找。
> 
> `global` 就是在说："这个绑定请作用于全局作用域"。

---

## `nonlocal` —— 给外层函数变量的"写入权"

```python-run
def make_counter():
    count = 0
    def increment():
        nonlocal count   # 找最近 Enclosing 作用域里的 count
        count += 1
        return count
    return increment

c = make_counter()
print(c())  # 1
print(c())  # 2
```

`nonlocal` 逐层向外（跳过 Local），找到最近一层 Enclosing 作用域的同名变量。去掉它同样会因为 `count += 1` 引发的赋值而报 `UnboundLocalError`。

> 更多闭包话题见 [闭包深度解析](closures_deep.md)，本章只概述 `nonlocal` 在 LEGB 中的作用。LEGB 只解释了"变量"的查找规则——想理解变量和属性（`obj.x`）的本质区别，见 [命名空间深度解析](namespaces_deep.md)。

## `del` —— 删的是名字，不是对象

```python-run
a = [1, 2, 3]
b = a               # 两个标签，一个列表
del a               # 删除名字 a，列表对象仍存活
print(b)            # [1, 2, 3]
# print(a)          # NameError: name 'a' is not defined
```

> `del` 的语义是**解除名字绑定**（减少一个引用计数），不是销毁对象。对象在没有任何名字指向它时才由 GC 回收。

`del` 还能删除容器元素和切片：

```python-run
lst = [10, 20, 30, 40]
del lst[1:3]
print(lst)  # [10, 40]

d = {"a": 1, "b": 2}
del d["a"]
print(d)    # {'b': 2}
```

## 作用域陷阱：编译阶段就决定了

```python-run
x = 10

def bad():
    print(x)   # 想读全局 x？也报错！
    x = 20     # ← 函数体内有 x 的赋值，整个函数的 x 都是局部变量

# bad()        # UnboundLocalError
```

Python 在**编译函数时**就确定了每个名字属于哪个作用域，不是运行时一行行判断。函数体内只要存在对 `x` 的绑定——哪怕它在 `print(x)` 之后——整个函数里的 `x` 都被视为局部变量。

解法：`global x`（或 `nonlocal x`），明确声明名字的来源。

## `globals()` 和 `locals()` —— 运行时内省

+ locals() 永远取：当前调用帧,也就是当前栈顶那个帧的命名空间字典(临时复制的),通常只读;全局时指向globals() 的返回,此时可修改
+ globals() 整个模块的全局命名空间字典(自身,修改有效)
```python-run
def show():
    a, b = 1, "hello"
    print("局部变量:", dict(locals()))

show()

# 模块级别：globals() 和 locals() 是同一个字典
print(globals() is locals())  # True（在模块作用域）
```

> 在函数内部，`locals()` 返回的是该调用帧命名空间的**快照拷贝**——修改它不影响实际变量(通常是只读的)。

## 本章小结

- `=` 是名字绑定而非赋值：在当前作用域创建名字到对象的映射，从不复制对象
- LEGB 规则：从内向外（Local → Enclosing → Global → Built-in）查找名字，找到即停
- `global` 让函数内赋值作用于全局作用域；`nonlocal` 让闭包内赋值作用于外层 Enclosing 作用域
- `del` 删除的是名字（解除绑定），不是对象——对象在引用计数归零时才被回收
- Python 在编译阶段确定每个名字的作用域——函数体内只要有赋值，该名字就被标记为局部变量

## 思考题

1. `def f(): x = 10; def g(): print(x); return g` —— `f()()` 输出什么？这里的 `x` 在 LEGB 的哪一层？
> 这其实是一个经典的闭包函数, 打印 10; x被保存在 Enclosing闭包中;
2. 模块级别 `globals() is locals()` 是 `True`，函数内部是什么？为什么？
> 模块级别：globals() is locals() → True（同空间）
> 
> 函数内部：globals() is locals() → False（独立隔离空间）
>
> 修改全局变量→ `global`
>
> 修改嵌套/闭包变量 → `nonlocal` 

3. `x = 5; def f(): print(x); x = 10; f()` —— 报什么错？错误发生在哪一行？用编译时的"名字归类"规则解释。
> `UnboundLocalError: local variable 'x' referenced before assignment 报错 print(x)
> 编译函数时,只要对变量做了赋值操作,就被标记为局部变量; 这里x 被标记为局部变量, 运行时先执行print(x),局部x还没有值导致报错;

1. `del lst[0]` 和 `del lst` 分别做了什么？各自影响的是名字还是对象？
> `del lst[0]: 触发 `__delitem__`,删除列表对象中的一个元素, 操作对象, 名字lst不受影响
>
> `del lst` 删除当前作用域中的名字lst, 操作的是命名空间, 对象会在无人引用后被垃圾回收
