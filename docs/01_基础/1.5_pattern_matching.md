# 1.5 模式匹配

## 不是 switch-case，是模式匹配

Python 3.10 引入的 **模式匹配（Pattern Matching）** 表面上像 C/Java 的 `switch-case`，但本质完全不同：它不是在做"值相等"的比较，而是在做"结构拆解 + 类型检查 + 值绑定"的**声明式编程**。

```python-run
# 这不是 switch——是"把数据按形状拆开"
def analyze(x):
    match x:
        case 0:
            return "零"
        case int(n) if n > 0:          # 类型 + 守卫
            return f"正整数 {n}"
        case int(n) if n < 0:
            return f"负整数 {n}"
        case str(s):
            return f"字符串，长度 {len(s)}"
        case [first, *rest]:            # 解构列表
            return f"列表，首元素 {first}，剩余 {rest}"
        case _:
            return "其他"

print(analyze(42))
print(analyze([1, 2, 3]))
```

> `case _` 是**通配符模式（Wildcard Pattern）**，匹配任何值且不绑定。它相当于其他语言中的 `default`，但更精准——`_` 在 Python 中约定表示"我不关心这个值"。

## 字面量模式与变量模式

最基础的两类模式：匹配具体值和绑定值。

```python-run
def describe(val):
    match val:
        case 1:             return "字面量匹配：1"
        case "hello":       return "字面量匹配：hello"
        case True:          return "字面量匹配：True（注意：True 不会匹配 1！）"
        case x:             return f"绑定到变量 x = {x!r}"  # 最后一个分支，捕获一切

print(describe(1))
print(describe("hello"))
print(describe(True))
print(describe([1, 2]))
```

> match case 严格区分字面量和变量名：`case x` 永远是**绑定变量**，不是把输入和已有的 `x` 比较。只有 `case "hello"`、`case 42` 这种字面值才是值比较。

---

## 序列模式——解构列表/元组

序列模式（Sequence Pattern）可以像解包一样把列表或元组"拆"成各个部分：

```python-run
def process_command(cmd):
    match cmd:
        case ["quit"]:
            return "退出"
        case ["echo", msg]:                   # 两元素列表
            return f"回显: {msg}"
        case ["echo", *msgs]:                 # echo 后面跟任意多个参数
            return f"回显: {' '.join(msgs)}"
        case ["save", filename]:              # 保存到指定文件
            return f"保存到 {filename}"
        case ["save", filename, "--force"]:   # 强制保存
            return f"强制保存到 {filename}"
        case _:
            return "未知命令"

print(process_command(["echo", "hello", "world"]))
print(process_command(["save", "data.txt", "--force"]))
print(process_command(["quit"]))
```

> 序列模式对 `list` 和 `tuple` 都有效，甚至对任何实现了 `__getitem__` 的序列都有效。`*rest` 语法和非匹配上下文中的 `*` 解包是完全一致的精神。

## 映射模式——解构字典

映射模式（Mapping Pattern）按 **key** 来匹配和提取字典中的值：

```python-run
def handle_api(data):
    match data:
        case {"type": "text", "content": content}:
            return f"文本消息: {content}"
        case {"type": "image", "url": url, "width": w, "height": h}:
            return f"图片 {url} ({w}x{h})"
        case {"type": "error", "code": code, **rest}:
            return f"错误码 {code}，额外信息: {rest}"
        case {"type": str(t), **rest}:        # type 是字符串，但不关心具体值
            return f"未知类型 '{t}'，内容: {rest}"
        case _:
            return "无效数据"

print(handle_api({"type": "text", "content": "你好"}))
print(handle_api({"type": "image", "url": "a.png", "width": 800, "height": 600}))
print(handle_api({"type": "error", "code": 500, "msg": "server down"}))
```

> 映射模式只检查**指定的 key**——字典中多余的 key 会被忽略（除非使用了 `**rest`）。所以 `case {"type": "text"}` 能匹配 `{"type": "text", "extra": ...}`。

## 类模式——匹配对象属性

类模式（Class Pattern）让你可以"反向构造"一个对象——只关心特定属性的值：

```python-run
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

@dataclass
class Circle:
    center: Point
    radius: float

@dataclass
class Rect:
    top_left: Point
    bottom_right: Point

def describe_shape(shape):
    match shape:
        case Point(x=0, y=0):
            return "原点"
        case Point(x=0, y=y):              # 在 y 轴上
            return f"在 y 轴上，y={y}"
        case Point(x=x, y=0):
            return f"在 x 轴上，x={x}"
        case Circle(center=Point(x=0, y=0), radius=r):  # 嵌套匹配
            return f"圆心在原点的圆，半径 {r}"
        case Rect(top_left=Point(x=x1, y=y1), bottom_right=Point(x=x2, y=y2)):
            return f"矩形: ({x1},{y1}) → ({x2},{y2})"
        case _:
            return "其他图形"

print(describe_shape(Point(0, 5)))
print(describe_shape(Circle(Point(0, 0), 10)))
print(describe_shape(Rect(Point(0, 0), Point(100, 50))))
```

类模式的工作原理：`Point(x=0, y=y)` 先检查 `isinstance(shape, Point)`，然后按**位置参数名**（构造时的参数名）匹配对应的属性值。`x=0` 是字面量约束，`y=y` 是把属性 `y` 的值绑定到变量 `y`。

## 守卫——用条件约束匹配

**守卫（Guard）** 是 `case` 后面附加的 `if` 条件，在模式匹配成功后才执行，进一步筛选：

```python-run
def classify_number(n):
    match n:
        case 0:
            return "零"
        case int(x) if x % 2 == 0:
            return f"偶数: {x}"
        case int(x) if x > 0:
            return f"正奇数: {x}"
        case int(x):
            return f"负奇数: {x}"
        case _:
            return "不是整数"

print(classify_number(10))
print(classify_number(7))
print(classify_number(-3))
```

守卫弥补了模式匹配无法表达"范围"的局限——你不能写 `case range(0, 10)`，但可以写 `case int(x) if 0 <= x < 10`。

## OR 模式——一组值共享一个分支

用 `|` 把多个模式合在一起，任一匹配即进入该分支：

```python-run
def classify_status(code):
    match code:
        case 200 | 201 | 204:
            return "成功"
        case 301 | 302 | 307:
            return "重定向"
        case 401 | 403:
            return "权限问题"
        case 404 | 410:
            return "资源不存在"
        case 500 | 502 | 503:
            return "服务器错误"
        case _:
            return f"未知状态码: {code}"

print(classify_status(301))
print(classify_status(502))
```

OR 模式的每个分支也可以绑定变量——但要求**每个子模式绑定的变量名和类型完全一致**。

## 实战：遍历抽象语法树

以下示例演示用模式匹配优雅地处理嵌套结构——这在实际项目中非常实用的"AST 遍历"场景：

```python-run
class Expr:  # 模拟一个简化的表达式 AST
    pass

@dataclass
class Number(Expr):
    value: int

@dataclass
class Add(Expr):
    left: Expr
    right: Expr

@dataclass
class Mul(Expr):
    left: Expr
    right: Expr

@dataclass
class Negate(Expr):
    operand: Expr

def evaluate(expr: Expr) -> int:
    match expr:
        case Number(value=n):
            return n
        case Add(left=l, right=r):
            return evaluate(l) + evaluate(r)
        case Mul(left=l, right=r):
            return evaluate(l) * evaluate(r)
        case Negate(operand=inner):
            return -evaluate(inner)
        case _:
            raise TypeError(f"未知表达式: {expr!r}")

# (1 + 2) * (-3)
ast = Mul(
    Add(Number(1), Number(2)),
    Negate(Number(3))
)
print(evaluate(ast))  # -9
```

## match case vs if-elif-isinstance 链

同一个任务，两种写法：

| | `match case` | `if-elif-isinstance` |
|--|-------------|---------------------|
| 可读性 | **高**：一眼看出"按形状匹配" | 低：层层嵌套的 `isinstance` 和属性取值 |
| 安全性 | **高**：变量只在匹配成功时绑定，没有命名冲突 | 低：容易在分支间泄漏变量，需手动管理 |
| 性能 | 快速路径优化（CPython 3.10+ 对字面量有跳转表） | 线性逐个判断，无优化空间 |
| 结构解构 | **内置**：`[first, *rest]` / `Point(x=0, y=y)` | 手写解包，重复且易出错 |
| 类型检查 | `case int(x)` 一步完成 | 需要 `isinstance()` + 单独类型转换/断言 |
| Python 版本 | 3.10+ | 所有版本 |

```python-run
# 对比：解析 JSON-like 数据
data = {"type": "user", "name": "Dean", "age": 25}

# if-elif 版本 —— 9 行，嵌套取值的噪音掩盖了业务逻辑
if isinstance(data, dict):
    if data.get("type") == "user":
        name = data.get("name", "unknown")
        age = data.get("age")
        result = f"用户 {name}，{age} 岁"
    elif data.get("type") == "error":
        result = f"错误: {data.get('msg')}"
    else:
        result = "未知"
else:
    result = "不是字典"

# match 版本 —— 5 行，结构就是文档
match data:
    case {"type": "user", "name": name, "age": age}:
        result = f"用户 {name}，{age} 岁"
    case {"type": "error", "msg": msg}:
        result = f"错误: {msg}"
    case _:
        result = "不是用户消息"

print(result)
```

## 局限性

模式匹配强大但**不能替代一切**：

| 不能做的事 | 替代方案 |
|-----------|---------|
| 匹配正则表达式 | `case str(s) if re.match(pattern, s):`（守卫 + regex） |
| 匹配数值范围 | `case int(x) if 0 <= x < 100:`（守卫） |
| 匹配自定义条件（如 `x > y + z`） | 守卫可以写任意表达式 |
| 匹配函数返回值（调用表达式） | 不支持 `case f(x):`——需要先计算再匹配 |
| 在 3.10 之前使用 | 回退到 `if-elif-isinstance` |

> 核心原则：**match case 匹配的是"结构"，不是"条件"**。一切需要动态求值的判断，都通过守卫来叠加。

## 本章小结

- `match case` 不是 C 的 `switch`，而是声明式的"结构拆解 + 类型检查 + 值绑定"
- 字面量模式（`case 42`）做值比较，变量模式（`case x`）做绑定——严格区分，不会把已有变量名当值匹配
- 序列模式（`[first, *rest]`）、映射模式（`{"type": t}`）、类模式（`Point(x=0, y=y)`）让解构数据像写数据一样自然
- 守卫（`if` 条件）弥补模式无法表达"数值范围"的局限；OR 模式（`|`）让多个模式共享同一分支
- 模式匹配适合处理有明确结构的数据（AST、JSON API 响应），不适合匹配正则表达式、数值范围或动态调用表达式
