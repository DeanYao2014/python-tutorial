# 1.4 字符串与字节

字符串和字节是 Python 处理"文本"和"二进制数据"的两套体系。理解它们之间的边界——**何时解码、何时编码、何时绕开**——能避免生产环境 90% 的乱码事故。

## Unicode 基础：码点（Code Point）vs 编码（Encoding）

**码点**是 Unicode 标准给每个字符分配的编号（`U+0000` ~ `U+10FFFF`）。**编码**是把码点变成可传输/存储的字节序列的规则。同一个码点（如 "中"）可以用 UTF-8、UTF-16、UTF-32 编码——包装不同，收件人相同。

```python-run
# ord() — 字符 → 码点
print(ord('中'))        # 20013 —— 十六进制 U+4E2D
print(hex(ord('中')))   # 0x4e2d

# chr() — 码点 → 字符
print(chr(20013))       # 中
print(chr(0x4e2d))      # 中
```

> 码点是"身份证号"，编码是"快递包装"——同一个人的快递可以用纸箱、木箱、信封，包装不同但收件人相同。

## UTF-8 编码原理：变长编码

UTF-8 用 **1 到 4 个字节**表示一个码点，核心设计是 **ASCII 兼容**：所有纯 ASCII 文本天然就是合法的 UTF-8。

| 码点范围 | UTF-8 字节数 | 典型内容 |
|-----------|-------------|---------|
| `U+0000 ~ U+007F` | 1 字节 | ASCII（英文、数字） |
| `U+0800 ~ U+FFFF` | 3 字节 | 中日韩文字（CJK） |
| `U+10000 ~ U+10FFFF` | 4 字节 | emoji、古代文字 |

```python-run
char = "中"
utf8  = char.encode("utf-8")   # 3 字节
utf16 = char.encode("utf-16")  # 2 字节 + BOM
utf32 = char.encode("utf-32")  # 4 字节（固定长度）

print(f"UTF-8:  {utf8.hex()} → {len(utf8)} 字节")
print(f"UTF-16: {utf16.hex()} → {len(utf16)} 字节")
print(f"UTF-32: {utf32.hex()} → {len(utf32)} 字节")
```

用 Unicode 转义：`\u` 跟 4 位十六进制，`\U` 跟 8 位十六进制（超出 BMP 的码点必须用 `\U`）。

```python-run
print("中")       # 中
print("\U00004e2d")   # 中 —— 不足 8 位前面补 0
print("\U0001f600")   # 😀
```

## str 的内部表示：PEP 393 灵活字符串

你可能以为 Python 字符串内部是 UTF-8——其实不是。从 Python 3.3 起（PEP 393），`str` 内部根据**字符串中最大码点**自动选择最紧凑的存储：

| 最大码点 | 每字符占用 | 使用场景 |
|----------|-----------|---------|
| 全部 `< 128` | 1 字节 | 纯英文文本 |
| `< 65536` | 2 字节 | 含中文但无 emoji |
| `≥ 65536` | 4 字节 | 含 emoji |

```python-run
import sys

ascii_s = "hello"
cjk_s   = "你好"
emoji_s = "hello😀"

print(f"纯 ASCII: {sys.getsizeof(ascii_s)} 字节")
print(f"含中文:   {sys.getsizeof(cjk_s)} 字节")
print(f"含 emoji: {sys.getsizeof(emoji_s)} 字节")
```

> 对开发者完全透明——`str` 对外就是"Unicode 码点序列"，你不需要关心内部编码。

## f-string 进阶

### 格式化说明符

```python-run
n = 1234.5678

print(f"{n:.2f}")        # 1234.57 —— 小数位数
print(f"{n:,.2f}")       # 1,234.57 —— 千位分隔符
print(f"{255:#x}")       # 0xff —— 带前缀十六进制
print(f"{0.25:.1%}")     # 25.0% —— 百分比
print(f"|{'hello':*^10}|")  # |***hello***| —— 居中对齐，* 填充

# 日期格式
from datetime import datetime
dt = datetime(2026, 5, 23, 14, 30)
print(f"{dt:%Y-%m-%d %H:%M}")  # 2026-05-23 14:30
```

### `=` 调试语法（Python 3.8+）

```python-run
x = 42
print(f"{x = }")        # x = 42 —— 同时显示表达式文本和值

from math import pi
print(f"{pi = :.3f}")   # pi = 3.142 —— 配合格式化说明符
```

这让你从 `print("x=", x)` 缩减为一个简洁的 `f"{x=}"`。

### 嵌套表达式

```python-run
width = 10
name = "Python"
print(f"{name:{width}}")    # Python    —— 动态指定字段宽度
print(f"{name:.{5}}")       # Pytho     —— 动态截断
```

## Python 3.12 f-string 增强（PEP 701）

```python-run
# 3.12+：f-string 内可使用与外层不同的引号
d = {"name": "Python", "version": 3.12}
print(f"{d["name"]}")   # 3.11 会 SyntaxError

# 3.12+：{ } 内可以使用多行表达式
print(f"最大值: {max(
    [1, 2, 3, 4, 5]
)}")

# 3.12+：{ } 内支持反斜杠
print(f"换行: {"hello\nworld"}")
```

之前需要把复杂表达式提取为中间变量再传入 f-string，现在可以直接写在 `{}` 里。

## bytes：不可变字节序列

`bytes` 是**不可变的 0-255 整数序列**，是二进制数据的载体。索引返回的是 `int`，不是单字符 `bytes`：

```python-run
b1 = b"hello"                  # 字面量——只允许 ASCII 字符
b2 = "你好".encode("utf-8")    # str → bytes

print(b1)                      # b'hello'
print(b1[0])                   # 104 —— 索引返回整数！
print(b2)                      # b'\xe4\xbd\xa0\xe5\xa5\xbd'
print(b2.hex(" "))             # e4 bd a0 e5 a5 bd —— hex 可读形式

# bytes → str
print(b2.decode("utf-8"))      # 你好
```

> **规则**：`str` 只负责 `encode()`（文本 → 字节），`bytes` 只负责 `decode()`（字节 → 文本）。没有"暗中转换"，也没有"有时自动有时不行"——方向搞错直接报 `AttributeError`。

```python-run
try:
    b"hello".encode("utf-8")
except AttributeError as e:
    print(f"bytes 没有 .encode(): {e}")
```

---

## bytearray：可变的 bytes

当你需要原地修改字节序列时，用 `bytearray`：

```python-run
ba = bytearray(b"hello")
ba[0] = ord("H")       # 原地修改——传 0-255 的整数
ba.append(33)          # 追加（33 是 '!'）
print(ba)              # bytearray(b'Hello!')
```

`bytearray` 之于 `bytes`，就像 `list` 之于 `tuple`。

## memoryview：零拷贝操作二进制数据

`memoryview` 让你在**不复制数据**的前提下，以不同的方式"观察"同一块内存——切片和 cast 都是 O(1) 的视图操作：

```python-run
# 原始数据：16 字节
data = bytearray(b"\x00\x00\x00\x41\x00\x00\x00\x42"
                 b"\x00\x00\x00\x43\x00\x00\x00\x44")

mv = memoryview(data)
ints = mv.cast('i')    # 将内存重新解释为 4 字节整数序列
print(list(ints))       # [65, 66, 67, 68] —— 即 A, B, C, D

# 零拷贝切片：不复制底层数据
half = ints[:2]
print(half.obj is data) # True —— 指向同一个 bytearray
```

对比 `struct.unpack`：每次调用都会复制数据。`memoryview` 处理网络包、多媒体文件等大量二进制数据时性能差异巨大。

## 常见陷阱

**UnicodeDecodeError**——用错编码解码：

```python-run
try:
    "你好".encode("gbk").decode("utf-8")
except UnicodeDecodeError as e:
    print(repr(e))
```

编码时用的是什么，解码时必须一致。如果非要从乱码中"抢救"，传 `errors="replace"` 或 `errors="ignore"`。

**`\u` vs `\U`**——`\u` 跟 4 位，`\U` 跟 8 位，不是大小写关系：

```
print("中")      # ✅ 4 位
print("\U0001f600")  # ✅ 8 位，不足前补零
# print("\U4e2d")    # ❌ 只有 4 位，SyntaxError
```

**BOM（Byte Order Mark）**——Windows 记事本保存 UTF-8 文件时会在文件头加 `﻿`（`U+FEFF`）。`encoding="utf-8-sig"` 可以自动跳过 BOM。

## 总结

| 概念 | 一句话 |
|------|--------|
| **str** | 人类可读的 Unicode 文本，内部编码对开发者透明 |
| **bytes** | 不可变字节序列，索引返回 `int`，网络/文件的"中介语言" |
| **bytearray** | 可变的 bytes，需要原地修改时用它 |
| **memoryview** | 不复制数据的"视图"，处理大量二进制数据时的性能利器 |
| **f-string** | 不止插值——格式化、调试、动态宽度，现代 Python 首选 |
| **encode/decode** | str ↔ bytes 的唯一通道，方向必须记牢 |

## 本章小结

- `str` 是 Unicode 码点序列，内部编码（PEP 393）对开发者透明；`bytes` 是不可变的 0-255 整数序列，索引返回 `int`
- `encode()` 将 str 转为 bytes，`decode()` 将 bytes 转为 str——方向固定，没有暗中转换
- f-string 不止插值：格式化说明符、`=` 调试语法（3.8+）、多行表达式和引号混用（3.12+）
- `bytearray` 是可变的 bytes；`memoryview` 提供零拷贝的二进制数据视图，适合处理网络包和多媒体数据
- 编码和解码必须一致；`errors="replace"` 或 `errors="ignore"` 能"抢救"乱码，但最好从源头解决
