# 元编程（Metaprogramming）—— 编写操作代码的代码

元编程的核心思想：**程序可以读写、分析、修改、生成程序自身**。在 Python 中，这不仅是可能的，而且被大量用于框架和工具中（pytest、Django ORM、FastAPI、dataclasses）。

> **一句话定义**：元编程 = 编写操作代码的代码（Code that operates on code）。

---

## `exec` / `eval` / `compile`：动态执行代码

### `exec`：执行动态代码块

`exec(code, globals, locals)` 将字符串作为 Python 代码执行。返回值永远是 `None`：

```python-run
namespace = {}
code = """
total = sum(range(1, 101))
greeting = "Hello, " + "World"
"""
exec(code, namespace)
print(namespace["total"])     # 5050
print(namespace["greeting"])  # Hello, World
```

> ⚠️ **命名空间隔离**：不给第二个参数的话，`exec` 在调用者的当前作用域执行。**永远显式传入命名空间字典**以避免副作用。

### `eval`：求值单个表达式

`eval(expr, globals, locals)` 只能处理**单一表达式**，返回表达式的值：

```python-run
print(eval("3 ** 10"))                      # 59049
print(eval("[i**2 for i in range(5)]"))     # [0, 1, 4, 9, 16]
# eval("x = 1")  # SyntaxError — 不能执行语句
```

`exec` vs `eval` 对比：

| | `exec` | `eval` |
|---|--------|--------|
| 输入 | 任意代码块（多行、语句） | 单个表达式 |
| 返回值 | `None` | 表达式的值 |
| 用途 | 执行一段代码 | 计算一个值 |

### `compile`：编译为代码对象

`compile(source, filename, mode)` 将源码编译为**代码对象（Code Object）**——这是 `exec` 和 `eval` 的内部步骤。分离编译和执行可以获得更好的性能（一次编译、多次执行）：

```python-run
# mode='exec': 多行代码块
code = compile("a = 1; b = 2; c = a + b", "<string>", "exec")

# mode='eval': 单个表达式
expr = compile("x ** y", "<math>", "eval")

# mode='single': 交互式单条语句 (如 Python REPL 用的)
```

三种模式的含义：
- `'exec'`：任意多行代码，可以有语句、函数定义等
- `'eval'`：单个表达式（`eval()` 内部用的就是这个）
- `'single'`：单条交互式语句（自动打印表达式结果，类似 REPL）

### 安全：永远不要 eval/exec 不可信的输入

```python-run
# ❌ 绝对不要这样做:
# user_input = request.GET['code']
# exec(user_input)  # 用户可以执行任何代码!

# ✅ 安全做法: 严格限制命名空间
safe_builtins = {"abs": abs, "min": min, "max": max}
safe_globals = {"__builtins__": safe_builtins}
# 结果只能使用白名单中的函数
try:
    eval("open('/etc/passwd')", safe_globals)
except NameError:
    print("安全拦截: open 不在白名单中")

# 更安全的替代方案: ast.literal_eval
import ast
print(ast.literal_eval("[1, 2, {'key': 'value'}]"))  # 只解析字面量
```

**安全法则**：
1. **永远不要** `eval`/`exec` 用户输入
2. 即使是配置文件/DSL，用 `ast.literal_eval`、`json.loads` 或自定义解析器
3. 如果必须用 `exec`，严格限制 `__builtins__`，传入空字典

> 📖 官方文档：[exec](https://docs.python.org/3/library/functions.html#exec), [eval](https://docs.python.org/3/library/functions.html#eval), [compile](https://docs.python.org/3/library/functions.html#compile), [ast.literal_eval](https://docs.python.org/3/library/ast.html#ast.literal_eval)

## `type()` 动态创建类

### 三参数用法：`type(name, bases, namespace)`

```python-run
# type 的三参数形式: 动态创建类
Dog = type("Dog", (object,), {
    "species": "Canis familiaris",
    "__init__": lambda self, name: setattr(self, "name", name),
    "bark": lambda self: f"{self.name} says woof!",
})

dog = Dog("Buddy")
print(dog.bark())  # Buddy says woof!
print(type(dog))   # <class '__main__.Dog'>
```

这等价于：
```python
class Dog:
    species = "Canis familiaris"
    def __init__(self, name): setattr(self, "name", name)
    def bark(self): return f"{self.name} says woof!"
```

### 实战：动态 ORM 模型

`type()` 最常见的实际用途之一是**根据运行时元数据生成数据模型**：

```python-run
def create_model(table_name, **fields):
    """从字段定义动态创建模型类"""
    def __init__(self, **kwargs):
        for k, v in fields.items():
            setattr(self, k, kwargs.get(k, v))
    def __repr__(self):
        parts = ", ".join(f"{k}={getattr(self, k)!r}" for k in fields)
        return f"{table_name}({parts})"
    return type(table_name, (object,), {
        "__init__": __init__, "__repr__": __repr__,
        "_table": table_name, "_fields": fields,
    })

User = create_model("User", name="", age=0, email="")
u = User(name="Alice", age=30)
print(u)  # User(name='Alice', age=30, email='')
```

> 这就是 Django ORM、SQLAlchemy 等框架底层实现的核心机制——`ModelBase` 元类调用 `type.__new__` 来根据字段定义构造类。

### `__prepare__`：自定义类命名空间

`__prepare__` 是元类的一个特殊方法，在类体执行**之前**被调用，返回一个映射对象作为类体的临时命名空间：

```python-run
from collections import OrderedDict

class OrderedMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases):
        return OrderedDict()  # ← 类体中的定义按插入顺序保存

    def __new__(mcs, name, bases, namespace):
        return super().__new__(mcs, name, bases, namespace)

class MyClass(metaclass=OrderedMeta):
    z = 3
    a = 1
    m = 2

# 查看类属性的顺序——与定义顺序一致
print([name for name in MyClass.__dict__ if not name.startswith('_')])
# ['z', 'a', 'm']  而非按字母排序的 ['a', 'm', 'z']
```

> 💡 Python 3.6+ 起，普通类的 `__dict__` 默认就保持插入顺序（得益于 dict 的 ordering guarantee），所以 `__prepare__` 的使用场景已经大幅减少。仅在需要自定义映射类型（如禁止重复属性名、自动转换属性名等）时才需要。

> 📖 官方文档：[type](https://docs.python.org/3/library/functions.html#type), [__prepare__](https://docs.python.org/3/reference/datamodel.html#determining-the-appropriate-metaclass)

## `inspect` 模块：内省（Introspection）

`inspect` 是 Python 的"代码之眼"——让你在运行时查看活对象的内部结构。

### `inspect.signature()`：读取函数签名

```python-run
import inspect

def connect(host: str, port: int = 5432, *, timeout: float = 30.0):
    """建立数据库连接"""
    pass

sig = inspect.signature(connect)
for name, param in sig.parameters.items():
    print(f"{name}: kind={param.kind.name}, "
          f"annotation={param.annotation}, "
          f"default={param.default!r}")
```

输出：
```
host: kind=POSITIONAL_OR_KEYWORD, annotation=<class 'str'>, default=<class 'inspect.Parameter.empty'>
port: kind=POSITIONAL_OR_KEYWORD, annotation=<class 'float'>, default=5432
timeout: kind=KEYWORD_ONLY, annotation=<class 'float'>, default=30.0
```

### `inspect.getsource()`：获取源码

```python-run
# （注意: 仅对磁盘上的 .py 文件中的函数有效）
import inspect
print(inspect.getsource(connect)[:80])  # def connect(host: str, port: int ...
```

### `inspect.getmembers()`：遍历对象的所有成员

```python-run
class Demo:
    x = 1
    def method(self): pass

# 只获取方法
methods = inspect.getmembers(Demo, predicate=inspect.isfunction)
print([name for name, _ in methods])  # ['method']
```

### `inspect.currentframe()` / `inspect.stack()`：访问调用栈

```python-run
def outer():
    return middle()

def middle():
    return inner()

def inner():
    stack = inspect.stack()
    for frame_info in stack[:3]:
        print(f"  {frame_info.function} @ {frame_info.filename}:{frame_info.lineno}")
    return 42

outer()
```

这是调试、日志框架、性能分析器的基础设施。

### 实战：自动生成 API 文档

```python-run
import inspect

def auto_doc(func):
    """从类型注解和 docstring 自动生成 API 文档"""
    sig = inspect.signature(func)
    lines = [f"### {func.__name__}{sig}", ""]
    if func.__doc__:
        lines.append(f"{func.__doc__.strip()}\n")
    lines.append("| 参数 | 类型 | 默认值 |")
    lines.append("|------|------|--------|")
    for name, param in sig.parameters.items():
        annot = param.annotation if param.annotation is not param.empty else "-"
        default = param.default if param.default is not param.empty else "-"
        lines.append(f"| {name} | {annot} | {default} |")
    return "\n".join(lines)

print(auto_doc(connect))
```

> 📖 官方文档：[inspect](https://docs.python.org/3/library/inspect.html)

## AST（Abstract Syntax Tree）：语法树级操作

### `ast.parse()` 和 `ast.dump()`：看到源码的内部结构

```python-run
import ast

source = "x + y * 2"
tree = ast.parse(source)
print(ast.dump(tree, indent=2))
```

每一层都是一棵嵌套的树：
```
Module
  Expr
    BinOp (left=x, op=Add(), right=...)
      Name(id='x')
      BinOp (left=y, op=Mult(), right=...)
        Name(id='y')
        Constant(value=2)
```

### `ast.NodeVisitor`：遍历（只读分析）

```python-run
class FunctionCollector(ast.NodeVisitor):
    """收集所有函数定义"""
    def __init__(self):
        self.functions = []
    def visit_FunctionDef(self, node):
        self.functions.append(node.name)
        self.generic_visit(node)  # 继续遍历子节点

source = """
def foo(): pass
def bar(): pass
"""
tree = ast.parse(source)
fc = FunctionCollector()
fc.visit(tree)
print(fc.functions)  # ['foo', 'bar']
```

### `ast.NodeTransformer`：转换（修改 AST）

```python-run
class InjectPrints(ast.NodeTransformer):
    """在每个函数体开头插入 print("进入 <name>")"""
    def visit_FunctionDef(self, node):
        call = ast.Expr(
            value=ast.Call(
                func=ast.Name(id="print", ctx=ast.Load()),
                args=[ast.Constant(value=f"进入 {node.name}")],
                keywords=[],
            )
        )
        node.body.insert(0, call)
        return self.generic_visit(node)

source = """
def add(a, b):
    return a + b
"""
tree = ast.parse(source)
tree = InjectPrints().visit(tree)
ast.fix_missing_locations(tree)

# 编译修改后的 AST 并执行
code = compile(tree, "<injected>", "exec")
exec(code)
add(1, 2)  # 打印 "进入 add" 然后返回 3
```

### `ast.literal_eval`：安全的字面量求值

```python-run
import ast
# ast.literal_eval 只能解析 Python 字面量
print(ast.literal_eval("{'a': 1, 'b': [2, 3]}"))  # 安全
# ast.literal_eval("__import__('os').system('ls')")  # ValueError: 不允许!
```

### 实战：简单代码规则检测器

```python-run
class Linter(ast.NodeVisitor):
    """检测不推荐的代码模式"""
    def __init__(self):
        self.warnings = []

    def visit_Call(self, node):
        if isinstance(node.func, ast.Name) and node.func.id in ("eval", "exec"):
            self.warnings.append(f"行{node.lineno}: 使用了 {node.func.id}()")
        self.generic_visit(node)

    def visit_ExceptHandler(self, node):
        if node.type is None:
            self.warnings.append(f"行{node.lineno}: 裸 except, 应指定异常类型")
        self.generic_visit(node)

source = "try:\n    eval(data)\nexcept:\n    pass"
Linter().visit(ast.parse(source))
```

这就是 pylint、flake8、pytest（用于 assertion rewriting）等工具的核心技术。

> 📖 官方文档：[ast](https://docs.python.org/3/library/ast.html), [Green Tree Snakes - AST Guide](https://greentreesnakes.readthedocs.io/)

## 实际应用场景

### 场景 1：简易模板引擎

将 `{{ variable }}` 模板编译为 Python 函数，执行快于字符串替换：

```python-run
import re

def compile_template(template_str: str):
    """将 {{ var }} 模板编译为 Python 代码"""
    parts = []
    last = 0
    for m in re.finditer(r"\{\{(.+?)\}\}", template_str):
        parts.append(repr(template_str[last:m.start()]))
        parts.append(f"str(_c[{m.group(1).strip()!r}])")
        last = m.end()
    parts.append(repr(template_str[last:]))

    code = f"def render(_c): return ''.join([{', '.join(parts)}])"
    namespace = {}
    exec(code, namespace)
    return namespace["render"]

render = compile_template("Hello, {{ name }}! 余额: {{ balance }}")
print(render({"name": "Dean", "balance": 99.9}))
```

生成的 Python 代码：
```python
def render(_c): return ''.join(['Hello, ', str(_c['name']), '! 余额: ', str(_c['balance'])])
```

### 场景 2：代码生成

从 schema 自动生成样板代码（boilerplate）：

```python-run
def generate_model(class_name: str, fields: dict) -> str:
    """从字段 schema 生成 Python 类源码"""
    lines = [f"class {class_name}:", '    """Auto-generated model."""', ""]
    params = ", ".join(f"{k}={v!r}" for k, v in fields.items())
    lines.append(f"    def __init__(self, {params}):")
    for k in fields:
        lines.append(f"        self.{k} = {k}")
    return "\n".join(lines)

print(generate_model("Book", {"title": "", "price": 0.0}))
```

这是代码生成器（如 protobuf compiler、GraphQL codegen、OpenAPI generator 的 Python 后端）的基础技术。

### 场景 3：简易依赖注入框架

利用 `inspect.signature` 实现自动依赖注入：

```python-run
import inspect, functools

def inject(**dependencies):
    """根据函数参数名自动注入依赖"""
    def decorator(func):
        sig = inspect.signature(func)
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for name in sig.parameters:
                if name in dependencies:
                    kwargs.setdefault(name, dependencies[name]())
            return func(*args, **kwargs)
        return wrapper
    return decorator

# 工厂函数
def get_db(): return "<Database>"
def get_cache(): return "<Cache>"

@inject(db=get_db, cache=get_cache)
def handler(user_id, db, cache):
    return f"user={user_id}, db={db}, cache={cache}"

print(handler(42))
# user=42, db=<Database>, cache=<Cache>
```

> FastAPI 的 `Depends()` 就是这种模式的工业级实现。

### 场景 4：pytest 的断言改写

pytest 可能是 Python 领域最知名的 AST 应用。当你这样写：

```python
assert a == b
```

pytest 在导入测试文件时，用 `ast.NodeTransformer` 将 `assert` 语句改写为包含丰富上下文信息的版本。这样测试失败时，pytest 能精确告诉你 `a` 和 `b` 各自的**实际值**，而不只是 "AssertionError"。

核心原理与上述 `NodeTransformer` 示例完全相同，只是更复杂。

## 本章小结

- **动态执行**：`exec` 执行任意代码块，`eval` 求值单个表达式，`compile` 分离编译和执行；永远不要对不可信输入使用，用 `ast.literal_eval` 安全解析字面量。
- **动态创建类**：`type(name, bases, namespace)` 三参数形式根据运行时元数据动态构造类，`__prepare__` 自定义类体命名空间，这是 ORM 等框架底层实现的核心机制。
- **运行时内省**：`inspect` 模块是 Python 的"代码之眼"——`signature()` 读取函数签名、`getsource()` 获取源码、`stack()` 访问调用栈，是调试、日志、依赖注入的基础设施。
- **AST 操作**：`ast` 模块在语法树级别操作代码——`NodeVisitor` 遍历分析（如代码质量检测），`NodeTransformer` 转换修改（如 pytest 断言改写），`ast.literal_eval` 安全求值字面量。
- **应用场景**：元编程典型应用包括模板引擎（编译模板为 Python 函数）、代码生成（从 schema 生成样板代码）、依赖注入（利用 `inspect.signature` 自动注入）、代码质量检测（AST 规则检测器）。
