# 3.4 描述符协议

如果说元类（Metaclass）控制"类的创建"，那描述符（Descriptor）控制的是"属性的访问"——每一次你写下 `obj.attr`，Python 背后都可能触发描述符协议。它是 `@property`、`@staticmethod`、`@classmethod`、`__slots__` 的底层统一机制。

本章回答三个问题：**描述符协议到底是什么**、**数据描述符和非数据描述符的优先级差异如何影响你的代码**、**如何用描述符写出可复用的属性逻辑**。

## 4.1 描述符协议本质

### 它是什么

描述符是实现了以下任意一个或多个方法的对象：

| 方法 | 签名 | 触发时机 |
|------|------|---------|
| `__get__` | `(self, instance, owner=None)` | 属性被访问 |
| `__set__` | `(self, instance, value)` | 属性被赋值 |
| `__delete__` | `(self, instance)` | `del obj.attr` |
| `__set_name__` | `(self, owner, name)` | 类创建完成时（Python 3.6+） |

**只要 Python 在类的 `__dict__` 中找到一个实现了 `__get__` 的对象，它就认为这是一个描述符，属性的访问行为由描述符接管。**

> "Descriptors are a powerful, general purpose protocol. They are the mechanism behind properties, methods, static methods, class methods, and `super()`." —— Python 官方文档

> 📖 官方文档：[Descriptor HowTo Guide](https://docs.python.org/3/howto/descriptor.html), [Implementing Descriptors](https://docs.python.org/3/reference/datamodel.html#implementing-descriptors)

### 最小描述符

```python-run
class RevealAccess:
    """打印每一次属性访问——最简单的描述符"""

    def __get__(self, instance, owner=None):
        print(f"__get__: instance={instance}, owner={owner}")
        return 42

    def __set__(self, instance, value):
        print(f"__set__: instance={instance}, value={value}")

    def __delete__(self, instance):
        print(f"__delete__: instance={instance}")


class MyClass:
    x = RevealAccess()


obj = MyClass()
print(obj.x)       # __get__ → 42
obj.x = 100        # __set__
del obj.x          # __delete__
print(MyClass.x)   # __get__: instance=None → 类级别访问也能触发
```

三个关键细节：

1. **`instance` 可能是 `None`**：当通过类访问而非实例访问（`MyClass.x`）时，`instance` 为 `None`。描述符必须处理这种情形。
2. **描述符存在类 `__dict__` 中**：描述符必须在**类**上定义，实例上的同名属性不会触发描述符协议。
3. **`owner`**：始终是描述符所属的类（或子类）。

### `__set_name__`：描述符的自我认知

Python 3.6（PEP 487）引入 `__set_name__`，解决了描述符"不知道自己叫什么名字"的经典问题：

```python-run
class Typed:
    def __init__(self, expected_type: type):
        self.expected_type = expected_type
        self.private_name: str = ""   # __set_name__ 填入

    def __set_name__(self, owner, name: str):
        """类创建完成时自动调用，name='x' 中就是我们声明的属性名"""
        self.private_name = f"_{name}"

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        return getattr(instance, self.private_name)

    def __set__(self, instance, value):
        if not isinstance(value, self.expected_type):
            raise TypeError(
                f"期望 {self.expected_type.__name__}，实际 {type(value).__name__}"
            )
        setattr(instance, self.private_name, value)


class Point:
    x = Typed(int)      # __set_name__ 被调用，private_name = "_x"
    y = Typed(float)    # __set_name__ 被调用，private_name = "_y"

    def __init__(self, x: int, y: float):
        self.x = x
        self.y = y
```

没有 `__set_name__` 之前，你得写成 `x = Typed(int, "x")`——属性名写两遍，迟早不一致。

> 📖 官方文档：[object.__set_name__](https://docs.python.org/3/reference/datamodel.html#object.__set_name__), [PEP 487](https://peps.python.org/pep-0487/)

## 4.2 数据描述符 vs 非数据描述符

这是描述符理论中**最重要的一个区别**。不理解它，你就无法解释为什么 `@property` 的行为和 `@staticmethod` 不同。

### 分类

| 类型 | 实现了 | 实例 `__dict__` 优先级 |
|------|--------|----------------------|
| **数据描述符** | `__get__` + `__set__`（或 `__delete__`） | 描述符**高于**实例字典 |
| **非数据描述符** | 只有 `__get__` | 实例字典**高于**描述符 |

### 优先级链实验

```python-run
class DataDesc:
    """数据描述符：__get__ + __set__"""
    def __get__(self, instance, owner=None):
        print("  DataDesc.__get__ 被调用")
        return "来自数据描述符"

    def __set__(self, instance, value):
        print(f"  DataDesc.__set__ 被调用({value})")


class NonDataDesc:
    """非数据描述符：只有 __get__"""
    def __get__(self, instance, owner=None):
        print("  NonDataDesc.__get__ 被调用")
        return "来自非数据描述符"


class Test:
    data_des = DataDesc()
    nondata_des = NonDataDesc()


t = Test()

# ── 数据描述符：写入被拦截，无法写入实例 __dict__ ──
print("── 数据描述符 ──")
print(t.data_des)         # DataDesc.__get__ 被调用
t.data_des = "尝试覆盖"    # DataDesc.__set__ 被调用！
print(t.__dict__)          # {} —— 没有 data_des

# ── 非数据描述符：先查实例 __dict__ ──
print("── 非数据描述符 ──")
print(t.nondata_des)       # NonDataDesc.__get__ 被调用
t.__dict__['nondata_des'] = "实例属性"
print(t.nondata_des)       # "实例属性" —— 覆盖了描述符！
```

**完整优先级链**（由高到低）：

```
1. 数据描述符 (__get__ + __set__/__delete__)  ← @property 在此
2. instance.__dict__                           ← 普通实例属性
3. 非数据描述符 (仅有 __get__)                  ← @staticmethod 在此
4. class.__dict__（普通类属性）                  ← 默认类变量
5. __getattr__（兜底）                          ← 仅在以上全部失败后
```

### 这解释了为什么

| 现象 | 原因 |
|------|------|
| `@property` 可以阻止实例属性覆盖 | 它是数据描述符，优先级高于 `instance.__dict__` |
| `instance.method = 1` 可以覆盖方法 | 普通函数是非数据描述符，实例属性优先级更高 |
| `__slots__` 不能随意新增属性 | `__slots__` 成员是数据描述符，写入时会校验槽位是否存在 |
| `@staticmethod` 可以被实例属性遮蔽 | `staticmethod` 是非数据描述符，实例 `__dict__` 优先 |

```python-run
# 验证：@staticmethod 可以被实例属性遮蔽
class Demo:
    @staticmethod
    def greet():
        return "hello"

d = Demo()
print(d.greet())           # "hello"
d.greet = lambda: "goodbye"  # 把 greet 当作实例属性覆盖
print(d.greet())           # "goodbye" —— @staticmethod 被遮蔽了！

# 对比：@property 不能被遮蔽
class Demo2:
    @property
    def value(self):
        return 42

d2 = Demo2()
print(d2.value)            # 42
d2.__dict__['value'] = 99  # 写入实例字典
print(d2.value)            # 仍然 42 —— @property 是数据描述符，优先级更高
```

> 📖 官方文档：[Invoking Descriptors](https://docs.python.org/3/reference/datamodel.html#invoking-descriptors), [Data model — Implementing Descriptors](https://docs.python.org/3/reference/datamodel.html#descriptors)

## 4.3 内置描述符解构

Python 的内置语法糖几乎都是描述符。当你理解了描述符协议，`@property`、`@staticmethod` 就不再是"魔法"——它们只是实现了 `__get__`（和 `__set__`）的普通类。

### `@property` 的等价手动实现

```python-run
class Property:
    """简化版 property——展示它是如何作为数据描述符工作的"""
    def __init__(self, fget=None, fset=None, fdel=None):
        self.fget = fget
        self.fset = fset
        self.fdel = fdel

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        if self.fget is None:
            raise AttributeError("属性不可读")
        return self.fget(instance)

    def __set__(self, instance, value):
        if self.fset is None:
            raise AttributeError("属性不可写")
        self.fset(instance, value)

    def __delete__(self, instance):
        if self.fdel is None:
            raise AttributeError("属性不可删")
        self.fdel(instance)

    def setter(self, fset):
        return type(self)(self.fget, fset, self.fdel)

    def deleter(self, fdel):
        return type(self)(self.fget, self.fset, fdel)


# 使用简化版 Property
class Temperature:
    def __init__(self, celsius=0):
        self._celsius = celsius

    def get_celsius(self):
        return self._celsius

    def set_celsius(self, value):
        if value < -273.15:
            raise ValueError("低于绝对零度")
        self._celsius = value

    celsius = Property(get_celsius, set_celsius)

t = Temperature(100)
print(t.celsius)     # 100
t.celsius = 50
print(t.celsius)     # 50
```

核心要点：`property` 是一个数据描述符——有 `__get__` 也有 `__set__`，所以它的优先级高于实例 `__dict__`。

### `@staticmethod` 的等价手动实现

```python-run
class StaticMethod:
    """简化版 staticmethod——非数据描述符"""
    def __init__(self, func):
        self.func = func

    def __get__(self, instance, owner=None):
        # 不管 instance 是谁，直接返回原始函数——不绑定
        return self.func

    # 注意：没有 __set__，所以是非数据描述符
    # 这意味着实例可以覆盖它


class Demo:
    @StaticMethod
    def add(a, b):
        return a + b

# 无论是通过类还是实例调用，都不会自动传入 self/cls
print(Demo.add(1, 2))       # 3
d = Demo()
print(d.add(3, 4))           # 3  (Python 真正的 staticmethod 是这么处理的)
```

### `@classmethod` 的等价手动实现

```python-run
class ClassMethod:
    """简化版 classmethod——非数据描述符，但绑定到类"""
    def __init__(self, func):
        self.func = func

    def __get__(self, instance, owner=None):
        if owner is None:
            owner = type(instance)

        def bound(*args, **kwargs):
            # 第一个参数自动传入类
            return self.func(owner, *args, **kwargs)

        return bound


class Demo:
    @ClassMethod
    def from_string(cls, text: str):
        obj = cls()
        obj.text = text
        return obj

d = Demo.from_string("hello")
print(d.text)  # hello
```

`classmethod` 也是非数据描述符（只有 `__get__`）。它和 `staticmethod` 的区别在于 `__get__` 做了什么：`classmethod` 把类绑定为第一个参数，`staticmethod` 什么都不绑。

### `__slots__` 的描述符本质

```python-run
class Point:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

# __slots__ 中声明的属性名对应的是"成员描述符"（member descriptor）
# 它们是类级别的数据描述符，存储在类 __dict__ 中
print(type(Point.__dict__['x']).__name__)  # member_descriptor
print(Point.__dict__['x'])                 # <member 'x' of 'Point' objects>

# member_descriptor 是 C 实现的，你不能用 Python 模拟
# 但它本质上就是一个数据描述符：
# __get__ → 从固定的槽位（C 数组）读取
# __set__ → 写入到固定的槽位
# 没有 __delete__，因为 __slots__ 槽位不能被删除（可以赋值，不能删除属性）

p = Point(1, 2)
print(p.x)           # 1 —— 通过 member_descriptor.__get__ 从槽位读取
p.x = 10             # 通过 member_descriptor.__set__ 写入槽位
# del p.x            # AttributeError! member_descriptor 不支持删除
```

`__slots__` 省内存的秘密：用固定大小的 C 结构体数组替代了 `__dict__` 哈希表。每个槽位对应一个 `member_descriptor`，它们是 C 实现的数据描述符，直接通过数组索引读写——比 `__dict__` 的哈希查找快一个数量级。

> 📖 官方文档：[Implementing Descriptors](https://docs.python.org/3/reference/datamodel.html#descriptors), [The standard type hierarchy — slot descriptor](https://docs.python.org/3/reference/datamodel.html#the-standard-type-hierarchy)

## 4.4 描述符实战

### ValidatedField：类型检查 + 值校验

```python-run
from typing import Any, Callable


class ValidatedField:
    """带类型检查和自定义校验的描述符"""

    def __init__(
        self,
        expected_type: type,
        validators: list[Callable[[Any], bool]] | None = None,
    ):
        self.expected_type = expected_type
        self.validators = validators or []
        self.storage_name = ""

    def __set_name__(self, owner, name: str):
        self.storage_name = f"_{name}"

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        return getattr(instance, self.storage_name)

    def __set__(self, instance, value):
        if not isinstance(value, self.expected_type):
            raise TypeError(
                f"期望 {self.expected_type.__name__}，"
                f"实际 {type(value).__name__}"
            )
        for v in self.validators:
            if not v(value):
                raise ValueError(f"校验失败: {value}")
        setattr(instance, self.storage_name, value)


class User:
    name = ValidatedField(str, validators=[lambda v: len(v) > 0])
    age = ValidatedField(int, validators=[lambda v: v >= 0])

    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age


u = User("Alice", 30)
# u.age = -1  # ValueError: 校验失败
# u.name = ""  # ValueError: 校验失败
print(f"User: name={u.name}, age={u.age}")
```

### LazyProperty：计算一次，永久缓存

```python-run
class LazyProperty:
    """非数据描述符——只有 __get__，计算后写入实例 __dict__ 覆盖自身"""

    def __init__(self, func):
        self.func = func
        self.__doc__ = func.__doc__

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        # 计算值并写入实例 __dict__
        # 因为是非数据描述符，下次访问时 instance.__dict__ 优先
        value = self.func(instance)
        instance.__dict__[self.func.__name__] = value
        return value


class DataSet:
    def __init__(self, numbers: list[int]):
        self.numbers = numbers

    @LazyProperty
    def avg(self):
        print("  (正在计算 avg...)")
        return sum(self.numbers) / len(self.numbers)


ds = DataSet([1, 2, 3, 4, 5])
print(ds.avg)  # (正在计算 avg...) → 3.0
print(ds.avg)  # 3.0 —— 直接从 __dict__ 读取，不再计算
```

`LazyProperty` 用**非数据描述符**实现的关键：第一次 `__get__` 后把值写入 `instance.__dict__[name]`。由于非数据描述符优先级低于实例字典，下次访问直接从 `__dict__` 读取，描述符的 `__get__` 不再被触发。

### AliasField：属性别名

```python-run
class Alias:
    """将一个属性名映射到另一个属性名——数据描述符"""

    def __init__(self, target: str):
        self.target = target

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        return getattr(instance, self.target)

    def __set__(self, instance, value):
        setattr(instance, self.target, value)

    def __delete__(self, instance):
        delattr(instance, self.target)


class Contact:
    email = "alice@example.com"
    mail = Alias("email")   # mail 是 email 的别名


c = Contact()
print(c.mail)          # alice@example.com
c.mail = "bob@test.com"
print(c.email)         # bob@test.com —— 别名修改了原始属性
```

### PositiveNumber：约束值域

```python-run
class Positive:
    """强制属性值为正数"""

    def __set_name__(self, owner, name: str):
        self.name = name
        self.storage = f"_{name}"

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        return getattr(instance, self.storage, 0)  # 默认 0

    def __set__(self, instance, value):
        if not isinstance(value, (int, float)):
            raise TypeError(f"{self.name} 必须是数字")
        if value <= 0:
            raise ValueError(f"{self.name} 必须 > 0，得到 {value}")
        setattr(instance, self.storage, value)


class Rectangle:
    width = Positive()
    height = Positive()

    def __init__(self, width: float, height: float):
        self.width = width
        self.height = height

    @property
    def area(self):
        return self.width * self.height


r = Rectangle(5.0, 3.0)
print(f"面积: {r.area}")   # 15.0
# r.width = -1              # ValueError: width 必须 > 0
```

### DRF 风格序列化器

Django REST Framework 的 `Serializer` 用描述符来表示字段。下面是这个模式的简化实现：

```python-run
class Field:
    """序列化器的字段描述符"""
    def __set_name__(self, owner, name: str):
        self.name = name

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        # 从实例的内部数据字典中取值
        return instance._data.get(self.name)

    def __set__(self, instance, value):
        if isinstance(value, str):
            instance._data[self.name] = self._clean(value)
        else:
            instance._data[self.name] = value

    def _clean(self, value):
        return value


class CharField(Field):
    def _clean(self, value):
        return str(value).strip()


class IntegerField(Field):
    def _clean(self, value):
        return int(value)


class Serializer:
    def __init__(self, data: dict):
        self._data: dict = {}
        for key, value in data.items():
            if hasattr(type(self), key):
                setattr(self, key, value)
            else:
                self._data[key] = value

    @property
    def validated_data(self):
        return dict(self._data)


class UserSerializer(Serializer):
    name = CharField()
    age = IntegerField()


s = UserSerializer({"name": " Alice ", "age": "30"})
print(s.name)                  # "Alice" —— CharField 做了 strip
print(s.age)                   # 30 —— IntegerField 做了 int()
print(s.validated_data)        # {'name': 'Alice', 'age': 30}
```

## 4.5 描述符 + 元类：ORM 列收集

描述符和元类经常协同工作。经典模式：**描述符定义"列"的行为（类型、校验）**，**元类在类创建时收集所有列**。

```python-run
class Column:
    """数据库列描述符"""
    def __set_name__(self, owner, name: str):
        self.name = name

    def __get__(self, instance, owner=None):
        if instance is None:
            return self
        return instance._row.get(self.name)

    def __set__(self, instance, value):
        instance._row[self.name] = value


class ModelMeta(type):
    """元类：自动收集 Column 实例"""
    def __new__(mcs, name, bases, namespace):
        cls = super().__new__(mcs, name, bases, namespace)
        # 收集所有 Column 类型描述符
        columns = {
            k: v for k, v in namespace.items()
            if isinstance(v, Column)
        }
        cls._columns = columns
        return cls


class Model(metaclass=ModelMeta):
    _columns: dict = {}

    def __init__(self, **kwargs):
        self._row: dict = {}
        for col_name in self._columns:
            setattr(self, col_name, kwargs.get(col_name))

    def save(self):
        cols = ", ".join(self._columns)
        vals = ", ".join(repr(self._row.get(c)) for c in self._columns)
        print(f"INSERT INTO {type(self).__name__.lower()} "
              f"({cols}) VALUES ({vals})")


class User(Model):
    name = Column()
    age = Column()
    email = Column()


print(f"User 列: {User._columns.keys()}")  # dict_keys(['name', 'age', 'email'])
u = User(name="Bob", age=30, email="bob@test.com")
u.save()  # INSERT INTO user (name, age, email) VALUES ('Bob', 30, 'bob@test.com')
```

**分工清晰**：
- `Column`（描述符）：负责单个属性的读写行为。
- `ModelMeta`（元类）：负责类级别的元信息收集——哪些属性是列。
- `Model`（基类）：负责实例化、`save()` 等公共行为。

这就是 SQLAlchemy、Django ORM 的核心架构思路——用描述符刻画字段，用元类组装元信息。

## 本章小结

- **描述符协议**：实现了 `__get__`/`__set__`/`__delete__` 的对象即为描述符，控制属性的访问行为；描述符必须定义在类上，实例上的同名属性不会触发协议。
- **数据描述符 vs 非数据描述符**：数据描述符（`__get__` + `__set__`/`__delete__`）优先级高于实例 `__dict__`，非数据描述符（仅有 `__get__`）反之——这解释了为什么 `@property` 不能被实例覆盖而 `@staticmethod` 可以。
- **内置描述符**：`@property`、`@staticmethod`、`@classmethod`、`__slots__` 都是描述符的具体实现，核心差异在于 `__get__` 返回什么以及是否实现了 `__set__`。
- **`__set_name__`**：类创建时自动调用，让描述符获取属性名，避免手动重复声明；`LazyProperty` 利用非数据描述符优先级实现"计算一次、永久缓存"。
- **描述符 + 元类**：是 ORM 的核心架构——描述符定义字段的读写行为，元类在类创建时收集所有字段元信息（如 SQLAlchemy 的 Column + ModelMeta）。
