# 0.3 list 与 tuple

## list（列表）—— 可变序列

list 是 Python 中最常用的容器，可以放任意类型的元素：

```python-run
items = ["hello", 42, 3.14, True]
print(f"长度: {len(items)}")
print(f"第一个: {items[0]}")
print(f"倒数第一个: {items[-1]}")   # 负索引从末尾开始
```

### 切片（Slicing）：`seq[start:stop:step]`

```python-run
nums = [0, 1, 2, 3, 4, 5]
print(f"前三个: {nums[:3]}")
print(f"后三个: {nums[-3:]}")
print(f"隔一个取一个: {nums[::2]}")
print(f"倒序: {nums[::-1]}")
```

切片返回**新 list**，不修改原 list。

### 增删改

```python-run
fruits = ["苹果", "香蕉"]
fruits.append("橘子")          # 末尾追加
fruits.insert(1, "西瓜")       # 指定位置插入
print(fruits)

fruits.remove("香蕉")          # 按值删除第一个匹配
popped = fruits.pop()          # 删除并返回末尾元素
print(f"pop: {popped}, 剩余: {fruits}")
```

## tuple（元组）—— 不可变序列

tuple 一旦创建，**结构**不可改变。但它比"不可变的 list"有更深的意义：

```python-run
# tuple 适合表示"一组有结构的固定数据"
point = (3, 4)
person = ("张三", 25, "北京")
# point[0] = 5   # TypeError! 不可修改
```

### "不可变"是浅层的

tuple 不可变指的是**不能增删改元素引用**，但如果元素本身是可变的，那元素的内容可以变：

```python-run
t = ([1, 2], [3, 4])
t[0].append(99)        # OK — list 元素没变，变的是 list 内部
print(t)               # ([1, 2, 99], [3, 4])
```

> 这正是 **tuple 适合做 dict 的 key** 的原因：只要元素引用不变，哈希值就不变。

## 解包（Unpacking）—— Python 的"模式匹配"雏形

```python-run
# 平行赋值
a, b, c = [1, 2, 3]

# * 收集"剩余"
first, *middle, last = [1, 2, 3, 4, 5]
print(first, middle, last)  # 1 [2, 3, 4] 5

# * 展开
a = [1, 2, 3]
b = [*a, 4, 5]
print(b)  # [1, 2, 3, 4, 5]
```

## list vs tuple — 什么时候用哪个？

| 场景 | 用 |
|------|-----|
| 同类型、同含义的多个元素 | `list` — `scores = [85, 92, 78]` |
| 异构、固定结构的多个元素 | `tuple` — `point = (x, y, z)` |
| 需要增删改 | `list` |
| 需要可哈希（做 key） | `tuple` |

## 本章小结

- `list` 是可变序列，支持切片 `[start:stop:step]`、`append`、`insert`、`remove`、`pop` 等操作
- `tuple` 是不可变序列，但"不可变"是浅层的——元素引用不可变，可变元素内部仍可修改
- 解包（Unpacking）支持平行赋值、`*` 收集剩余元素和 `*` 展开——是 Python 模式匹配的雏形
- 选 `list` 还是 `tuple`：同类型同含义用 `list`，异构固定结构用 `tuple`；需要可哈希（做 dict key）时必须用 `tuple`
