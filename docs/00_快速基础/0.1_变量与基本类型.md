# 0.1 变量与基本类型

## 变量就是"名字标签"

在 Python 中，**变量不是"装数据的盒子"，而是贴在对象上的"标签"**。这是理解 Python 对象模型的第一课。

```python-run
x = 42
print(f"x 指向: {x}, 类型: {type(x)}")

x = "hello"   # x 现在指向另一个完全不同的对象
print(f"x 现在指向: {x}, 类型: {type(x)}")
```

同一行可以给多个名字贴标签：

```python-run
a, b, c = 1, 2, "python"
print(a, b, c)

# 交换值一行搞定——背后是先构造元组再解包
a, b = b, a
print(f"交换后: a={a}, b={b}")
```

> `=` 的真正含义是**名字绑定**（Name Binding），不是赋值。每次 `=` 都在"贴标签"或"换标签"。

## 四个基本类型

### int — 整数

Python 3 的 `int` **没有溢出**——它内部用变长数组存储，内存有多大就能有多大：

```python-run
n = 2 ** 100
print(f"2 的 100 次方 = {n}")
print(f"这个数有 {n.bit_length()} 位")
```

> Python 3 把 `int` 和 `long` 统一了，不再需要区分。

### float — 浮点数

`float` 是 IEEE 754 双精度浮点数，和其他语言一样有精度限制：

```python-run
print(f"0.1 + 0.2 = {0.1 + 0.2}")   # 不是 0.3
print(f"0.1 + 0.2 == 0.3: {0.1 + 0.2 == 0.3}")
```

需要精确十进制运算时用 `Decimal`（后面讲）。

### bool — 布尔值

`bool` 是 `int` 的子类！`True` 就是 `1`，`False` 就是 `0`：

```python-run
print(f"True == 1: {True == 1}")
print(f"isinstance(True, int): {isinstance(True, int)}")
print(f"True + True = {True + True}")  # 2
```

### None — 空

`None` 是整个解释器中唯一的单例对象，表示"没有值"。**判断是否 None 用 `is`，不用 `==`**：

```python-run
result = None
print(result is None)  # True — 推荐写法
```

## `type()` vs `isinstance()`

```python-run
# type() — 查精确类型
print(type(42))

# isinstance() — 查继承链（推荐）
print(isinstance(42, int))       # True
print(isinstance(True, int))     # True! bool 是 int 的子类
print(isinstance(42, float))     # False
```

> **规则**：需要精确类型时用 `type()`，检查"能不能当作某类型用"时用 `isinstance()`。后者是鸭子类型（Duck Typing）的基础。

## 本章小结

- 变量是贴在对象上的"名字标签"，`=` 的本质是名字绑定而非赋值——同一变量可以先后指向不同类型的对象
- Python 3 的 `int` 没有溢出；`float` 是 IEEE 754 双精度，`0.1 + 0.2 != 0.3`——需要精确运算时用 `Decimal`
- `bool` 是 `int` 的子类，`True == 1`、`False == 0`；`None` 是唯一单例，判断用 `is` 而非 `==`
- 查精确类型用 `type()`，查"能不能当某类型用"用 `isinstance()`——后者是鸭子类型的基础
