# 6.8 递归与分治

递归是**一个函数调用自己**，分治是**把大问题拆成小问题再合并**。这不仅是算法技巧，更是编程思维方式——很多问题用递归表达比迭代简洁十倍。

> 📖 **官方文档**：[sys.setrecursionlimit](https://docs.python.org/3/library/sys.html#sys.setrecursionlimit)

---

## 1.1 递归三要素

每个递归函数都必须满足三个条件，缺一不可：

| 要素 | 说明 | 示例（阶乘） |
|------|------|-------------|
| **基案（Base Case）** | 不再递归，直接返回 | `if n <= 1: return 1` |
| **递归情况** | `问题规模缩小`的自我调用 | `return n * factorial(n-1)` |
| **收敛保证** | 每次调用都向基案靠近 | n 每次减 1，最终到达 1 |

```python-run
def fib(n):
    # 结束条件: 基案 不再递归直接返回
    if n <=0:
        return 0
    if n <= 2:
        return 1
    
    return fib(n=n-1) + fib(n-2) # 递归情况(n缩小的自我调用, 同时向基案收敛)

def fib_opt(n,memo=None):
    '''
    使用缓存提升效率
    '''
    
    if memo is None:
        memo = {}
    if n in memo:
        return memo[n]
    # 结束条件
    if n <= 0:
        return 0
    if n <= 2:
        return 1
    
    memo[n] = fib_opt(n-1,memo) + fib_opt(n-2,memo)
    return memo[n]

from functools import lru_cache
@lru_cache
def fib_opt2(n):
    '''
    lru_cache: 底层使用c, 但是包装函数有额外的开销, 数量和运行数量多的时候提升明显 
    '''
    
    if n <= 0:
        return 0
    if n <= 2:
        return 1
    return fib_opt2(n-1) + fib_opt2(n-2)
```

## 1.2 Python 递归的限制

```python-run
import sys

print(f"递归深度限制: {sys.getrecursionlimit()}")

def bad_recursion(n):
    """没有基案的递归——必定堆栈溢出"""
    if n == 0:
        return
    return bad_recursion(n - 1)   # 还好这个有基案

# Python 不做尾递归优化（TCO），Guido 明确拒绝了 PEP 510
# 原因：TCO 会丢失调用栈信息，不利于调试
```

> **Python 不支持尾递归优化**。深度递归（> 1000 层）要么用迭代改写，要么 `sys.setrecursionlimit()`，或者用显式栈模拟。

## 1.3 递归 vs 迭代：两种思维

```python-run
# 同一问题，两种表达——递归更"声明式"，迭代更"过程式"

# 递归版：像数学定义
def sum_recursive(n):
    if n <= 0:
        return 0
    return n + sum_recursive(n - 1)

# 迭代版：像过程指令
def sum_iterative(n):
    total = 0
    for i in range(1, n + 1):
        total += i
    return total

print(f"sum_recursive(100) = {sum_recursive(100)}")
print(f"sum_iterative(100) = {sum_iterative(100)}")
```

| 维度 | 递归 | 迭代 |
|------|------|------|
| **代码简洁度** | ★★★★★（声明式） | ★★★ |
| **性能** | 每次调用有开销 | ★★★★★ |
| **栈溢出风险** | 有（Python 默认 ~1000 层） | 无 |
| **适合场景** | 树/图遍历、分治、回溯 | 简单循环 |

## 1.4 分治（Divide and Conquer）

分治 = **分解 → 解决 → 合并**。最经典的三个应用：

```python-run
# 1. 归并排序 — O(n log n)
def merge_sort(arr):
    if len(arr) <= 1:                  # 基案
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])       # 分解 + 递归解决
    right = merge_sort(arr[mid:])
    return merge(left, right)          # 合并

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

print(f"归并排序: {merge_sort([38, 27, 43, 3, 9, 82, 10])}")


# 2. 快速排序 — 平均 O(n log n)
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[0]
    less = [x for x in arr[1:] if x <= pivot]
    greater = [x for x in arr[1:] if x > pivot]
    return quick_sort(less) + [pivot] + quick_sort(greater)

print(f"快速排序: {quick_sort([38, 27, 43, 3, 9, 82, 10])}")


# 3. 二分查找 — O(log n)
def binary_search(arr, target, lo=0, hi=None):
    if hi is None:
        hi = len(arr) - 1
    if lo > hi:                         # 基案：没找到
        return -1
    mid = (lo + hi) // 2
    if arr[mid] == target:
        return mid
    if arr[mid] > target:
        return binary_search(arr, target, lo, mid - 1)
    return binary_search(arr, target, mid + 1, hi)

arr = [3, 9, 10, 27, 38, 43, 82]
print(f"二分查找 27: 索引={binary_search(arr, 27)}")
```

## 1.5 递归的常见陷阱

### 陷阱 1：重复计算

```python-run
# 斐波那契递归：O(2^n) — 大量重复计算
def fib_naive(n):
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)  # 相同的 fib 值算了无数次

# 解决：记忆化（缓存中间结果）
from functools import lru_cache

@lru_cache(maxsize=None)
def fib_cache(n):
    if n <= 1:
        return n
    return fib_cache(n - 1) + fib_cache(n - 2)   # O(n)！

print(f"fib(30) 记忆化 = {fib_cache(30)}")
```

### 陷阱 2：循环中的闭包延迟绑定

```python-run
# ❌ 经典错误：循环中 Lambda 捕获的都是最后一次迭代的值
funcs_bad = [lambda: i for i in range(3)]
print([f() for f in funcs_bad])       # [2, 2, 2] — 不是 [0, 1, 2]

# ✅ 修复：用默认参数"冻结"当前值
funcs_good = [lambda x=i: x for i in range(3)]
print([f() for f in funcs_good])      # [0, 1, 2]
```

## 1.6 分治的主定理速览

对于递归式 `T(n) = a·T(n/b) + f(n)`：

| 情况 | 条件 | T(n) | 例子 |
|------|------|------|------|
| f(n) 增长更慢 | f(n) = O(n^(log_b a - ε)) | Θ(n^(log_b a)) | 二分查找（a=1, b=2）→ Θ(log n) |
| f(n) 同等增长 | f(n) = Θ(n^(log_b a)) | Θ(n^(log_b a) · log n) | 归并排序（a=2, b=2）→ Θ(n log n) |
| f(n) 增长更快 | f(n) = Ω(n^(log_b a + ε)) | Θ(f(n)) | Strassen 矩阵乘法 |

> 不需要背——知道"分治比暴力快多少"就够。

## 本章小结

1. **递归三要素**：基案、递归情况、收敛保证——缺一个就栈溢出
2. **Python 不做尾递归优化**（调试 > 性能），深度递归要么改迭代，要么 `setrecursionlimit`
3. **递归 = 声明式（像数学定义），迭代 = 过程式（像指令）**——树遍历用递归，简单计数用迭代
4. **分治 = 分解 + 递归解决 + 合并**。归并排序、快速排序、二分查找都是分治
5. **记忆化（`@lru_cache`）消除重复计算**，是递归 → DP 的桥梁
