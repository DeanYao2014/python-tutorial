# 6.10 搜索算法

搜索是程序员每天在写却不一定意识到自己在写的算法。"在集合中找到目标"这件事，数据结构的选择决定了效率的下限，算法的选择决定了上限。

> 📖 **官方文档**：[bisect — Array bisection algorithm](https://docs.python.org/3/library/bisect.html)

---

## 1.1 `bisect`：Python 二分查找标准答案

不要手写 `while lo < hi`——`bisect` 模块是二分查找的官方实现。

```python-run
import bisect

nums = [1, 3, 3, 3, 5, 7, 9]

# bisect_left：找插入位置，相等时在左边
print(f"bisect_left(3) = {bisect.bisect_left(nums, 3)}")   # 1
print(f"  元素 3 第一次出现的位置")

# bisect_right：找插入位置，相等时在右边
print(f"bisect_right(3) = {bisect.bisect_right(nums, 3)}") # 4
print(f"  元素 3 之后的位置")

# insort：插入并用 O(n) 保持有序
sorted_list = [1, 3, 5, 7]
bisect.insort(sorted_list, 4)
print(f"insort(4) → {sorted_list}")  # [1, 3, 4, 5, 7]
```

### bisect 对照表

| 场景 | 用的函数 | 效果 |
|------|---------|------|
| 找 target 第一次出现 | `bisect_left(arr, target)` | 返回索引，`arr[i]` 是第一个 >= target 的元素 |
| 找 target 的插入位置 | `bisect_right(arr, target)` | 在已有相等元素之后插入 |
| 判断 target 是否存在 | `bisect_left(arr, target) < len(arr) and arr[i] == target` | — |
| 有序插入 | `bisect.insort(arr, x)` | O(log n) 定位 + O(n) 插入 |

## 1.2 二分查找的变体

```python-run
import bisect

# 变体 1：找第一个 >= target 的位置
def first_ge(arr, target):
    idx = bisect.bisect_left(arr, target)
    return idx if idx < len(arr) else -1

# 变体 2：找最后一个 <= target 的位置
def last_le(arr, target):
    idx = bisect.bisect_right(arr, target) - 1
    return idx if idx >= 0 else -1

# 变体 3：在旋转排序数组中查找（如 [4,5,6,7,0,1,2]）
def search_rotated(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid
        # 判断哪半部分是有序的
        if nums[lo] <= nums[mid]:        # 左半有序
            if nums[lo] <= target < nums[mid]:
                hi = mid - 1
            else:
                lo = mid + 1
        else:                            # 右半有序
            if nums[mid] < target <= nums[hi]:
                lo = mid + 1
            else:
                hi = mid - 1
    return -1

arr = [4, 5, 6, 7, 0, 1, 2]
print(f"旋转数组找 0: 索引={search_rotated(arr, 0)}")   # 4
print(f"旋转数组找 3: 索引={search_rotated(arr, 3)}")   # -1
```

## 1.3 二分答案：把优化问题变成判定问题

二分查找的思想远不止"在有序数组中找元素"。**当答案是单调的（越大越好或越小越好），可以用二分在答案空间上搜索**。

```python-run
# 经典题：分巧克力——n 块巧克力分给 m 个小朋友
# 每人分一块，要求分到的最少的那块尽量大
def can_divide(sizes, k, min_size):
    """判定：每人至少分 min_size，够不够分给 k 个人"""
    count = sum(s // min_size for s in sizes)
    return count >= k

def max_min_chocolate(sizes, k):
    """二分答案：最大化的最小块"""
    lo, hi = 1, max(sizes)
    while lo < hi:
        mid = (lo + hi + 1) // 2
        if can_divide(sizes, k, mid):
            lo = mid       # 够分 → 试试更大的
        else:
            hi = mid - 1   # 不够 → 减小
    return lo

chocolates = [10, 20, 30]
k = 5
print(f"每人至少分到: {max_min_chocolate(chocolates, k)}")
# 10/2 + 20/2 + 30/2 = 5+10+15=30 >=5 ✓, 再大就不行了
```

> 二分答案的核心：**问题有单调性**（答案越大越难满足 / 答案越小越容易满足），于是可以把"找最优解"变成"猜答案 + 判定"。

## 1.4 滑动窗口（Two Pointers）

滑动窗口解决的是"连续子数组"类问题——用两个指针维护一个窗口，O(n) 代替 O(n²)。

```python-run
# 最长无重复字符子串
def length_of_longest_substring(s):
    seen = {}
    left = 0
    max_len = 0

    for right, ch in enumerate(s):
        if ch in seen and seen[ch] >= left:
            left = seen[ch] + 1      # 窗口左边界右移
        seen[ch] = right
        max_len = max(max_len, right - left + 1)
    return max_len

print(f"'abcabcbb' 最长无重复: {length_of_longest_substring('abcabcbb')}")  # 3
print(f"'pwwkew' 最长无重复: {length_of_longest_substring('pwwkew')}")    # 3
```

```python-run
# 长度最小的子数组（sum >= target）
def min_subarray_len(target, nums):
    left = total = 0
    min_len = float('inf')

    for right, v in enumerate(nums):
        total += v
        while total >= target:
            min_len = min(min_len, right - left + 1)
            total -= nums[left]
            left += 1
    return 0 if min_len == float('inf') else min_len

print(f"target=7, nums=[2,3,1,2,4,3] → {min_subarray_len(7, [2,3,1,2,4,3])}") # 2
```

## 1.5 搜索策略选择表

| 场景 | 策略 | 复杂度 |
|------|------|--------|
| 有序数组中查找 | `bisect` 二分查找 | O(log n) |
| "存在吗？" | `in set` / `in dict` | O(1) |
| "在有序数组中找边界" | `bisect_left` / `bisect_right` | O(log n) |
| 单调最优解问题 | 二分答案 | O(log range × 判定时间) |
| 连续子数组问题 | 滑动窗口 | O(n) |
| 无权图最短路径 | BFS | O(V + E) |
| 暴力 O(n²) 的数组查找 | 先排序 O(n log n) 再二分 O(log n) | O(n log n) |

## 本章小结

1. **`bisect` 是二分查找的标准答案**——不手写 `while lo < hi`，用 `bisect_left`/`bisect_right`
2. **二分答案的思想**：遇到"最大化的最小值 / 最小化的最大值"型问题，用二分在答案空间搜索
3. **滑动窗口**：两个指针 + 条件判断，O(n) 解决连续子数组问题——关键是窗口何时收缩
4. **搜索的效率由数据结构决定**：O(1) 用 `set`/`dict`，O(log n) 用 `bisect`，O(n) 用遍历/滑动窗口
