# 6.4 哈希表原理

哈希表是计算机科学中最伟大的发明之一——O(1) 的查找、插入、删除。Python 的 `dict` 和 `set` 就是哈希表的实现。这一章不重复 1.3 节的 dict 底层分析，而是聚焦**算法视角**：哈希函数设计、冲突解决策略、以及哈希思想的应用模式。

> 📖 **官方文档**：[dict](https://docs.python.org/3/library/stdtypes.html#dict), [hash()](https://docs.python.org/3/library/functions.html#hash)

---

## 1.1 哈希表的三个核心问题

任何哈希表都必须回答三个问题：

| 问题 | Python 的答案 |
|------|--------------|
| 如何计算哈希值？ | `hash(key)` — 对内置类型保证 `a == b ⇒ hash(a) == hash(b)` |
| 如何映射到数组索引？ | `hash(key) & mask` — 用位掩码取模（表大小是 2 的幂） |
| 哈希冲突怎么处理？ | **开放寻址法** — 探测下一个位置直到找到空位 |

```python-run
# hash() 的基本行为
print(f"hash(42) = {hash(42)}")
print(f"hash('hello') = {hash('hello')}")
print(f"hash((1, 2, 3)) = {hash((1, 2, 3))}")

# 相等 ⇒ 哈希相同
print(f"hash(1.0) = {hash(1.0)}, hash(1) = {hash(1)}")
print(f"hash(1.0) == hash(1): {hash(1.0) == hash(1)}")   # True

# 可变对象不可哈希
try:
    hash([1, 2, 3])
except TypeError as e:
    print(f"列表不可哈希: {e}")
```

> 可变对象不可哈希的原因很简单：如果 key 内容变了，`hash(key)` 就变了，dict 在原来位置找不到它。

## 1.2 哈希冲突的两种解决策略

### 拉链法（Separate Chaining）

每个桶存一个链表（或红黑树）。冲突时往链表里追加。Java 的 `HashMap` 用这种。

```
桶数组: [0] → None
        [1] → (key1, val1) → (key2, val2) → None  ← 冲突的两个键挂在同一桶
        [2] → (key3, val3) → None
```

### 开放寻址法（Open Addressing）— Python 的选择

冲突时在数组内部**线性探测**下一个空位。Python 3.6+ 还加入了**紧凑实现**：

```
索引表 = [0, -1, 2, 1, ...]    ← 只存 entry 编号（稀疏，每项 1 字节）
entries = [[hash, key, value], ...] ← 紧密排列，无空洞
```

关键优势：**更好的 CPU 缓存命中率** + **自动保持插入顺序**（3.7+）。

## 1.3 手写一个简化版哈希表

```python-run
class SimpleDict:
    """40 行实现一个拉链法哈希表，展示核心原理"""
    def __init__(self, capacity=8):
        self.capacity = capacity
        self.size = 0
        self.buckets = [[] for _ in range(capacity)]

    def _hash(self, key) -> int:
        return hash(key) % self.capacity

    def _resize(self):
        """负载因子 > 0.75 时扩容"""
        old_buckets = self.buckets
        self.capacity *= 2
        self.size = 0
        self.buckets = [[] for _ in range(self.capacity)]
        for bucket in old_buckets:
            for k, v in bucket:
                self.set(k, v)

    def set(self, key, value):
        if self.size / self.capacity > 0.75:
            self._resize()
        idx = self._hash(key)
        bucket = self.buckets[idx]
        # 如果 key 已存在，更新
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return
        # 新 key，追加
        bucket.append((key, value))
        self.size += 1

    def get(self, key, default=None):
        idx = self._hash(key)
        for k, v in self.buckets[idx]:
            if k == key:
                return v
        return default

    def delete(self, key):
        idx = self._hash(key)
        bucket = self.buckets[idx]
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket.pop(i)
                self.size -= 1
                return

# 演示
d = SimpleDict()
d.set("name", "Python")
d.set("type", "language")
d.set("year", 1991)
print(f"get('name'): {d.get('name')}")
print(f"get('missing'): {d.get('missing', 'N/A')}")
d.delete("year")
print(f"删除 'year' 后 get('year'): {d.get('year', 'N/A')}")
```

## 1.4 Counter：哈希表的经典应用

`collections.Counter` 用哈希表做**频次统计**——是哈希思想最优雅的应用之一。

```python-run
from collections import Counter

text = "abracadabra"
counter = Counter(text)
print(f"字符频次: {counter}")
print(f"最多的 3 个: {counter.most_common(3)}")

# Counter 就是 dict 的子类
print(f"isinstance(counter, dict): {isinstance(counter, dict)}")

# 算法题：找出出现次数最多的元素
nums = [1, 1, 1, 2, 2, 3, 3, 3, 3]
print(f"\n{nums} 中最常见的: {Counter(nums).most_common(1)}")
```

> 📖 **官方文档**：[collections.Counter](https://docs.python.org/3/library/collections.html#collections.Counter)

## 1.5 `defaultdict`：消灭 `if key not in dict`

很多算法题的核心操作是"把元素分组"。`defaultdict` 省掉了每次检查 key 是否存在的样板代码。

```python-run
from collections import defaultdict

# ❌ 没有 defaultdict
groups = {}
words = ["cat", "car", "bar", "bat", "rat"]
for w in words:
    key = w[0]           # 按首字母分组
    if key not in groups:
        groups[key] = []
    groups[key].append(w)

# ✅ 有 defaultdict
groups2 = defaultdict(list)
for w in words:
    groups2[w[0]].append(w)

print(f"defaultdict 结果: {dict(groups2)}")

# 经典用法：字母异位词分组
anagrams = defaultdict(list)
for w in ["eat", "tea", "tan", "ate", "nat", "bat"]:
    anagrams[''.join(sorted(w))].append(w)
print(f"异位词分组: {list(anagrams.values())}")
```

## 1.6 哈希思想的应用模式

| 模式 | 描述 | Python 工具 |
|------|------|------------|
| **去重** | 利用 key 唯一性 | `set`, `dict.fromkeys()` |
| **计数** | key→出现次数 | `Counter`, `defaultdict(int)` |
| **分组** | key→元素列表 | `defaultdict(list)` |
| **映射/缓存** | key→计算结果 | `dict`, `@lru_cache` |
| **存在性检查** | O(1) 判断元素是否见过 | `set` |
| **两数之和** | 遍历时在哈希表中找 target-x | `dict` |

```python-run
# 两数之和：哈希表 O(n) vs 暴力 O(n²)
def two_sum(nums, target):
    """返回两个加起来等于 target 的数的下标"""
    seen = {}                     # val → index
    for i, v in enumerate(nums):
        need = target - v
        if need in seen:          # O(1) 检查
            return [seen[need], i]
        seen[v] = i
    return []

print(two_sum([2, 7, 11, 15], 9))   # [0, 1]
```

## 本章小结

1. **哈希表 = 数组 + 哈希函数 + 冲突解决**。Python 用开放寻址法，3.6+ 紧凑实现省内存
2. **可变对象不可哈希**——内容变了哈希值就变了，dict 找不到原来的位置
3. **负载因子**决定何时扩容。Python dict 在负载约 2/3 时扩容
4. `Counter` 做频次统计，`defaultdict` 省去 key 存在性检查——都是哈希思想的产品化
5. "在集合中找某个元素"、"判断某个元素是否存在"、"将元素按特征分组"——这三类问题优先考虑哈希表
