# 6.3 栈与队列

栈和队列是操作受限的线性结构——它们定义了"数据进出的规则"而非"数据如何存储"。Python 的 `list` 和 `collections.deque` 让它们的实现变成一行代码的事，但理解背后的设计模式才值钱。

> 📖 **官方文档**：[collections.deque](https://docs.python.org/3/library/collections.html#collections.deque)

---

## 1.1 栈（Stack）：后进先出 LIFO

### 是什么

栈就像一摞盘子——**最后放上去的，最先取下来**。核心操作只有两个：`push`（压入）和 `pop`（弹出），都是 O(1)。

```python-run
# Python list 天然就是栈
stack = []

stack.append(1)     # push
stack.append(2)
stack.append(3)
print(f"栈: {stack}")

print(f"pop → {stack.pop()}")   # 3（最后进的先出）
print(f"pop → {stack.pop()}")   # 2
print(f"剩余: {stack}")         # [1]
```

### 经典应用场景

| 场景 | 为什么用栈 |
|------|-----------|
| **括号匹配** | 左括号入栈，右括号匹配栈顶 |
| **函数调用栈** | 每个函数调用压入调用栈，return 时弹出 |
| **撤销操作** | 每次操作压栈，Ctrl+Z 弹出最近操作 |
| **DFS 深度优先** | 用栈记住"待回头的岔路口" |
| **表达式求值** | 运算符栈 + 操作数栈 |

```python-run
# 括号匹配：20 行判断括号是否合法
def is_valid(s: str) -> bool:
    pairs = {')': '(', ']': '[', '}': '{'}
    open_brackets = set('([{')  # 定义左括号集合
    stack = []
    for ch in s:
        if ch in pairs:                              # 右括号
            if not stack or stack.pop() != pairs[ch]:
                return False
        elif ch in open_brackets:                    # 只有左括号才入栈
            stack.append(ch)
        # 其他普通字符直接跳过，什么都不做
    return not stack 

print(is_valid("()[]{}"))     # True
print(is_valid("([)]"))       # False
print(is_valid("{[]}"))       # True
```

## 1.2 队列（Queue）：先进先出 FIFO

### 是什么

队列就像排队——**先来的先服务**。核心操作：`enqueue`（入队）和 `dequeue`（出队）。

```python-run
from collections import deque

# deque 是专职双端队列——popleft() 是 O(1)
queue = deque()

queue.append(1)        # 入队
queue.append(2)
queue.append(3)
print(f"队列: {list(queue)}")

print(f"出队 → {queue.popleft()}")   # 1（最早进的先出）
print(f"出队 → {queue.popleft()}")   # 2
print(f"剩余: {list(queue)}")        # [3]
```

> 为什么不用 `list.pop(0)`？O(n) 每次要搬移所有元素。`deque.popleft()` 是 O(1)。

### 经典应用场景

| 场景 | 为什么用队列 |
|------|-------------|
| **BFS 广度优先** | 逐层扩展，"先发现的先处理" |
| **消息队列** | 生产者-消费者解耦 |
| **打印队列** | 先提交的文档先打印 |
| **滑动窗口** | 窗口右移 = 右边入队 + 左边出队 |

```python-run
from collections import deque

# BFS 遍历二叉树的逐层访问
def bfs(root):
    """层序遍历：一层一层地访问"""
    if not root:
        return
    q = deque([root])
    while q:
        node = q.popleft()
        print(node.val, end=' ')
        if node.left:
            q.append(node.left)
        if node.right:
            q.append(node.right)

# 简化的树结点
class Node:
    def __init__(self, val, left=None, right=None):
        self.val, self.left, self.right = val, left, right

tree = Node(1, Node(2, Node(4), Node(5)), Node(3, None, Node(6)))
bfs(tree)  # 输出: 1 2 3 4 5 6
```

## 1.3 单调栈与单调队列

这是栈/队列最精妙的高级应用——**维护数据结构内的元素单调递增或递减**。

### 单调栈：找到每个元素"下一个更大值"

```python-run
def next_greater_elements(nums: list[int]) -> list[int]:
    """找到每个元素右边第一个比它大的数。单调栈 O(n)"""
    n = len(nums)
    result = [-1] * n
    stack = []                           # 存下标，栈内值单调递减
    for i, v in enumerate(nums):
        while stack and nums[stack[-1]] < v:
            idx = stack.pop()
            result[idx] = v
        stack.append(i)
    return result

print(next_greater_elements([2, 1, 2, 4, 3]))
# [4, 2, 4, -1, -1]
#  2→4, 1→2, 2→4, 4→无, 3→无
```

**核心思想**：栈里存的是"还没找到答案的元素"。新来一个更大的数，它就同时是一批栈内元素的答案。

### 单调队列：滑动窗口最大值

```python-run
from collections import deque

def max_sliding_window(nums: list[int], k: int) -> list[int]:
    """返回每个大小为 k 的窗口中的最大值。单调队列 O(n)"""
    result = []
    dq = deque()                         # 存下标，队首=当前窗口最大值

    for i, v in enumerate(nums):
        # 移除滑出窗口的元素
        if dq and dq[0] <= i - k:
            dq.popleft()
        # 维护单调递减：移除所有比新元素小的
        while dq and nums[dq[-1]] < v:
            dq.pop()
        dq.append(i)
        # 窗口形成后，队首就是最大值
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result

print(max_sliding_window([1, 3, -1, -3, 5, 3, 6, 7], 3))
# [3, 3, 5, 5, 6, 7]
```

## 1.4 双端队列 `deque` 深度

`collections.deque` 是 Python 的专职双端队列——双向链表实现，两端操作均为 O(1)。

```python-run
from collections import deque

dq = deque([1, 2, 3])

# 两端操作，全部 O(1)
dq.append(4)       # 右边加 → [1, 2, 3, 4]
dq.appendleft(0)   # 左边加 → [0, 1, 2, 3, 4]
right = dq.pop()   # 右边弹 → 4
left = dq.popleft()# 左边弹 → 0

print(f"dq: {list(dq)}, left={left}, right={right}")

# maxlen 参数：自动移除旧元素（天然滑动窗口）
dq2 = deque(maxlen=3)
for v in [1, 2, 3, 4, 5]:
    dq2.append(v)
    print(f"加入 {v}: {list(dq2)}")
```

| 方法 | 复杂度 | 说明 |
|------|--------|------|
| `append(x)` | O(1) | 右侧插入 |
| `appendleft(x)` | O(1) | 左侧插入 |
| `pop()` | O(1) | 右侧删除 |
| `popleft()` | O(1) | 左侧删除 |
| `d[i]` | O(1) | 随机访问 |
| `rotate(n)` | O(k) | 循环移动 k 步 |

## 1.5 `queue` 模块：线程安全的队列

`queue.Queue` 不是给算法用的——它是**线程间通信**的工具，内置了锁和条件变量。

```python
from queue import Queue, LifoQueue, PriorityQueue

# 普通队列（线程安全 FIFO）
q = Queue(maxsize=3)

# 栈形式（线程安全 LIFO）
stack = LifoQueue()

# 优先级队列（线程安全）
pq = PriorityQueue()
pq.put((2, "次要任务"))
pq.put((1, "优先任务"))
```

> 📖 **官方文档**：[queue — A synchronized queue class](https://docs.python.org/3/library/queue.html)

## 本章小结

1. **栈 = LIFO**：`list.append` + `list.pop` 就是栈，天然 O(1)
2. **队列 = FIFO**：用 `deque` 而非 `list.pop(0)`——O(1) vs O(n)
3. **双端队列**：`deque` 两端操作 O(1)，`maxlen` 参数天然做滑动窗口
4. **单调栈/单调队列** 是面试高频考点：都是 O(n) 解决"下一个更大/最大值"问题
5. `queue.Queue` 是**线程安全**的队列，不是算法用的——别在单线程算法里用它
