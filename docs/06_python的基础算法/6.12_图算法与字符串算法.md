# 6.12 图算法与字符串算法

本章聚焦图算法的三个核心篇章——Dijkstra/Bellman-Ford 最短路径、拓扑排序的两种实现，以及字符串领域的 KMP、Trie、并查集。这些算法是对 6.7 章遍历能力的升华：有了 BFS/DFS，你能"走遍"图；学完本章，你能"高效地解决问题"。

## 1.1 Dijkstra 最短路径

### 1.1.1 问题是什么

给定**加权有向图**（边权非负），求从起点到所有其他节点的最短距离。地图导航、网络路由、物流规划都归结为这个问题。

### 1.1.2 为什么 BFS 不行

BFS 按"经过的边数"计算距离——它假设每条边的权重都是 1。在加权图中，经过更多边但权重更小的路径才是最短的：

```
BFS 看到的:          实际最短路径:
  A → B (10)           A → C (2) → B (3) = 5
```
BFS 先到 B 就认为距离是 10，但实际上绕经 C 只需 5。

**Dijkstra 的洞察**：不再按"边数"探索，而是按"当前已知距离"从小到大探索——每次取出距离最近的节点，更新它的邻居。

### 1.1.3 手工推演

以这个图为例，求 A 到所有节点的最短距离：

```
     A ──4──> B
     |\      /
    2| \1   /5
     |  \  /
     v   vv
     C ──8──> D ──2──> E
           \         /
            10───────
```

**初始状态**：`dist = {A:0, B:∞, C:∞, D:∞, E:∞}`，`heap = [(0, A)]`

```
第 1 轮 — 弹出 (0, A):
  检查邻居 B(4): ∞ > 0+4 → dist[B]=4, heap=[(4,B)]
  检查邻居 C(2): ∞ > 0+2 → dist[C]=2, heap=[(2,C), (4,B)]

第 2 轮 — 弹出 (2, C):
  C 的邻居 D(8): ∞ > 2+8 → dist[D]=10, heap=[(4,B), (10,D)]
  C 的邻居 E(10): ∞ > 2+10 → dist[E]=12, heap=[(4,B), (10,D), (12,E)]

第 3 轮 — 弹出 (4, B):
  B 的邻居 C(1): 2 < 4+1 → 不更新
  B 的邻居 D(5): 10 > 4+5 → dist[D]=9, heap=[(9,D), (10,D), (12,E)]
                                ↑ 新插入，旧的 (10,D) 成为"过时条目"

第 4 轮 — 弹出 (9, D):
  D 的邻居 E(2): 12 > 9+2 → dist[E]=11, heap=[(10,D), (11,E), (12,E)]

第 5 轮 — 弹出 (10, D):  10 > dist[D]=9 → 跳过（过时条目）

第 6 轮 — 弹出 (11, E):  无未处理邻居 → heap 为空 → 结束
```

**最终结果**：`dist = {A:0, B:4, C:2, D:9, E:11}`

### 1.1.4 代码实现

```python-run
import heapq

def dijkstra(graph, start):
    """
    返回 (dist, prev)
    dist:  起点到各节点的最短距离
    prev:  每个节点的前驱，用于回溯路径
    graph: {node: [(neighbor, weight), ...]}
    """
    dist = {start: 0}
    heap = [(0, start)]          # (到起点的距离, 节点)
    prev = {}

    while heap:
        d, u = heapq.heappop(heap)     # 取出当前已知最近的节点

        # 关键：跳过过时条目
        # 如果堆中记录的距离 > 当前已知最短距离，说明这是之前留下的旧记录
        if d > dist.get(u, float('inf')):
            continue

        for v, w in graph[u]:
            new_dist = d + w
            if new_dist < dist.get(v, float('inf')):
                dist[v] = new_dist
                prev[v] = u
                heapq.heappush(heap, (new_dist, v))  # 不修改旧条目，直接插入新条目
    return dist, prev

def get_path(prev, target):
    """回溯构造最短路径"""
    path = []
    cur = target
    while cur in prev:
        path.append(cur)
        cur = prev[cur]
    path.append(cur)
    return path[::-1]


# 测试图（和手工推演同一个图）
g = {
    'A': [('B', 4), ('C', 2)],
    'B': [('C', 1), ('D', 5)],
    'C': [('D', 8), ('E', 10)],
    'D': [('E', 2)],
    'E': [],
}

dist, prev = dijkstra(g, 'A')
print(f"最短距离: {dist}")
for target in ['B', 'C', 'D', 'E']:
    path = get_path(prev, target)
    print(f"A→{target} (距离 {dist[target]}): {' → '.join(path)}")
```

### 1.1.5 三个关键细节

**过时条目（stale entry）**：`if d > dist.get(u, float('inf')): continue` 是必要的。当发现更短路径到达某个已在堆中的节点时，我们不修改旧条目（堆不支持随机访问），而是插入新条目。旧条目留在堆里，弹出时通过这个判断跳过。

**为什么不用 BFS**：BFS 的队列按"何时入队"排序，Dijkstra 的堆按"距离远近"排序——差这一个数据结构，能力完全不同。

**复杂度**：O((V + E) log V)。每个节点最多被 push E 次（每条边触发一次松弛），堆大小最多 O(E)。

> Dijkstra 要求**边权非负**。反例：A→B(-5)→C, A→C(2)。Dijkstra 先确定 A→C=2，但实际 A→B→C=-5+? 需要绕路。有负权边用下一节的 Bellman-Ford。

## 1.2 Bellman-Ford 算法

### 1.2.1 负权边的问题

Dijkstra 的核心假设是"一旦确定某个节点的最短距离，就不会再被更新"。这个假设在负权边面前不成立——后面经过负权边可能"回头"缩短前面已确定节点的距离。

```
  A ──4──> B
   \       /
    \3   /-2
     \  /
      C

Dijkstra: A→B(4) "确定" → A→C→B(3+(-2)=1) 更短！但 B 已出堆，不会更新
```

### 1.2.2 核心思想

Bellman-Ford 放弃了"每次确定一个节点"的策略，改为**对所有边反复松弛 V-1 轮**。V-1 轮后如果还能松弛，说明存在负环。

```python-run
def bellman_ford(edges, V, start):
    """
    edges: [(u, v, weight), ...]  边列表
    V:     节点数（节点用 0..V-1 编号）
    start: 起点编号
    返回 (dist, prev) 或 报告负环
    """
    INF = float('inf')
    dist = [INF] * V
    dist[start] = 0
    prev = [-1] * V

    # 松弛 V-1 轮
    for _ in range(V - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != INF and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                prev[v] = u
                updated = True
        if not updated:       # 提前终止：没有更新 → 已经稳定
            break

    # 第 V 轮：检测负环
    for u, v, w in edges:
        if dist[u] != INF and dist[u] + w < dist[v]:
            return None, None   # 存在负权环

    return dist, prev


# 测试：含负权边的图（无负环）
#   0 ──4──> 1
#    \       /
#     \3   /-2
#      \  /
#       2
edges = [(0, 1, 4), (0, 2, 3), (2, 1, -2)]
dist, prev = bellman_ford(edges, 3, 0)
print(f"最短距离: {dist}")   # [0, 1, 3]  — A→C→B = 3+(-2)=1 比直接 A→B=4 更短
print(f"0→1: {dist[1]} (Dijkstra 会错误地输出 4)")
```

### 1.2.3 Dijkstra vs Bellman-Ford

| 维度 | Dijkstra | Bellman-Ford |
|------|----------|-------------|
| 数据结构 | 最小堆 | 边列表 |
| 复杂度 | O((V+E) log V) | O(VE) |
| 负权边 | ❌ 不支持 | ✅ 支持 |
| 负环检测 | ❌ | ✅ |
| 适用场景 | 边权非负的日常场景 | 金融套利、差分约束系统 |

> 全源最短路径用 **Floyd-Warshall**：O(V³)，基于 DP，五行代码。适用于 V 较小（< 500）的稠密图。

## 1.3 拓扑排序的两种实现

DAG 才有拓扑序——节点的一种线性排列，使得所有边都"从左指向右"。这恰好是 6.7 中介绍的"入度/出度"概念的直接应用。

### 1.3.1 Kahn 算法（BFS + 入度表）

**思路**：反复找"没人依赖"的节点，移除它，更新剩余节点的入度。

```python-run
from collections import deque

def kahn(graph):
    """Kahn 算法：BFS + 入度表 — 天然能检测环"""
    # 入度计算（6.7 讲过的"被射中加分"思维）
    indegree = {node: 0 for node in graph}
    for u in graph:
        for v in graph[u]:
            indegree[v] = indegree.get(v, 0) + 1

    # 入度为 0 → 入队（没人依赖，可以先做）
    q = deque([n for n in indegree if indegree[n] == 0])
    order = []

    while q:
        u = q.popleft()
        order.append(u)
        for v in graph[u]:
            indegree[v] -= 1        # "移除"u → v 的入度减少
            if indegree[v] == 0:
                q.append(v)

    return order if len(order) == len(indegree) else None


dag = {'A': ['B', 'C'], 'B': ['D'], 'C': ['D'], 'D': ['E'], 'E': []}
print(f"Kahn 拓扑序: {' → '.join(kahn(dag))}")   # A → B → C → D → E
```

### 1.3.2 DFS 后序逆序（回顾 6.7）

6.7 的 1.6.3 节讲过：**DFS 的逆后序（Reverse Postorder）= DAG 的拓扑排序**。原理是 DFS 完成后序意味着"所有后继都处理完了才记录自己"——逆过来就是"依赖先于被依赖"。

```python-run
def dfs_topo(graph):
    """DFS 逆后序实现拓扑排序——三色标记法判环"""
    WHITE, GRAY, BLACK = 0, 1, 2
    color = {node: WHITE for node in graph}
    order = []

    def dfs(node):
        color[node] = GRAY
        for neighbor in graph[node]:
            if color[neighbor] == GRAY:
                raise ValueError("图中存在环，无法拓扑排序")
            if color[neighbor] == WHITE:
                dfs(neighbor)
        color[node] = BLACK
        order.append(node)          # 后序

    for node in graph:
        if color[node] == WHITE:
            dfs(node)
    return order[::-1]              # 逆后序


dag = {'A': ['B', 'C'], 'B': ['D'], 'C': ['D'], 'D': ['E'], 'E': []}
print(f"DFS  拓扑序: {' → '.join(dfs_topo(dag))}")
```

### 1.3.3 两种实现对比

| 维度 | Kahn（BFS + 入度表） | DFS 逆后序 |
|------|---------------------|-----------|
| 核心数据结构 | 队列 + 入度数组 | 递归栈 + 三色标记 |
| 判环方式 | 结果长度 < 节点数 | 遇到灰色节点 = 环 |
| 顺序特点 | 同层节点顺序不保证 | 取决于 DFS 遍历顺序 |
| 适合场景 | 需要流式处理（逐个输出） | 需要自然的递归表达 |
| 能否中断 | ✅ 可以随时停止取结果 | ❌ 必须递归到底 |

## 1.4 KMP 字符串匹配

### 1.4.1 朴素匹配为什么慢

```python-run
# 朴素匹配的回退代价
text =    "ABABABABC"
pattern = "ABABC"

# 匹配过程：
# ABABA...  在 text[4]='A' ≠ pattern[4]='C' 失配
# 朴素算法：回退到 text[1]，pattern 归零从头开始
# ✗ 已匹配的 "ABAB" 信息全部浪费
```

**KMP 的关键问题**：失配时，已匹配的部分（"ABAB"）中，最长相同前后缀是什么？答案是 "AB"（长度为 2）。这意味着 pattern 的前 2 个字符 "AB" 和刚刚匹配的 text 后缀 "AB" 是一样的——**不需要回退重比**，直接让 pattern 从索引 2 继续匹配。

### 1.4.2 LPS 表的手工构建

LPS[i] = pattern[0..i] 这个前缀中，最长相同前后缀的长度。

```
pattern = "ABABCABAB"

i=0: "A"        → 无相同前后缀 → lps[0]=0
i=1: "AB"       → 无相同前后缀 → lps[1]=0
i=2: "ABA"      → 前缀"A"=后缀"A" → lps[2]=1
i=3: "ABAB"     → 前缀"AB"=后缀"AB" → lps[3]=2
i=4: "ABABC"    → 无 → lps[4]=0
i=5: "ABABCA"   → 前缀"A"=后缀"A" → lps[5]=1
i=6: "ABABCAB"  → 前缀"AB"=后缀"AB" → lps[6]=2
i=7: "ABABCABA" → 前缀"ABA"=后缀"ABA" → lps[7]=3
i=8: "ABABCABAB"→ 前缀"ABAB"=后缀"ABAB" → lps[8]=4

LPS = [0, 0, 1, 2, 0, 1, 2, 3, 4]
```

### 1.4.3 代码实现

```python-run
def build_lps(pattern):
    """构建 LPS 表：线性时间，核心在于回退机制"""
    lps = [0] * len(pattern)
    length = 0                     # 当前匹配的"最长前后缀长度"
    i = 1
    while i < len(pattern):
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length        # 前后缀延长 1 位
            i += 1
        elif length > 0:
            length = lps[length - 1]  # 回退到上一个可能匹配的位置
        else:
            lps[i] = 0                # 完全没有匹配
            i += 1
    return lps

def kmp(text, pattern):
    """O(n+m) 字符串匹配"""
    if not pattern:
        return []
    lps = build_lps(pattern)
    matches = []
    i = j = 0                        # i→text, j→pattern

    while i < len(text):
        if text[i] == pattern[j]:
            i += 1; j += 1
            if j == len(pattern):    # 完全匹配
                matches.append(i - j)
                j = lps[j - 1]       # 继续搜索下一个匹配
        elif j > 0:
            j = lps[j - 1]           # ★ 关键：跳过失配但不回退 i
        else:
            i += 1
    return matches

text = "ABABDABACDABABCABAB"
pattern = "ABABCABAB"
print(f"LPS 表:  {build_lps(pattern)}")
print(f"匹配位:  {kmp(text, pattern)}")
```

> Python `str.find()` 内部使用 Fast Search（Boyer-Moore 简化变体），实际中通常比 KMP 更快。KMP 的价值在于**确定性 O(n)**——无论输入是什么，字符比较次数严格受控。

## 1.5 Trie 前缀树

### 1.5.1 为什么需要 Trie

哈希表能做精确查找，但做不了**前缀匹配**。"找出所有以 'ca' 开头的词"——哈希表只能枚举所有 key。Trie 沿字符路径直接定位到前缀末端，然后收集所有子孙节点。

### 1.5.2 基础实现

```python-run
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self.root
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_end

    def starts_with(self, prefix: str) -> bool:
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True

    def autocomplete(self, prefix: str):
        """自动补全：返回所有以 prefix 开头的词"""
        # 走到前缀末端
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return []
            node = node.children[ch]

        # 收集所有子孙单词
        result = []
        def collect(node, current):
            if node.is_end:
                result.append(current)
            for ch, child in node.children.items():
                collect(child, current + ch)

        collect(node, prefix)
        return result


trie = Trie()
for w in ["cat", "car", "care", "cart", "dog", "door", "doll"]:
    trie.insert(w)

print(f"search('cat'):  {trie.search('cat')}")
print(f"search('ca'):   {trie.search('ca')}")
print(f"ca* 补全:       {trie.autocomplete('ca')}")
print(f"do* 补全:       {trie.autocomplete('do')}")
```

### 1.5.3 Trie 的工程价值

| 场景 | 用法 |
|------|------|
| 搜索框自动补全 | `autocomplete()` |
| 拼写检查 | 查找编辑距离 < 2 的所有词 |
| IP 路由最长前缀匹配 | 按位走 Trie，找最后匹配的 `is_end` 节点 |
| 词频统计 | 节点存计数，取代简单 `dict` 做前缀统计 |

> Trie 的空间瓶颈：每个节点一个 `dict` 开销不小。工业界用 **Radix Tree**（路径压缩，合并单孩子链）和 **DAWG**（共享同构子树）来优化。

## 1.6 并查集 Union-Find

### 1.6.1 解决什么问题

动态判断两个元素是否在同一集合中——支持**合并**（union）和**查询**（find）两种操作。Kruskal 最小生成树（6.7 1.7.4）判断"加入这条边会不会构成环"的核心就是并查集。

### 1.6.2 两个优化

- **路径压缩**：`find` 时把路径上所有节点直接挂到根节点——下次查询直接 O(1)
- **按秩合并**：`union` 时把小树挂到大树下——防止退化成链

两者结合，均摊复杂度 ≈ **O(α(n))**，其中 α 是反阿克曼函数，实际中 α(n) ≤ 4。

```python-run
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        """找根 + 路径压缩"""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        """合并 + 按秩"""
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1
        return True

    def connected(self, x, y):
        return self.find(x) == self.find(y)


# 示例：无向图判环
# 遍历每条边，如果两端已在同一集合 → 加这条边会形成环
def has_cycle_uf(n, edges):
    uf = UnionFind(n)
    for u, v in edges:
        if not uf.union(u, v):     # 已在同一组 → 环
            return True
    return False

tree_edges = [(0, 1), (1, 2), (2, 3)]        # 无环（一棵树）
cycle_edges = [(0, 1), (1, 2), (2, 3), (0, 3)]  # 有环

print(f"树边有环:   {has_cycle_uf(4, tree_edges)}")    # False
print(f"环边有环:   {has_cycle_uf(4, cycle_edges)}")   # True
```

### 1.6.3 并查集 vs DFS 判环

| 维度 | 并查集 | DFS |
|------|--------|-----|
| 场景 | 边**逐步添加**（动态） | 图已构建好（静态） |
| 复杂度 | ≈ O(1) / 条边 | O(V+E) 一次性 |
| 只能判环？ | 还能回答"两点是否连通" | 能同时得到路径 |

> 6.7 的 Kruskal 算法就是并查集的经典应用——按权重升序遍历边，`union` 成功就加入 MST，失败（已在同组）就跳过。

## 本章小结

1. **Dijkstra = BFS 把队列换成堆**。核心差别：BFS 按"边数"排序，Dijkstra 按"距离"排序。过时条目的 `if d > dist[u]: continue` 是关键。
2. **Bellman-Ford 对所有边松弛 V-1 轮**，O(VE)，能处理负权边并检测负环。Dijkstra 更快但不能有负权。
3. **拓扑排序两种实现互补**：Kahn（BFS）适合流式处理、天然判环；DFS 逆后序（6.7 就讲过）代码更简洁，三色标记判环。
4. **KMP 的 LPS 表本质是"失配时跳过多少不用重比"**——利用已匹配部分的前后缀信息，达到确定性的 O(n+m)。
5. **Trie = 按字符路径组织的树**，前缀匹配的终极工具。加 `autocomplete` 方法就是搜索框的底层引擎。
6. **并查集 ≈ O(1) 判连通**，路径压缩 + 按秩合并是 Kruskal 算法的基石（6.7 已实战）。
