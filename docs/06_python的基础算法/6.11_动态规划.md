# 6.11 动态规划

动态规划（DP）是算法中最强大的思想之一。它的本质：**把大问题分解成重叠子问题，用表格记住中间结果，避免重复计算**。Python 的 `@lru_cache` 让 DP 的自顶向下写法变得极其优雅。

> 📖 **官方文档**：[functools.lru_cache](https://docs.python.org/3/library/functools.html#functools.lru_cache)

---

## 1.1 DP 的核心思想：从斐波那契说起

```python-run
from functools import lru_cache

# 版本 1：朴素递归 — O(2^n)
def fib_naive(n):
    return n if n <= 1 else fib_naive(n - 1) + fib_naive(n - 2)

# 版本 2：记忆化递归 — O(n)，优雅且直观
@lru_cache(maxsize=None)
def fib_memo(n):
    return n if n <= 1 else fib_memo(n - 1) + fib_memo(n - 2)

# 版本 3：自底向上 DP — O(n)，O(1) 空间
def fib_dp(n):
    if n <= 1:
        return n
    prev, curr = 0, 1
    for _ in range(2, n + 1):
        prev, curr = curr, prev + curr
    return curr

print(f"fib_memo(30) = {fib_memo(30)}")
print(f"fib_dp(30) = {fib_dp(30)}")
```

**DP 的两个方向**：

| 方向 | 描述 | Python 实现 |
|------|------|------------|
| **自顶向下**（记忆化） | 从大问题出发，递归分解 | `@lru_cache` 装饰器 |
| **自底向上**（建表） | 从最小子问题开始，逐步构建 | `for` 循环填表 |

> 记忆化 = 递归 + 缓存。用了 `@lru_cache` 就等于在写 DP，只是方向不同。

---

## 1.2 DP 的四个步骤

1. **定义状态**：`dp[i]` 表示什么？
2. **推导转移方程**：`dp[i]` 和之前的状态有什么关系？
3. **确定基案**：最小的 `dp[0]` 或 `dp[1]` 是多少？
4. **填表顺序**：从小到大还是从大到小？

## 1.3 经典 DP 问题

### 跳台阶（爬楼梯）

```python-run
# 一次 1 步或 2 步，n 级台阶有几种走法？
def climb_stairs(n):
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2
    for i in range(3, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

print(f"10 级台阶: {climb_stairs(10)} 种走法")   # 89
```

### 0-1 背包

```python-run
def knapsack(weights, values, capacity):
    """每个物品只能选或不选一次（0-1 背包）"""
    n = len(weights)
    # dp[i][w] = 前 i 个物品，容量 w 下的最大价值
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        w, v = weights[i - 1], values[i - 1]
        for c in range(capacity + 1):
            if c < w:
                dp[i][c] = dp[i - 1][c]          # 装不下
            else:
                dp[i][c] = max(dp[i - 1][c],      # 不选
                               dp[i - 1][c - w] + v)  # 选
    return dp[n][capacity]

weights = [2, 3, 4, 5]
values  = [3, 4, 5, 6]
print(f"背包最大价值: {knapsack(weights, values, 8)}")  # 10
```

### 最长公共子序列（LCS）

```python-run
def lcs(s1, s2):
    """两个字符串的最长公共子序列长度"""
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i - 1] == s2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]

print(f"LCS('abcde', 'ace'): {lcs('abcde', 'ace')}")      # 3
print(f"LCS('python', 'java'): {lcs('python', 'java')}")   # 1
```

## 1.4 DP 空间优化技巧

```python-run
# 背包问题空间优化：二维 → 一维（滚动数组）
def knapsack_1d(weights, values, capacity):
    dp = [0] * (capacity + 1)
    for w, v in zip(weights, values):
        # 关键：倒序遍历，保证 dp[c-w] 是上一轮的
        for c in range(capacity, w - 1, -1):
            dp[c] = max(dp[c], dp[c - w] + v)
    return dp[capacity]

print(f"背包最大价值（1D优化）: {knapsack_1d([2,3,4,5], [3,4,5,6], 8)}")
```

> 倒序的原因：正序会让同一个物品被选多次（变成完全背包），倒序保证每轮只用上一轮的状态。

## 1.5 什么时候能用 DP

DP 不是万能药。问题必须满足：

| 条件 | 含义 |
|------|------|
| **最优子结构** | 最优解包含子问题的最优解 |
| **重叠子问题** | 不同的递归分支反复遇到相同的子问题 |
| **无后效性** | 当前状态确定后，之后的决策不影响"如何到达当前状态" |

```python-run
# 反例：贪心和 DP 的分界线
# 0-1 背包 → DP（物品不可拆分，有最优子结构）
# 分数背包 → 贪心（物品可拆分，贪心选性价比最高的）
```

## 本章小结

1. **DP = 重叠子问题 + 最优子结构**。避免重复计算是目的，状态转移方程是核心
2. **自顶向下（`@lru_cache`）写起来最快**；**自底向上（建表）更省空间，且能优化成滚动数组**
3. **0-1 背包的倒序遍历**是关键——正序 = 完全背包（无限次选），倒序 = 0-1 背包（只选一次）
4. **不是所有递归都适合加缓存就是 DP**——必须有重叠子问题才有效果
5. DP 的难点不在理解概念，**而是从问题中提炼出 dp[i] 的含义和转移方程**
