# 6.7 图结构

图（Graph）是计算机科学中最通用、最强大的数据结构——**万物皆可图**。社交网络中人是节点、好友关系是边；地图上路是边、路口是节点；编译器用图表示数据流；推荐系统用图发现兴趣关联。理解图，就理解了复杂关系的建模方式。

Python 没有内置图类型，但 `dict` + `set`/`list` 的组合让图的表达极其自然。本章从基础概念出发，逐步深入到 BFS/DFS 变体、连通性分析，再到 Kosaraju、Prim、Kruskal 等进阶算法。

> 📖 **官方文档**：[collections — defaultdict](https://docs.python.org/3/library/collections.html) · [Graph Theory (Wikipedia)](https://en.wikipedia.org/wiki/Graph_theory)

---

## 1.1 图是什么：从现实到抽象

数学上，图由两个集合定义：**G = (V, E)**。V（Vertex）是节点集合，E（Edge）是边集合——每条边连接两个节点。

现实世界中的图无处不在：

| 场景 | 节点 | 边 | 方向 |
|------|------|----|------|
| 社交网络 | 用户 | 好友/关注关系 | 好友无向，关注有向 |
| 地图导航 | 路口/地点 | 道路 | 单行道有向，双行道无向 |
| 依赖管理 | 包/模块 | 依赖关系 | 有向（A 依赖 B） |
| Web 链接 | 网页 | 超链接 | 有向 |
| 编译器 IR | 基本块 | 控制流 | 有向 |
| 电路板 | 元件 | 导线 | 无向 |

:::tip
**图 = 节点 + 边**。所有复杂关系建模最终都归结为这两个要素的组合方式——有向还是无向、加权还是无权。
:::

## 1.2 图的分类体系

### 1.2.1 按边的方向

```python-run
# 无向图：边是双向的——"我们是好友"（互为好友）
undirected = {
    'A': {'B', 'C'},
    'B': {'A', 'D'},
    'C': {'A', 'D'},
    'D': {'B', 'C'},
}

# 有向图：边有方向——"我关注了你"（你不一定关注我）
directed = {
    'A': ['B', 'C'],      # A → B, A → C
    'B': ['D'],           # B → D
    'C': ['D'],           # C → D
    'D': [],              # D 没有出边
}

print(f"无向图 A 的邻居: {undirected['A']}")
print(f"有向图 A 的出边: {directed['A']}")
print(f"有向图 D 的出边: {directed['D']}")      # [] — D 没有指向任何节点
```

**无向图的边等价于两条有向边**。在 Python 中，无向图添加边 `(u, v)` 时需要同时 `graph[u].add(v)` 和 `graph[v].add(u)`。

### 1.2.2 按边的权重

```python-run
# 无权图：边只有"存在"或"不存在"——好友关系
unweighted = {
    'A': {'B', 'C'},
    'B': {'A', 'D'},
    'C': {'A', 'D'},
    'D': {'B', 'C'},
}

# 加权图：边有数值权重——地图上的距离、网络中的带宽
# 邻接表存 (邻居, 权重) 元组
weighted = {
    'A': [('B', 4), ('C', 2)],
    'B': [('A', 4), ('D', 5)],
    'C': [('A', 2), ('D', 8)],
    'D': [('B', 5), ('C', 8)],
}

print(f"加权图 A→B 的距离: {dict(weighted)['A'][0]}")
```

> 无权图可视为所有边权重 = 1 的加权图。BFS 找无权图最短路径，Dijkstra 找加权图最短路径（见第 6.12 章）。

### 1.2.3 稀疏图 vs 稠密图

这个区分直接影响算法选择——**复杂度里 E（边数）究竟是 ≈ V 还是 ≈ V²，结论完全不同**。

```python-run
# 稀疏图：边数接近节点数，E ≈ V
# 典型：社交网络（大多数人不认识对方）、道路网（每个路口只连几条路）
sparse = {
    'A': {'B'},
    'B': {'A', 'C'},
    'C': {'B', 'D'},
    'D': {'C'},
}  # 4 节点，3 条边（无向）

# 稠密图：边数接近 V²，E ≈ V²
# 典型：小型完全图、锦标赛图
dense = {
    'A': {'B', 'C', 'D'},
    'B': {'A', 'C', 'D'},
    'C': {'A', 'B', 'D'},
    'D': {'A', 'B', 'C'},
}  # 4 节点，6 条边 → 接近 4²/2 = 8
```

| 类型 | 边数范围 | 邻接表遍历邻居 | 邻接矩阵遍历邻居 |
|------|----------|---------------|-----------------|
| 稀疏图 | E ≈ O(V) | O(1) 平均 | O(V) — 很差 |
| 稠密图 | E ≈ O(V²) | O(V) 平均 | O(V) — 可以接受 |

### 1.2.4 图的术语速查

```
         A ──── B
        / \     / \
       /   \   /   \
      C ──── D       E ──── F
      
    连通图              孤立的连通分量
    (A-B-C-D 互通)     (E-F 互通，但与左边不连通)
```

```python-run
# 度（Degree）：一个节点连接的边数
graph = {
    'A': {'B', 'C', 'D'},   # A 的度 = 3
    'B': {'A'},             # B 的度 = 1
    'C': {'A'},             # C 的度 = 1
    'D': {'A'},             # D 的度 = 1
}

# 有向图中度分为入度（in-degree）和出度（out-degree）
digraph = {
    'A': ['B', 'C', 'D'],   # A 出度 = 3，入度 = 0
    'B': [],                # B 出度 = 0，入度 = 1（来自 A）
    'C': ['B'],             # C 出度 = 1（指向 B），入度 = 1（来自 A）
    'D': [],                # D 出度 = 0，入度 = 1（来自 A）
}

def compute_degrees(dg):
    indegree = {node: 0 for node in dg}
    outdegree = {node: len(dg[node]) for node in dg}
    for u in dg:
        for v in dg[u]:
            indegree[v] += 1
    return indegree, outdegree

indeg, outdeg = compute_degrees(digraph)
print(f"出度: {outdeg}")
print(f"入度: {indeg}")
```

**核心概念一览**：

| 概念 | 英文 | 含义 |
|------|------|------|
| **节点/顶点** | Vertex/Node | 图中的实体 |
| **边** | Edge | 两个节点之间的连接 |
| **度** | Degree | 无向图中该节点的边数；有向图分入度/出度 |
| **路径** | Path | 从一个节点到另一个节点经过的边的序列 |
| **简单路径** | Simple Path | 不重复经过节点的路径 |
| **环** | Cycle | 首尾相接的路径（起点 = 终点） |
| **连通图** | Connected Graph | 任意两点间存在路径 |
| **连通分量** | Connected Component | 极大连通子图 |
| **DAG** | Directed Acyclic Graph | 有向无环图——拓扑排序的前提 |
| **完全图** | Complete Graph | 每对节点之间都有边，n 个节点有 n(n-1)/2 条边 |
| **二部图** | Bipartite Graph | 节点可分成两组，所有边都在两组之间 |
| **树** | Tree | 连通且无环的无向图（n 个节点 = n-1 条边） |

## 1.3 图的三种 Python 表达

### 1.3.1 邻接表——Python 首选

**邻接表 = `dict[node, set|list[neighbor]]`**。每个节点存一个邻居集合或列表。遍历某节点的邻居是 O(度)，空间 O(V+E)。

```python-run
# 无向无权图 — 邻接表用 set（快速查重、O(1) 判断邻居）
graph = {
    'A': {'B', 'C'},
    'B': {'A', 'D', 'E'},
    'C': {'A', 'F'},
    'D': {'B'},
    'E': {'B', 'F'},
    'F': {'C', 'E'},
}

# 判断 A 和 C 是否直接相连
print(f"A 和 C 相邻: {'C' in graph['A']}")   # O(1) 平均
print(f"A 和 F 相邻: {'F' in graph['A']}")   # O(1) 平均

# 遍历某个节点的所有邻居
print(f"B 的邻居: {graph['B']}")
```

```python-run
# 加权有向图 — 邻接表存 (邻居, 权重) 列表
wgraph = {
    'A': [('B', 4), ('C', 2)],
    'B': [('D', 5)],
    'C': [('D', 8), ('E', 10)],
    'D': [('E', 2)],
    'E': [],
}

# 查询 A 到 B 的权重
for neighbor, weight in wgraph['A']:
    if neighbor == 'B':
        print(f"A→B 权重: {weight}")
```

### 1.3.2 邻接矩阵——适合稠密图

**邻接矩阵 = V×V 的二维矩阵**。`matrix[i][j]` 表示从节点 i 到节点 j 的边（无权时用 0/1，加权时用权重值或 ∞）。

```python-run
# 邻接矩阵实现无权有向图
nodes = ['A', 'B', 'C', 'D']
node_idx = {name: i for i, name in enumerate(nodes)}   # 名字→索引映射
N = len(nodes)
adj_matrix = [[0] * N for _ in range(N)]

# 添加边：A→B, A→C, B→D, C→D
edges = [('A', 'B'), ('A', 'C'), ('B', 'D'), ('C', 'D')]
for u, v in edges:
    i, j = node_idx[u], node_idx[v]
    adj_matrix[i][j] = 1    # 无权图用 1

print("邻接矩阵:")
for row, name in zip(adj_matrix, nodes):
    print(f"  {name}: {row}")

# 判断是否有边：O(1)
print(f"A→B 有边: {adj_matrix[node_idx['A']][node_idx['B']] == 1}")
print(f"B→A 有边: {adj_matrix[node_idx['B']][node_idx['A']] == 1}")
```

```python-run
# 加权图邻接矩阵 — 用 math.inf 表示无边
import math

nodes = ['A', 'B', 'C', 'D']
idx = {name: i for i, name in enumerate(nodes)}
N = len(nodes)
INF = math.inf
wmatrix = [[INF] * N for _ in range(N)]

# 对角线：自己到自己距离为 0
for i in range(N):
    wmatrix[i][i] = 0

# 加权边：A→B(4), A→C(2), B→D(5), C→D(1)
for u, v, w in [('A', 'B', 4), ('A', 'C', 2), ('B', 'D', 5), ('C', 'D', 1)]:
    wmatrix[idx[u]][idx[v]] = w

for row, name in zip(wmatrix, nodes):
    print(f"  {name}: {row}")
```

### 1.3.3 边列表——适合特定算法

**边列表 = 所有边组成的列表**。Kruskal 最小生成树、Bellman-Ford 算法都基于边列表操作。

```python-run
# 边列表：每条边是一个元组 (u, v) 或 (u, v, weight)
edges_unweighted = [
    ('A', 'B'), ('A', 'C'), ('B', 'D'),
    ('B', 'E'), ('C', 'F'), ('E', 'F'),
]

edges_weighted = [
    ('A', 'B', 4), ('A', 'C', 2),
    ('B', 'D', 5), ('C', 'D', 8),
    ('C', 'E', 10), ('D', 'E', 2),
]

print(f"边数: {len(edges_weighted)}")
print(f"所有边: {edges_weighted}")
```

### 1.3.4 三种表达对比

| 维度 | 邻接表 `dict[set]` | 邻接矩阵 `list[list]` | 边列表 `list[tuple]` |
|------|--------------------|-----------------------|---------------------|
| 空间 | **O(V+E)** | O(V²) | **O(E)** |
| 判断 u,v 是否相邻 | O(1) 平均 | **O(1)** | O(E) |
| 遍历节点 u 的邻居 | **O(度)** | O(V) | O(E) |
| 添加边 | O(1) | O(1) | O(1) |
| 删除边 | O(1) | O(1) | O(E) |
| 适合场景 | 大多数情况（默认选择） | 稠密图、需要 O(1) 查边 | Kruskal、Bellman-Ford |

**决策原则**：

```
边数 E 和节点数 V 的关系?
  ├─ E 较小 (稀疏图，如社交网络)
  │   └─ 邻接表 dict[set]  ← 绝大多数情况选这个
  ├─ E 接近 V² (稠密图，如小型锦标赛)
  │   └─ 邻接矩阵 list[list]     ← 频繁判断两节点是否相邻
  └─ 算法需要遍历所有边 (如 Kruskal)
      └─ 边列表 list[tuple]
```

## 1.4 构建图的 Python 惯用法

### 1.4.1 `defaultdict` 模式

```python-run
from collections import defaultdict

# 无向无权图：从边列表构建
def build_undirected(edges):
    graph = defaultdict(set)
    for u, v in edges:
        graph[u].add(v)
        graph[v].add(u)         # 无向图：双向添加
    return dict(graph)

edges = [('A', 'B'), ('A', 'C'), ('B', 'D'), ('B', 'E'), ('C', 'F')]
g = build_undirected(edges)
print(f"无向图: {g}")

# 有向加权图
def build_directed_weighted(edges):
    graph = defaultdict(list)
    for u, v, w in edges:
        graph[u].append((v, w))
    return dict(graph)

wedges = [('A', 'B', 4), ('A', 'C', 2), ('B', 'D', 5), ('C', 'D', 1)]
wg = build_directed_weighted(wedges)
print(f"有向加权图: {wg}")
```

### 1.4.2 邻接表 ↔ 邻接矩阵互转

```python-run
# 邻接矩阵 → 邻接表
def matrix_to_adjlist(matrix, nodes):
    adjlist = {node: [] for node in nodes}
    for i, u in enumerate(nodes):
        for j, v in enumerate(nodes):
            if matrix[i][j] != 0:
                adjlist[u].append(v)
    return adjlist

# 邻接表 → 邻接矩阵
def adjlist_to_matrix(adjlist, nodes):
    idx = {name: i for i, name in enumerate(nodes)}
    N = len(nodes)
    matrix = [[0] * N for _ in range(N)]
    for u in adjlist:
        for v in adjlist[u]:
            matrix[idx[u]][idx[v]] = 1
    return matrix

nodes = ['A', 'B', 'C', 'D']
adjlist = {'A': ['B', 'C'], 'B': ['D'], 'C': ['D'], 'D': []}
m = adjlist_to_matrix(adjlist, nodes)
print(f"邻接表: {adjlist}")
print(f"邻接矩阵: {[f'{n}: {row}' for n, row in zip(nodes, m)]}")
```

### 1.4.3 出入度统计与图的基本属性

```python-run
from collections import defaultdict

def graph_stats(graph, directed=False):
    """统计图的基本属性"""
    V = len(graph)
    # 统计边数
    E = sum(len(neighbors) for neighbors in graph.values())
    if not directed:
        E //= 2   # 无向图每条边算了两次

    # 度统计
    degrees = {node: len(neighbors) for node, neighbors in graph.items()}
    max_deg_node = max(degrees, key=degrees.get)
    min_deg_node = min(degrees, key=degrees.get)

    # 入度（仅对有向图有意义）
    if directed:
        indegree = defaultdict(int)
        for u in graph:
            for v in graph[u]:
                indegree[v] += 1
        # 找出度 = 0 的节点（汇点）和入度 = 0 的节点（源点）
        sources = [node for node in graph if indegree[node] == 0]
        sinks   = [node for node in indegree if len(graph.get(node, [])) == 0]

    print(f"节点数: {V}, 边数: {E}")
    print(f"最大度: {max_deg_node}(度={degrees[max_deg_node]})")
    print(f"最小度: {min_deg_node}(度={degrees[min_deg_node]})")
    if directed:
        print(f"入度为 0 的节点（源点）: {sources}")
        print(f"出度为 0 的节点（汇点）: {sinks}")

g = {'A': {'B', 'C'}, 'B': {'A', 'D', 'E'}, 'C': {'A', 'F'},
     'D': {'B'}, 'E': {'B', 'F'}, 'F': {'C', 'E'}}
graph_stats(g)

dg = {'A': ['B', 'C'], 'B': ['D'], 'C': ['D'], 'D': []}
graph_stats(dg, directed=True)
```

## 1.5 BFS 广度优先搜索深度解析

**BFS 核心：队列控制"先宽后深"**。从起点出发，先访问距离为 1 的所有节点，再访问距离为 2 的，依此类推——像水波扩散。

### 1.5.1 BFS 基础遍历

```python-run
from collections import deque

def bfs_traverse(graph, start):
    """基础 BFS 遍历，返回访问顺序"""
    queue = deque([start])
    visited = {start}
    order = []

    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return order

g = {'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'],
     'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E']}

print(f"BFS 遍历(A→): {' → '.join(bfs_traverse(g, 'A'))}")
```

**BFS 的 `visited` 在入队时标记**——不是出队时才标记。如果在出队时标记，同一个节点可能被重复入队多次，导致错误。

### 1.5.2 层级追踪——按距离分组

```python-run
from collections import deque

def bfs_levels(graph, start):
    """BFS 按层级分组返回，levels[d] = 距离为 d 的所有节点"""
    queue = deque([start])
    visited = {start}
    levels = []                     # levels[d] = [距离为 d 的节点]

    while queue:
        level_size = len(queue)     # 当前层的节点数
        current_level = []
        for _ in range(level_size):
            node = queue.popleft()
            current_level.append(node)
            for neighbor in graph[node]:
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)
        levels.append(current_level)
    return levels

g = {'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'],
     'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E']}

for d, nodes in enumerate(bfs_levels(g, 'A')):
    print(f"距离 {d}: {nodes}")
```

**`len(queue)` 是分层的关键**——每次循环处理完一层所有节点，再进行下一层。

### 1.5.3 无权图最短路径

```python-run
from collections import deque

def bfs_shortest_path(graph, start, target):
    """返回从 start 到 target 的最短路径"""
    queue = deque([[start]])           # 每条路径是一个 list
    visited = {start}

    while queue:
        path = queue.popleft()
        node = path[-1]

        if node == target:
            return path

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(path + [neighbor])
    return None                         # 不可达

g = {'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'],
     'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E']}

path = bfs_shortest_path(g, 'A', 'F')
print(f"最短路径 A→F: {' → '.join(path)}")
```

```python-run
# 更高效的做法：存前驱节点 + 最后回溯构造路径
from collections import deque

def bfs_shortest_path_v2(graph, start, target):
    """用 prev 字典记录前驱，最后回溯——避免存整条路径"""
    queue = deque([start])
    visited = {start}
    prev = {}

    while queue:
        node = queue.popleft()
        if node == target:
            # 回溯构造路径
            path = []
            cur = target
            while cur in prev:
                path.append(cur)
                cur = prev[cur]
            path.append(start)
            return path[::-1]

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                prev[neighbor] = node
                queue.append(neighbor)
    return None

g = {'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'],
     'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E']}
print(f"最短路径: {' → '.join(bfs_shortest_path_v2(g, 'A', 'F'))}")
```

**为什么 BFS 找到的一定是最短路径？** 队列按距离分层，第一次到达目标节点时走过的层数 = 最短距离。在无权图中，每层的距离恰好是边的条数。

### 1.5.4 连通分量

```python-run
from collections import deque

def connected_components(graph):
    """找出无向图的所有连通分量"""
    all_nodes = set(graph.keys())
    visited = set()
    components = []

    def bfs_component(start):
        queue = deque([start])
        visited.add(start)
        comp = []
        while queue:
            node = queue.popleft()
            comp.append(node)
            for neighbor in graph[node]:
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)
        return comp

    for node in all_nodes:
        if node not in visited:
            components.append(bfs_component(node))
    return components

# 两个互不连通的子图
g = {
    'A': {'B'}, 'B': {'A', 'C'}, 'C': {'B'},
    'D': {'E'}, 'E': {'D', 'F'}, 'F': {'E'},
    'G': set(),
}

comps = connected_components(g)
print(f"连通分量数: {len(comps)}")
for i, comp in enumerate(comps):
    print(f"  分量 {i + 1}: {comp}")
```

> `G` 是孤立节点——自身构成一个连通分量。`len(connected_components(graph))` 直接给出图的"片段数"。

## 1.6 DFS 深度优先搜索深度解析

**DFS 核心：栈（递归就是隐式栈）控制"一路走到底再回头"**。像走迷宫——遇到死路就回溯，换一条路继续。

### 1.6.1 DFS 递归 vs 迭代

```python-run
from collections import deque

# 递归版：代码简洁，但受递归深度限制
def dfs_recursive(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)
    print(node, end=' ')

    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)
    return visited

g = {'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'],
     'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E']}

print("DFS 递归: ", end='')
dfs_recursive(g, 'A')
print()
```

```python-run
# 迭代版：显式栈，不受递归深度限制
def dfs_iterative(graph, start):
    stack = [start]
    visited = set()
    order = []

    while stack:
        node = stack.pop()          # 弹出栈顶
        if node in visited:
            continue
        visited.add(node)
        order.append(node)
        # 邻居按逆序入栈，保证和递归版遍历顺序一致
        for neighbor in reversed(graph[node]):
            if neighbor not in visited:
                stack.append(neighbor)
    return order

g = {'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'],
     'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E']}
print(f"DFS 迭代: {' → '.join(dfs_iterative(g, 'A'))}")
```

### 1.6.2 三色标记法——有向图判环

**白（0）= 未访问，灰（1）= 正在栈中访问，黑（2）= 已处理完**。如果 DFS 过程中遇到"灰"节点，说明存在后向边，即有环。

```python-run
def has_cycle_directed(graph):
    """三色标记法判断有向图是否有环"""
    WHITE, GRAY, BLACK = 0, 1, 2
    color = {node: WHITE for node in graph}

    def dfs(node):
        color[node] = GRAY              # 正在访问
        for neighbor in graph[node]:
            if color[neighbor] == GRAY:
                return True             # 遇到正在栈中的节点 → 环
            if color[neighbor] == WHITE:
                if dfs(neighbor):
                    return True
        color[node] = BLACK             # 处理完毕
        return False

    for node in graph:
        if color[node] == WHITE:
            if dfs(node):
                return True
    return False

# DAG（有向无环图）
dag = {'A': ['B', 'C'], 'B': ['D'], 'C': ['D'], 'D': []}
print(f"DAG 有环: {has_cycle_directed(dag)}")      # False

# 有环有向图
cyclic = {'A': ['B'], 'B': ['C'], 'C': ['A']}
print(f"有向环图有环: {has_cycle_directed(cyclic)}")  # True
```

```python-run
# 无向图判环：DFS 时跳过直接父节点即可
def has_cycle_undirected(graph):
    visited = set()

    def dfs(node, parent):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor == parent:         # 跳过直接父节点
                continue
            if neighbor in visited:        # 访问过 + 不是父节点 = 环
                return True
            if dfs(neighbor, node):
                return True
        return False

    for node in graph:
        if node not in visited:
            if dfs(node, None):
                return True
    return False

g = {'A': {'B', 'C'}, 'B': {'A', 'D'}, 'C': {'A', 'D'}, 'D': {'B', 'C'}}
tree = {'A': {'B'}, 'B': {'A', 'C'}, 'C': {'B', 'D'}, 'D': {'C'}}

print(f"g 有环: {has_cycle_undirected(g)}")       # True
print(f"tree 有环: {has_cycle_undirected(tree)}") # False (树 = 无环连通图)
```

### 1.6.3 DFS 的三种遍历序

```python-run
def dfs_orders(graph, start):
    """演示 DFS 的前序、后序、逆后序遍历"""
    visited = set()
    preorder, postorder = [], []

    def dfs(node):
        visited.add(node)
        preorder.append(node)          # 前序：第一次访问时记录
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs(neighbor)
        postorder.append(node)         # 后序：所有邻居遍历完后记录

    dfs(start)
    return preorder, postorder, postorder[::-1]  # 逆后序 = 后序的逆序

g = {'A': ['B', 'C'], 'B': ['D'], 'C': ['D', 'E'], 'D': [], 'E': []}
pre, post, rev_post = dfs_orders(g, 'A')
print(f"前序:   {pre}")
print(f"后序:   {post}")
print(f"逆后序: {rev_post}   ← DAG 的拓扑排序")
```

> **逆后序（Reverse Postorder）就是 DAG 的拓扑排序**——DFS 版的拓扑排序。6.12 介绍了基于 BFS + 入度表的 Kahn 算法，两者等价。

### 1.6.4 枚举两点间所有路径

```python-run
def all_paths(graph, start, target):
    """DFS 回溯枚举从 start 到 target 的所有简单路径"""
    result = []

    def dfs(node, path, visited):
        if node == target:
            result.append(path[:])        # 找到一条路径，深拷贝
            return
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                path.append(neighbor)
                dfs(neighbor, path, visited)
                path.pop()                # 回溯
                visited.remove(neighbor)  # 回溯

    dfs(start, [start], {start})
    return result

g = {'A': ['B', 'C'], 'B': ['D'], 'C': ['B', 'D'], 'D': []}
paths = all_paths(g, 'A', 'D')
for i, p in enumerate(paths):
    print(f"路径 {i + 1}: {' → '.join(p)}")
```

## 1.7 进阶图算法

### 1.7.1 二部图判定（染色法）

**二部图：节点可以分成两组，所有边都跨组连接。** 判定方法：BFS/DFS 给相邻节点染不同颜色，如果冲突则不是二部图。

```python-run
from collections import deque

def is_bipartite(graph):
    """BFS 染色法判断无向图是否为二部图"""
    color = {}                          # 0 和 1 两种颜色

    def bfs(start):
        queue = deque([start])
        color[start] = 0
        while queue:
            node = queue.popleft()
            for neighbor in graph[node]:
                if neighbor not in color:
                    color[neighbor] = 1 - color[node]  # 染相反颜色
                    queue.append(neighbor)
                elif color[neighbor] == color[node]:
                    return False        # 同色相邻 → 不是二部图
        return True

    for node in graph:
        if node not in color:
            if not bfs(node):
                return False
    return True

# 二部图：{A,C} 一组，{B,D} 一组
bipartite = {'A': {'B', 'D'}, 'B': {'A', 'C'}, 'C': {'B', 'D'}, 'D': {'A', 'C'}}
# 奇数长度的环 → 不是二部图
odd_cycle = {'A': {'B', 'C'}, 'B': {'A', 'C'}, 'C': {'A', 'B'}}

print(f"bipartite 是二部图: {is_bipartite(bipartite)}")   # True
print(f"odd_cycle 是二部图: {is_bipartite(odd_cycle)}")   # False (三角形)
```

**实际应用**：推荐系统中判断用户-商品关系（天然二部图）、考试排座位（邻座不能同班同学）、稳定匹配问题。

### 1.7.2 强连通分量——Kosaraju 算法

**强连通分量（SCC）：有向图中，分量内任意两点互相可达。** Kosaraju 用两次 DFS 找到所有 SCC。

```python-run
def kosaraju(graph):
    """Kosaraju 算法：两次 DFS 找出所有强连通分量"""
    # 第一次 DFS：记录完成时间（后序）
    visited = set()
    finish_order = []

    def dfs1(node):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs1(neighbor)
        finish_order.append(node)          # 后序

    for node in graph:
        if node not in visited:
            dfs1(node)

    # 构建转置图（反转所有边的方向）
    reversed_graph = {node: [] for node in graph}
    for u in graph:
        for v in graph[u]:
            reversed_graph[v].append(u)

    # 第二次 DFS：按完成时间逆序在转置图上遍历
    visited.clear()
    sccs = []

    def dfs2(node, component):
        visited.add(node)
        component.append(node)
        for neighbor in reversed_graph[node]:
            if neighbor not in visited:
                dfs2(neighbor, component)

    for node in reversed(finish_order):    # 逆序处理
        if node not in visited:
            comp = []
            dfs2(node, comp)
            sccs.append(comp)

    return sccs

#       A → B → C
#       ↑   ↓   |
#       └───┘ ←─ D
# SCC: {A,B}, {C,D}
dg = {'A': ['B'], 'B': ['C', 'A'], 'C': ['D'], 'D': ['C']}
print(f"强连通分量: {kosaraju(dg)}")
```

**Kosaraju 的直觉**：
1. 第一次 DFS 确定节点的"处理完成"顺序（后序）
2. 转置图后，原本的 SCC 仍然互相可达，但 SCC 之间的"出口"变成了"入口"
3. 第二次 DFS 按后序逆序在转置图上遍历，每个连通分量就是一个 SCC

> SCC 算法还有 **Tarjan（单次 DFS）** 和 **Gabow**。Kosaraju 最直观、最好理解，Tarjan 在实际中更快。

### 1.7.3 最小生成树——Prim 算法

**最小生成树（MST）：连接所有节点的最小总权重子图。** Prim 算法像 Dijkstra，但维护的是"到生成树的最短距离"而非"到起点的最短距离"。

```python-run
import heapq

def prim(graph):
    """Prim 算法求最小生成树，返回 (总权重, 边列表)"""
    start = next(iter(graph))              # 任选起点
    heap = [(0, start, None)]              # (到树的距离, 节点, 来源)
    visited = set()
    mst_edges = []
    total_weight = 0

    while heap and len(visited) < len(graph):
        weight, node, prev = heapq.heappop(heap)
        if node in visited:
            continue
        visited.add(node)
        if prev is not None:
            mst_edges.append((prev, node, weight))
            total_weight += weight

        for neighbor, w in graph[node]:
            if neighbor not in visited:
                heapq.heappush(heap, (w, neighbor, node))

    return total_weight, mst_edges

# 加权无向图
wg = {
    'A': [('B', 4), ('C', 2)],
    'B': [('A', 4), ('C', 1), ('D', 5)],
    'C': [('A', 2), ('B', 1), ('D', 8), ('E', 10)],
    'D': [('B', 5), ('C', 8), ('E', 2), ('F', 6)],
    'E': [('C', 10), ('D', 2), ('F', 3)],
    'F': [('D', 6), ('E', 3)],
}

w, edges = prim(wg)
print(f"MST 总权重: {w}")
print(f"MST 边: {edges}")
```

> Prim = 从一点开始"生长"生成树，每次贪心选择到当前树最近的外部节点。复杂度 O(E log V)，和 Dijkstra 完全相同。

### 1.7.4 最小生成树——Kruskal 算法

**Kruskal 算法：按权重升序排列所有边，依次加入不构成环的边。** 需要并查集判断环（详见第 6.12 章）。

```python-run
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1
        return True

def kruskal(nodes, edges):
    """Kruskal 算法：(节点列表, [(u, v, weight), ...]) → MST"""
    idx = {name: i for i, name in enumerate(nodes)}
    uf = UnionFind(len(nodes))
    sorted_edges = sorted(edges, key=lambda e: e[2])  # 按权重排序

    mst = []
    total = 0
    for u, v, w in sorted_edges:
        if uf.union(idx[u], idx[v]):      # 不构成环
            mst.append((u, v, w))
            total += w
            if len(mst) == len(nodes) - 1:  # MST 恰好 V-1 条边
                break
    return total, mst

nodes = ['A', 'B', 'C', 'D', 'E', 'F']
edges = [
    ('A', 'B', 4), ('A', 'C', 2), ('B', 'C', 1), ('B', 'D', 5),
    ('C', 'D', 8), ('C', 'E', 10), ('D', 'E', 2), ('D', 'F', 6),
    ('E', 'F', 3),
]
w, mst_edges = kruskal(nodes, edges)
print(f"Kruskal MST 总权重: {w}")
print(f"Kruskal MST 边: {mst_edges}")
```

| 维度 | Prim | Kruskal |
|------|-------|---------|
| 数据结构 | 堆（heapq） | 并查集（Union-Find） |
| 复杂度 | O(E log V) | O(E log E) |
| 适合 | 稠密图 | 稀疏图、边列表输入 |
| 核心思想 | 从一点生长 | 排序边 + 贪心选择 |

### 1.7.5 欧拉路径

**欧拉路径：经过每条边恰好一次的路径。** 欧拉回路：起点 = 终点的欧拉路径。

**存在性判断（无向图）**：
- 欧拉回路：所有节点的度都是偶数
- 欧拉路径（非回路）：恰好两个节点的度是奇数（起终点）

```python-run
from collections import defaultdict

def euler_exists(graph, directed=False):
    """判断图是否存在欧拉路径/回路"""
    if directed:
        indegree = defaultdict(int)
        outdegree = {}
        for u in graph:
            outdegree[u] = len(graph[u])
            for v in graph[u]:
                indegree[v] += 1

        start_nodes = end_nodes = 0
        for node in set(list(indegree.keys()) + list(outdegree.keys())):
            out = outdegree.get(node, 0)
            inn = indegree.get(node, 0)
            if out - inn == 1:
                start_nodes += 1
            elif inn - out == 1:
                end_nodes += 1
            elif out != inn:
                return "无欧拉路径"
        if start_nodes == 0 and end_nodes == 0:
            return "欧拉回路"
        if start_nodes == 1 and end_nodes == 1:
            return "欧拉路径（非回路）"
        return "无欧拉路径"
    else:
        odd_count = sum(1 for node, neighbors in graph.items()
                       if len(neighbors) % 2 != 0)
        if odd_count == 0:
            return "欧拉回路"
        if odd_count == 2:
            return "欧拉路径（非回路）"
        return "无欧拉路径"

# 哥尼斯堡七桥问题（4 个节点，4 个奇数度 → 无解）
#    A──B     图中有 4 个奇数度节点
#    │\/│
#    C──D
konigsberg = {'A': {'B', 'C', 'D'}, 'B': {'A', 'C', 'D'},
              'C': {'A', 'B', 'D'}, 'D': {'A', 'B', 'C'}}
print(f"哥尼斯堡七桥: {euler_exists(konigsberg)}")

# 欧拉回路
euler_cycle = {'A': {'B', 'C'}, 'B': {'A', 'C'}, 'C': {'A', 'B'}}
print(f"全部偶数度: {euler_exists(euler_cycle)}")

# 欧拉路径
euler_path = {'A': {'B'}, 'B': {'A', 'C'}, 'C': {'B', 'D'}, 'D': {'C'}}
print(f"两个奇数度: {euler_exists(euler_path)}")
```

## 本章小结

1. **图 = 节点 + 边**。无向边是双向关系（好友），有向边是单向关系（关注）。Python 用 `dict[set]` 做邻接表是 90% 场景的最优解。
2. **邻接表 O(V+E) 空间**，遍历邻居 O(度)；邻接矩阵 O(V²) 空间但 O(1) 查边；边列表适合 Kruskal/Bellman-Ford 等需要遍历所有边的算法。
3. **BFS = 队列，按层扩散**——用于无权图最短路径、连通分量、层级分析。标记 `visited` 在入队时做，不是出队时。
4. **DFS = 栈，一条路走到底**——递归版简洁但有栈溢出风险，迭代版显式维护栈。三色标记法（白灰黑）是有向图判环的标准方法。
5. **`len(queue)` 是 BFS 分层的秘诀**；**`prev` 字典 + 回溯**避免在队列中存储完整路径，节省内存。
6. **Kosaraju 两次 DFS + 转置图**找出强连通分量；**Prim/Kruskal** 求最小生成树——前者从点生长用堆，后者排序边用并查集。
7. **图算法的选择取决于图的特征**：有向/无向、加权/无权、稀疏/稠密——不同特征对应不同的最优算法。
