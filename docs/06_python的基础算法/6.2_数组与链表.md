# 6.2 数组与链表

数组和链表是最基础的两种线性数据结构。Python 的 `list` 底层是动态数组，而链表在 Python 标准库中没有直接对应——这正是理解它们区别的起点。

> 📖 **官方文档**：[Built-in Types — list](https://docs.python.org/3/library/stdtypes.html#list)

---

## 1.1 数组：连续内存的魅力

### 是什么

数组是**连续内存**中存放**同类型元素**的结构。Python 的 `list` 在 C 层面是一个指针数组——连续内存里放的是指向每个 Python 对象的指针，元素本身可以不同类型。

```python-run
import sys

lst = []
print(f"空 list: {sys.getsizeof(lst)} 字节")

for i in range(8):
    lst.append(i)
    print(f"元素数={i+1:>2}, 占用={sys.getsizeof(lst):>3} 字节")
```

**核心操作复杂度**：

| 操作 | 复杂度 | 原因 |
|------|--------|------|
| `lst[i]` 随机访问 | **O(1)** | 基地址 + i × 指针大小，直接定位 |
| `lst.append(x)` | **O(1)** 均摊 | 超额分配，偶尔扩容 |
| `lst.insert(0, x)` | O(n) | 所有元素右移一位 |
| `lst.remove(x)` | O(n) | 查找 + 元素左移填补空洞 |
| `x in lst` | O(n) | 线性扫描 |

```python-run
# 头/尾插入的巨大差异
import time

n = 100_000
lst = list(range(n))

t0 = time.perf_counter()
for _ in range(1000):
    lst.append(42)          # O(1) — 尾部追加
t1 = time.perf_counter()
print(f"append 1000次: {t1-t0:.6f}s")

# 重建列表（insert(0) 太慢，直接重建）
lst = list(range(n))
t0 = time.perf_counter()
for _ in range(1000):
    lst.insert(0, 42)       # O(n) — 头部插入
t1 = time.perf_counter()
print(f"insert(0) 1000次: {t1-t0:.4f}s")
```

## 1.2 链表：零拷贝的插入删除

### 是什么

链表每个结点包含**数据 + 指向下一个结点的引用**。天然支持 O(1) 头部插入和删除——不需要移动任何元素。

### 为什么 Python 没有内置链表

Python 的 `list` 是指针数组而非直接存元素，且 `deque` 覆盖了"双端快速插入"的需求。手写链表主要出现在：
- 面试/教学场景
- 需要频繁在**中间位置**插入删除（避免 O(n) 搬移）
- LRU Cache 等组合数据结构

### 怎么实现

```python-run
from __future__ import annotations
from typing import Generic, TypeVar

T = TypeVar('T')

class Node(Generic[T]):
    def __init__(self, val: T, next: Node[T] | None = None):
        self.val = val
        self.next = next

class LinkedList(Generic[T]):
    def __init__(self):
        self.head: Node[T] | None = None

    def prepend(self, val: T) -> None:     # O(1)
        self.head = Node(val, self.head)

    def find(self, val: T) -> Node[T] | None:  # O(n)
        cur = self.head
        while cur:
            if cur.val == val:
                return cur
            cur = cur.next
        return None

    def delete(self, val: T) -> None:      # O(n)
        if not self.head:
            return
        if self.head.val == val:
            self.head = self.head.next
            return
        cur = self.head
        while cur.next:
            if cur.next.val == val:
                cur.next = cur.next.next
                return
            cur = cur.next

    def to_list(self) -> list[T]:
        result = []
        cur = self.head
        while cur:
            result.append(cur.val)
            cur = cur.next
        return result

# 演示
ll = LinkedList[int]()
ll.prepend(3)
ll.prepend(2)
ll.prepend(1)
print(f"链表内容: {ll.to_list()}")   # [1, 2, 3]
print(f"查找 2: {ll.find(2).val if ll.find(2) else '未找到'}")

ll.delete(2)
print(f"删除 2 后: {ll.to_list()}")  # [1, 3]
```

## 1.3 数组 vs 链表：一张表说清

| 维度 | 数组（Python `list`） | 链表（手写 `Node`） |
|------|----------------------|---------------------|
| 随机访问 `a[i]` | O(1) | O(n) — 从头遍历 |
| 头部插入 | O(n) | **O(1)** |
| 尾部插入 | O(1) 均摊 | O(n) 或 O(1)（维护尾指针） |
| 中间插入/删除 | O(n) | O(n) 查找 + O(1) 改指针 |
| 内存开销 | 超额分配，连续内存 | 每个结点额外存储 `next` 指针 |
| 缓存友好度 | **高**（连续内存） | 低（指针跳转） |
| 标准库对应 | `list`, `array.array` | 无（`deque` 是双向链表） |

> **实战指导**：99% 的场景直接用 `list`。需要头部快速操作用 `deque`。只有在需要频繁在**中间位置**插入/删除时才自己写链表。

## 1.4 跳表（Skip List）：链表 + 索引

跳表在有序链表之上添加**多层索引**，实现 **O(log n)** 的查找、插入、删除——Redis 的 ZSet 就是跳表实现的。

```python-run
# 跳表简化的 50 行实现（层数随机，只含 search + insert）
import random

class SkipNode:
    def __init__(self, val, level):
        self.val = val
        self.forward = [None] * (level + 1)   # 每层的前进指针

class SkipList:
    MAX_LEVEL = 16

    def __init__(self):
        self.head = SkipNode(-float('inf'), self.MAX_LEVEL)
        self.level = 0                        # 当前最高层

    def random_level(self):
        level = 0
        while random.random() < 0.5 and level < self.MAX_LEVEL:
            level += 1
        return level

    def insert(self, val):
        update = [None] * (self.MAX_LEVEL + 1)
        cur = self.head
        # 从最高层向下查找每层的前驱
        for i in range(self.level, -1, -1):
            while cur.forward[i] and cur.forward[i].val < val:
                cur = cur.forward[i]
            update[i] = cur

        level = self.random_level()
        if level > self.level:
            for i in range(self.level + 1, level + 1):
                update[i] = self.head
            self.level = level

        node = SkipNode(val, level)
        for i in range(level + 1):
            node.forward[i] = update[i].forward[i]
            update[i].forward[i] = node

    def search(self, val):
        cur = self.head
        for i in range(self.level, -1, -1):
            while cur.forward[i] and cur.forward[i].val < val:
                cur = cur.forward[i]
        cur = cur.forward[0]
        return cur is not None and cur.val == val

# 演示
sl = SkipList()
for v in [3, 6, 7, 9, 12, 19, 17, 26, 21, 25]:
    sl.insert(v)

print(f"search(19): {sl.search(19)}")  # True
print(f"search(15): {sl.search(15)}")  # False
```

> 跳表的核心思想：**用空间换时间**。多层索引让查找跳过大量元素，平均 O(log n)。它比平衡树实现更简单，Redis 和 LevelDB 都在用它。

## 本章小结

1. **数组 = 连续内存，O(1) 随机访问**，但头/中部插入要搬移数据 O(n)
2. **链表 = 分散内存 + 指针串联**，O(1) 改指针插入，但 O(n) 随机访问
3. Python `list` 是指针数组，不是存元素本身——比 C 数组灵活但访问慢于 C
4. **头部频繁操作用 `deque`**，中间频繁插入删除才手写链表
5. **跳表 = 有序链表 + 多层索引**，O(log n) 查找，比平衡树实现更简单

## 思考题

1. Python `list` 底层是指针数组，那 `list[0]` 访问经历了哪些步骤？
> 答: python通过对象头获取底层连续数组的首地址, 然后利用 `首地址 + 索引 x 指针大小` 计算出目标元素的内存地址, 然后解引用获取对象; 整个过程是纯粹的C语言指针运算,时间复杂度O(1).
2. 为什么 Python 官方不提供内置链表？如果提供了，什么场景会用它？
> 答:C语言的连续内存布局能最大化利用CPU缓存(Cache-friendly), 在绝大数数场景下性能远超指针跳转的链表; 只有在频繁需要再中间进行插入/删除切极少随机访问的极端场景才会使用链表.
3. 跳表查询时，为什么从最高层开始向下找，而不是从第 0 层开始？
> 答: 利用高层节点 跨度大的特性, 像二分查找一样以 O(log n)的步长快速逼近目标
4. 用双向链表实现一个 LRU Cache，需要哪些操作？各自复杂度是多少？
> 答: python3.7+,lru_cache底层是一个双向链表+字典的混合体 , 字典可以实现O(1)的键值查找,而双向链表用于维护元素的访问顺序, 从而在O(1)的时间内完成缓存淘汰.
