# 2.4 可迭代与可调用对象

Python 的 `for` 循环和函数调用看似简单，背后是一套精确的协议——可迭代协议（Iterable Protocol）和可调用协议（Callable Protocol）。理解了这些协议，你就能写出和内置类型一样自然的自定义类。本章回答三个问题：什么是迭代器（Iterator）和可迭代对象（Iterable）、为什么需要区分它们、怎么让自定义对象支持 `for` 和 `()` 调用。

## 4.1 迭代器协议：`__iter__` 与 `__next__`

### 是什么

一个对象要能被 `for` 循环消费，必须实现迭代器协议：

- **`__iter__`**：返回迭代器对象自身（`self`）
- **`__next__`**：返回下一个值，没有更多值时抛出 `StopIteration`

```python
class Countdown:
    """最简单的迭代器——倒计时"""
    def __init__(self, start: int):
        self.current = start

    def __iter__(self):
        return self            # 迭代器返回自身

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        self.current -= 1
        return self.current + 1

for n in Countdown(3):         # 3 → 2 → 1
    print(n)
```

### 为什么需要——Iterable vs Iterator

这可能是 Python 初学者最容易混淆的概念：

| | 可迭代对象（Iterable） | 迭代器（Iterator） |
|---|---|---|
| **协议** | 有 `__iter__`，返回一个迭代器 | 有 `__iter__`（返回 self）和 `__next__` |
| **能否被 `for` 消费** | 是，`for` 先调 `__iter__` 拿到迭代器 | 是 |
| **能否多次遍历** | 是，每次调 `__iter__` 返回**新的**迭代器 | **否**——到头了就是到头了 |
| **`next()` 能否手动访问** | 否 | 是 |
| **典型例子** | `list`, `tuple`, `dict`, `str` | `list_iterator`, 文件句柄, `generator` |

**核心区别：文件句柄是迭代器，列表只是可迭代对象。**

```python
# list 是可迭代对象，不是迭代器
nums = [1, 2, 3]
print(hasattr(nums, '__iter__'))    # True
print(hasattr(nums, '__next__'))    # False —— 需要用 iter() 获取迭代器

# 每次 iter(list) 生成新迭代器——可以多次遍历
it1 = iter(nums)
it2 = iter(nums)
print(list(it1))   # [1, 2, 3]
print(list(it2))   # [1, 2, 3] —— it2 不受 it1 耗尽的影响

# 文件句柄本身就是迭代器——只能遍历一次
with open('test.txt', 'w') as f:
    f.write('line1\nline2\n')
with open('test.txt') as f:
    print(hasattr(f, '__next__'))   # True —— 文件本身是迭代器！
    for line in f:
        print(line.strip())          # line1 → line2
    for line in f:
        print(line.strip())          # 什么都没有——迭代器已耗尽
```

> 这解释了为什么文件句柄只能读一次就得 `seek(0)` 重置——它本身就是迭代器，而不是每次返回新迭代器的可迭代对象。

> 📖 官方文档：[Iterator Types](https://docs.python.org/3/library/stdtypes.html#iterator-types)

## 4.2 `__getitem__` 降级路径

Python 在找 `__iter__` 失败时，会降级到 `__getitem__`：从 0 开始调 `obj[0]`, `obj[1]`, ... 直到 `IndexError`。

```python
class OldStyleIterable:
    """没有 __iter__，但 Python 能用 __getitem__ 遍历它"""
    def __init__(self, data):
        self.data = data

    def __getitem__(self, index: int):
        if index >= len(self.data):
            raise IndexError
        return self.data[index]

obj = OldStyleIterable(['a', 'b', 'c'])
for item in obj:           # Python 自动从 0 开始逐个索引
    print(item)            # a → b → c
```

**为什么需要知道这个？** 第一，你在调试老代码时可能遇到只定义 `__getitem__` 的类——它不是 bug，是故意的。第二，某些框架利用这个兼容特性做惰性求值——不一次性生成所有数据，而是被访问时才计算。

> 📖 官方文档：[The `for` statement](https://docs.python.org/3/reference/compound_stmts.html#the-for-statement) 底部说明

## 4.3 生成器（Generator）：最简单的迭代器

### 是什么

生成器函数（Generator Function）——含 `yield` 关键字的函数——是创建迭代器的捷径。你不需要写 `__iter__`/`__next__`，Python 自动生成。

```python
def countdown_gen(start: int):
    """生成器函数——等价于前面的 Countdown 类"""
    while start > 0:
        yield start
        start -= 1

g = countdown_gen(3)
print(type(g))        # <class 'generator'>
print(list(g))        # [3, 2, 1]
```

### 生成器 vs 手写迭代器

```python
# 手写迭代器——13 行
class CountdownClass:
    def __init__(self, start):
        self.current = start
    def __iter__(self):
        return self
    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        self.current -= 1
        return self.current + 1

# 生成器函数——4 行，语义完全等价
def countdown_func(start):
    while start > 0:
        yield start
        start -= 1
```

**结论：能用生成器就用生成器。** 只有当迭代逻辑非常复杂、需要多个方法协作维护状态时，才考虑手写迭代器类。

### `yield from`：委托子迭代器（3.3+）

```python
def flatten(nested):
    """展平多层嵌套的可迭代对象"""
    for item in nested:
        if isinstance(item, (list, tuple)):
            yield from flatten(item)   # 委托子生成器，不用手动嵌套 for
        else:
            yield item

print(list(flatten([1, [2, 3], [[4, 5], 6]])))  # [1, 2, 3, 4, 5, 6]
```

`yield from` 不仅简化代码，还会自动转发 `send()`、`throw()` 和 `close()`——这是实现协程委托的核心机制。

> 📖 官方文档：[Generators](https://docs.python.org/3/reference/expressions.html#yield-expressions), [`yield from`](https://docs.python.org/3/reference/expressions.html#yield-from)

## 4.4 `__call__`：让实例成为"函数"

### 是什么

实现了 `__call__` 的类，其实例可以像函数一样用 `obj()` 调用。这就是"可调用对象（Callable）"——它是有状态的函数。

```python
class Counter:
    """有状态的"函数"——每次调用计数加一"""
    def __init__(self, start: int = 0):
        self.count = start

    def __call__(self) -> int:
        self.count += 1
        return self.count

c = Counter(10)
print(c())   # 11
print(c())   # 12
print(c())   # 13
```

### 为什么需要——闭包做不到的事

闭包只能通过 `nonlocal` 额外保存两三个变量。当你需要大量状态或多个方法时，类更有优势：

```python
class RateLimiter:
    """基于 __call__ 的限流器——状态由实例属性维护"""
    def __init__(self, max_calls: int, per_seconds: float = 1.0):
        self.max_calls = max_calls
        self.per_seconds = per_seconds
        self._calls: list[float] = []

    def __call__(self) -> bool:
        import time
        now = time.time()
        self._calls = [t for t in self._calls if now - t < self.per_seconds]
        if len(self._calls) < self.max_calls:
            self._calls.append(now)
            return True
        return False

limiter = RateLimiter(max_calls=3, per_seconds=1.0)
print([limiter() for _ in range(5)])
# [True, True, True, False, False] —— 一秒内最多 3 次
```

### 用类实现装饰器（Decorator）

装饰器本质上是接受函数、返回函数的可调用对象。用类实现装饰器能利用实例属性保存状态：

```python
import functools

class Retry:
    """用 __call__ 实现的装饰器——重试失败的函数调用"""
    def __init__(self, max_attempts: int = 3):
        self.max_attempts = max_attempts

    def __call__(self, func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            last_exc = None
            for attempt in range(self.max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    last_exc = e
                    print(f"第 {attempt + 1} 次失败: {e}")
            raise last_exc  # type: ignore[misc]
        return wrapper

@Retry(max_attempts=3)
def unstable_request():
    import random
    if random.random() < 0.7:
        raise ConnectionError("网络故障")
    return "成功"

# 偶尔能看到：第 1 次失败 → 第 2 次失败 → 成功
```

> 📖 官方文档：[`__call__`](https://docs.python.org/3/reference/datamodel.html#object.__call__)

## 4.5 实战：实现一个"类文件"对象

下面把迭代器协议和可调用协议串起来：一个能在 `for` 循环中逐行读取、也能用 `read()` 一次性获取全部内容的自定义对象。

```python-run
class StringIO:
    """一个文件对象的简化实现——支持 for 行迭代 + read()"""
    def __init__(self, content: str):
        self._lines = content.splitlines(keepends=True)
        self._pos = 0          # 当前行位置

    # ── 迭代器协议：支持 for line in obj ──
    def __iter__(self):
        return self            # 自身是迭代器

    def __next__(self) -> str:
        if self._pos >= len(self._lines):
            self._pos = 0      # 耗尽后自动重置——实际文件不这样做
            raise StopIteration
        line = self._lines[self._pos]
        self._pos += 1
        return line

    # ── 可调用：支持 read(size) ──
    def read(self, size: int = -1) -> str:
        remaining = ''.join(self._lines[self._pos:])
        if size == -1 or size >= len(remaining):
            result, self._pos = remaining, len(self._lines)
        else:
            result, self._pos = remaining[:size], self._pos
        return result

    def __repr__(self) -> str:
        visible = self._lines[self._pos:self._pos + 2]
        return f"StringIO({visible!r}, pos={self._pos})"


# ── 使用演示 ──
text = "hello\nworld\npython"
io = StringIO(text)

# 方式 1：for 逐行迭代
for line in io:
    print(repr(line))          # 'hello\n' → 'world\n' → 'python'

# 方式 2：read() 一次性
io2 = StringIO(text)
print(repr(io2.read()))       # 'hello\nworld\npython'
```

核心知识点：迭代器是有状态的——一次 `for` 循环消费完毕后，指针到头。实际文件对象也是这样，`read()` 到末尾后返回空字符串。需要重置时必须显式 `seek(0)`。

## 4.6 `iter()` 的 sentinel 模式

`iter(callable, sentinel)` 是迭代器协议中最容易被忽略的一个用法：反复调用 `callable()`，直到返回值等于 `sentinel` 时停止。

```python
# 从文件中逐块读取，直到读取到空字节
with open('large_file.bin', 'rb') as f:
    for chunk in iter(lambda: f.read(4096), b''):
        process(chunk)   # 每次处理 4KB，直到文件末尾

# 另一个经典用例：读取控制台输入直到用户输入 'quit'
for line in iter(input, 'quit'):
    print(f"你输入了: {line}")
```

**和 `while` 循环的对比：**

```python
# while 循环写法——不够 Python
while True:
    chunk = f.read(4096)
    if not chunk:
        break
    process(chunk)

# iter(sentinel) 写法——C 风格的"读到哨兵为止"用 Python 方式表达
for chunk in iter(lambda: f.read(4096), b''):
    process(chunk)
```

```
                    Iterable (有 __iter__)
                        │
                        │ iter(obj)
                        ▼
              Iterator (有 __iter__ + __next__)
                        │
                        │ 也可以用生成器 yield

                        │
            ┌───────────┴───────────┐
            │                       │

            │                       │

                        │

                        │
      ┌─────────────────┴─────────────────┐
      │ for 自动捕获 StopIteration         │
      │ 停止循环，不抛给用户               │
      └───────────────────────────────────┘
```

## 4.8 迭代器 vs 普通序列：`iter()` 到底做了什么

```python
nums = [1, 2, 3]

# for 循环在幕后做的事：
it = iter(nums)           # 1. 调 nums.__iter__() → 拿到 list_iterator
while True:
    try:
        value = next(it)  # 2. 调 it.__next__() → 拿出下一个元素
        print(value)      # 3. 处理当前元素
    except StopIteration:
        break             # 4. 迭代器耗尽，退出
```

关键点：`for` 循环会自动捕获 `StopIteration`。所以你在手写 `__next__` 时只需放心地抛出 `StopIteration`——`for` 不会让它外溢。

## 本章小结

- 可迭代对象（Iterable）有 `__iter__`，返回一个迭代器；迭代器（Iterator）同时有 `__iter__` 和 `__next__`。`list` 只是可迭代对象，文件句柄自身就是迭代器。
- 生成器函数（`yield`）是创建迭代器的最简方式；`yield from` 实现子迭代器委托，自动转发 `send()`、`throw()` 和 `close()`。
- `__getitem__` 是迭代的降级路径——没有 `__iter__` 时 Python 从下标 0 开始逐个索引，直到 `IndexError`。
- `__call__` 让实例像函数一样被调用，适合有状态函数和用类实现装饰器（状态由实例属性维护）。
- `iter(callable, sentinel)` 模式以 Python 方式表达"反复调用直到哨兵值"，比手动 `while` 循环更优雅。
