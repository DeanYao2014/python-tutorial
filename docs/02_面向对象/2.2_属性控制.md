# 2.2 属性控制

本章回答三个问题：**Python 如何拦截属性访问**、**什么时候该用哪种拦截机制**、**滥用拦截的代价是什么**。

## 2.1 `@property`：把方法伪装成属性

`@property` 是 Python 中最常用的属性控制工具。它让你在"外部看起来是属性，内部执行的是方法"之间找到平衡——既保持了简洁的 `obj.attr` 语法，又能在访问时执行任意逻辑。

### 完整示例：带校验的温度类

```python-run
class Temperature:
    """摄氏度——property 的完整用法"""
    def __init__(self, celsius: float = 0):
        self._celsius = celsius  # 注意：直接赋值 _celsius，不走 setter

    @property
    def celsius(self) -> float:
        """摄氏度（读）"""
        return self._celsius

    @celsius.setter
    def celsius(self, value: float):
        """摄氏度（写）——带校验"""
        if value < -273.15:
            raise ValueError(f"低于绝对零度: {value}")
        self._celsius = value

    @celsius.deleter
    def celsius(self):
        """摄氏度（删）——重置为 0"""
        self._celsius = 0

    @property
    def fahrenheit(self) -> float:
        """华氏度（只读）——由摄氏度计算得出"""
        return self._celsius * 9 / 5 + 32

t = Temperature(100)
print(t.celsius)      # 100
print(t.fahrenheit)   # 212.0
t.celsius = 0
print(t.fahrenheit)   # 32.0
del t.celsius
print(t.celsius)      # 0
```

### `@property` 的正确用法 vs 错误用法

| 正确 ✅ | 错误 ❌ |
|---------|--------|
| 校验、计算、延迟加载 | 访问数据库、发送网络请求 |
| 不变式保护（如 `area` 只读） | 修改全局状态 |
| 兼容旧 API（方法 → 属性） | 在 getter 中抛出异常 |
| 简单转换（`celsius` ↔ `fahrenheit`） | 执行耗时操作（用户以为在访问属性） |

```python-run
# 反模式：property 中执行 I/O
import time

class BadAPI:
    @property
    def data(self):
        time.sleep(2)          # 用户写 BadAPI().data 会卡住 2 秒！
        return {"result": 42}   # 以为是属性访问，实际是网络调用

# 正确做法：property 只做简单计算，I/O 留给方法
class GoodAPI:
    def fetch_data(self):       # 明确告诉调用者：这是操作，不是读取
        time.sleep(2)
        return {"result": 42}
```

> 📖 官方文档：[property](https://docs.python.org/3/library/functions.html#property), [Descriptor HowTo Guide](https://docs.python.org/3/howto/descriptor.html)

## 2.2 `__getattr__` vs `__getattribute__`：两条截然不同的路径

这两个方法名字相似，但语义完全不同。搞混的后果是无限递归。

### `__getattr__`：正常查找失败后才调用

```python-run
class LazyDict:
    """按需从数据库加载属性——__getattr__ 的经典用法"""
    def __init__(self, db_data: dict):
        self._data = db_data

    def __getattr__(self, name: str):
        """仅在正常查找失败时触发"""
        if name.startswith('_'):
            raise AttributeError(name)  # 私有属性不处理，让调用方收到正常错误
        if name in self._data:
            value = self._data[name]
            # 缓存到实例属性——下次就不会走 __getattr__ 了
            setattr(self, name, value)
            return value
        raise AttributeError(f"'{type(self).__name__}' 没有属性 '{name}'")

d = LazyDict({"color": "红", "weight": 10})

# 第一次访问 color：走 __getattr__，加载并缓存
print(d.color)   # 红

# 第二次访问 color：直接从 __dict__ 读取，不走 __getattr__
print(d.color)   # 红

# 不存在的属性：__getattr__ 中 raise AttributeError
try:
    print(d.nonexistent)
except AttributeError as e:
    print(f"预期错误: {e}")
```

### `__getattribute__`：每一次属性访问都触发

```python-run
class Spy:
    """记录每一次属性访问——__getattribute__ 的用法"""
    def __init__(self, name):
        self._access_log: list[str] = []
        object.__setattr__(self, 'name', name)  # 绕过自己的 __setattr__

    def __getattribute__(self, name):
        # 关键：必须用 object.__getattribute__，不能用 self.xxx
        if not name.startswith('_'):
            try:
                log = object.__getattribute__(self, '_access_log')
                log.append(name)
            except AttributeError:
                pass
        return object.__getattribute__(self, name)

    def report(self):
        return f"访问过的属性: {self._access_log}"

s = Spy("spybot")
s.name        # 触发 __getattribute__ → 记录 'name'
s._access_log # 也不会被拦截（以下划线开头的跳过日志）
print(s.report())
```

**对比表格：**

| | `__getattr__` | `__getattribute__` |
|---|---|---|
| 触发时机 | 常规查找失败后 | **每**次属性访问前 |
| 性能影响 | 几乎无 | 有（每条 `obj.x` 都多一次函数调用） |
| 实现难度 | 简单——不用管正常属性 | 困难——必须逐个处理，极易无限递归 |
| 典型用途 | 代理、延迟加载、默认值 | 日志、权限控制、拦截一切 |
| 常见错误 | 忘了 `raise AttributeError` | 用 `self.name` 代替 `object.__getattribute__` |

> ⚠️ **黄金法则**：能用 `__getattr__` 就别用 `__getattribute__`。后者是最后的武器——当你确实需要拦截**每一个** `.` 操作时才用它。

> 📖 官方文档：[object.__getattr__](https://docs.python.org/3/reference/datamodel.html#object.__getattr__), [object.__getattribute__](https://docs.python.org/3/reference/datamodel.html#object.__getattribute__)

## 2.3 `__setattr__` 和 `__delattr__`：拦截写入和删除

这两个方法让你可以控制属性**写入**和**删除**。配合 `__getattribute__` 可以实现完整的属性访问审计。

### 实战：属性变更日志代理

```python-run
class LoggingProxy:
    """代理任意对象，记录所有属性变更"""
    def __init__(self, target):
        object.__setattr__(self, '_target', target)
        object.__setattr__(self, '_change_log', [])

    def __getattr__(self, name):
        # 对目标属性的读取委托给实际对象
        return getattr(self._target, name)

    def __setattr__(self, name, value):
        old = getattr(self._target, name, None)
        object.__setattr__(self._target, name, value)
        self._change_log.append(f"SET {name}: {old!r} → {value!r}")

    def __delattr__(self, name):
        old_value = getattr(self._target, name, f"<不存在>")
        object.__delattr__(self._target, name)
        self._change_log.append(f"DEL {name} (原值为 {old_value!r})")

    def show_log(self):
        return self._change_log

# 使用代理包装一个普通对象
class User:
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

user = User("Alice", 30)
proxy = LoggingProxy(user)

proxy.name = "Bob"
proxy.age = 31
del proxy.age

print(proxy.name)       # Bob（通过 __getattr__ 委托给 target）
for entry in proxy.show_log():
    print(f"  {entry}")

# 验证底层目标被真正修改了
print(f"\n原始对象: name={user.name}, age={user.age}")
```

> 📖 官方文档：[object.__setattr__](https://docs.python.org/3/reference/datamodel.html#object.__setattr__), [object.__delattr__](https://docs.python.org/3/reference/datamodel.html#object.__delattr__)

## 2.4 `__slots__`：用空间换时间

`__slots__` 是 Python 中少数几个能**同时**减少内存和加速属性访问的机制。原理很简单：默认情况下，每个实例都有一个 `__dict__` 字典存储属性；`__slots__` 用固定大小的 C 数组替代了这个字典。

### 内存对比

```python-run
import sys

class WithDict:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class WithSlots:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

d = WithDict(1, 2)
s = WithSlots(1, 2)

print(f"WithDict:  {sys.getsizeof(d.__dict__)} bytes (__dict__ 本身)")
# 通常约 104 bytes 以上（空字典 ~64 bytes，每加一个属性增加）
print(f"WithSlots: 没有 __dict__: {hasattr(s, '__dict__')}")
```

`__slots__` 具体做了什么：

1. **移除 `__dict__`**：实例不再有属性字典，不能动态添加属性。
2. **每个属性分配固定槽位**：用 C 数组存储，避免了字典的哈希开销。
3. **加速属性访问**：从数组索引直接读取比从字典散列快（实测微秒级差异，但在百万次访问中可感知）。

### 什么时候用、什么时候不用

| 用 `__slots__` | 不用 `__slots__` |
|---------------|-----------------|
| 创建数百万个小对象（如坐标点、像素） | 需要动态添加属性的场景 |
| 属性集合固定且已知 | 框架基类（会限制子类的灵活性） |
| 内存敏感（嵌入式、数据处理） | 需要 `__dict__` 做序列化/调试 |
| 属性频繁读写 | 对象数量少（< 10K）——不值得 |

### 常见陷阱

```python-run
class Base:
    __slots__ = ('x',)

class Child(Base):
    pass  # 没有定义 __slots__

c = Child()
c.y = 42  # 可以！Child 自动获得了 __dict__
# 这意味着 Child 没有继承"限制"

# 让子类也受约束：
class Child2(Base):
    __slots__ = ('y',)
    # Child2 现在有 __slots__ = ('x', 'y')
    # 仍然不能动态添加属性

c2 = Child2()
c2.x = 1
c2.y = 2
try:
    c2.z = 3  # AttributeError!
except AttributeError as e:
    print(f"预期: {e}")
```

> 📖 官方文档：[__slots__](https://docs.python.org/3/reference/datamodel.html#slots), [Using __slots__](https://docs.python.org/3/reference/datamodel.html#notes-on-using-slots)

## 2.5 描述符协议全景

第一章介绍了 `property` 是一个描述符。现在我们站在更高视角：**`property`、`staticmethod`、`classmethod`、`__slots__` 都"只是"实现了描述符协议的对象**。描述符协议是 Python 属性访问机制的底层统一接口。

### 描述符的分类

**数据描述符**（同时定义了 `__get__` 和 `__set__`/`__delete__`）的优先级**高于**实例字典。**非数据描述符**（只定义了 `__get__`）的优先级**低于**实例字典。

```python-run
class DataDescriptor:
    """数据描述符：__get__ + __set__"""
    def __get__(self, instance, owner):
        print(f"  DataDescriptor.__get__")
        return 42

    def __set__(self, instance, value):
        print(f"  DataDescriptor.__set__({value})")

class NonDataDescriptor:
    """非数据描述符：只有 __get__"""
    def __get__(self, instance, owner):
        print(f"  NonDataDescriptor.__get__")
        return 99

class Test:
    data_des = DataDescriptor()       # 数据描述符
    nondata_des = NonDataDescriptor() # 非数据描述符

t = Test()

# 数据描述符：优先级 > instance.__dict__
print("访问 t.data_des:")
print(f"  → {t.data_des}")
t.data_des = 100  # 触发 __set__，不会写入 instance.__dict__
print(f"  t.__dict__ 中: {'data_des' in t.__dict__}")

# 非数据描述符：优先级 < instance.__dict__
print("\n访问 t.nondata_des:")
print(f"  → {t.nondata_des}")
t.__dict__['nondata_des'] = 888      # 直接写入实例字典
print(f"写入实例属性后: {t.nondata_des}")  # 888 ← 覆盖了描述符！
```

**优先级链（从高到低）：**

```
数据描述符 > instance.__dict__ > 非数据描述符 > class.__dict__（普通类属性） > __getattr__
```

这就是为什么 `@property` 可以拦截写入——它是一个**数据描述符**（`property` 同时定义了 `__get__` 和 `__set__`）。

### 内置描述符一览

| 内置对象 | 描述符类型 | `__get__` 做什么 |
|---------|----------|-----------------|
| `property` | 数据描述符 | 调用 getter/setter/deleter |
| `staticmethod` | 非数据描述符 | 返回原始函数（忽略实例） |
| `classmethod` | 非数据描述符 | 返回 `MethodType(func, cls)` |
| 普通函数 | 非数据描述符 | 返回 `MethodType(func, instance)` |
| `__slots__` 成员 | 数据描述符 | 从固定槽位读取 |

> 📖 官方文档：[Descriptor HowTo Guide](https://docs.python.org/3/howto/descriptor.html), [Invoking Descriptors](https://docs.python.org/3/reference/datamodel.html#invoking-descriptors)

## 2.6 `AttributeError` 在属性访问中的特殊角色

`AttributeError` 在 Python 的属性访问中有两层含义：

1. **"这个属性不存在"**——触发 `__getattr__` 的信号
2. **描述符/属性内部的错误**——需要区分

```python-run
class SensitiveAttr:
    def __init__(self):
        self._secret = "hidden"

    def __getattr__(self, name):
        """仅在正常查找完全失败后调用"""
        print(f"\n  __getattr__ 被触发: 找不到 '{name}'")
        if name == 'secret':
            return f"解密: {self._secret}"
        # 如果不 raise AttributeError，调用方会拿到一个值
        # 这可能不是你想要的
        raise AttributeError(f"没有属性 '{name}'")

    @property
    def lazy(self):
        # 如果 property 内部抛了 AttributeError ...
        raise AttributeError("lazy 尚未就绪")

s = SensitiveAttr()
print(s._secret)     # hidden —— 正常属性，不走 __getattr__
print(s.secret)      # 解密: hidden —— __getattr__ 处理

# 陷阱：property 内部抛 AttributeError 会触发 __getattr__
try:
    s.lazy           # property 内部 raise AttributeError
except AttributeError as e:
    print(f"收到错误: {e}")
# Python 3.12+ 的行为：如果描述符的 __get__ 抛了非 AttributeError，
# 它不会回退到 __getattr__。但 AttributeError 仍然会被当作"不存在"
# 的信号。所以 property 内部尽量不要抛 AttributeError。
```

> ⚠️ **原则**：在 property getter、`__getattribute__`、描述符的 `__get__` 中，**不要抛 `AttributeError`**——应该抛 `RuntimeError`、`ValueError` 等具体异常。这是 Python 社区的共识但尚未成为硬性规范。

> 📖 官方文档：[object.__getattr__](https://docs.python.org/3/reference/datamodel.html#object.__getattr__)

## 2.7 属性控制的选型指南

面对多种属性控制机制，选择标准如下：

```python-run
# ── 场景 1：固定几个属性的校验/计算 ──
# 选: @property
class Account:
    def __init__(self, balance: float):
        self._balance = balance

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, v):
        if v < 0:
            raise ValueError("余额不能为负")
        self._balance = v

# ── 场景 2：拦截"未知"属性（代理、RPC 客户端） ──
# 选: __getattr__
class RemoteClient:
    def __getattr__(self, method_name):
        def remote_call(*args, **kwargs):
            print(f"RPC 调用: {method_name}({args}, {kwargs})")
            return f"result of {method_name}"
        return remote_call

client = RemoteClient()
print(client.login("user", "pass"))      # 任意方法名都可以
print(client.query("SELECT * FROM t"))    # 不需要预先定义

# ── 场景 3：全局属性访问审计 ──
# 选: __getattribute__ + __setattr__ + __delattr__
```

| 需求 | 用哪个 | 原因 |
|------|--------|------|
| 校验单个属性 | `@property` | 最小侵入，最清晰 |
| 多个属性共享同一校验逻辑 | 自定义描述符 | 可复用验证代码 |
| 动态生成属性/代理 | `__getattr__` | 只拦截不存在的属性 |
| 审计/权限控制 | `__getattribute__` | 拦截所有访问 |
| 千万个小对象 | `__slots__` | 省内存、加速 |

## 本章小结

- `@property` 把方法伪装成属性，适合校验和计算，但不该执行 I/O 等耗时操作——用户以为在读取属性。
- `__getattr__` 仅在正常查找失败后触发（适合代理/延迟加载），`__getattribute__` 拦截每一次属性访问（适合审计/权限控制）。
- 数据描述符（含 `__set__`）优先级高于实例字典，非数据描述符优先级低于实例字典——`@property` 正是数据描述符。
- `__slots__` 用固定 C 数组替代 `__dict__`，减少内存开销、加速属性访问，适合创建数百万个小对象的场景。
- 选型路线图：固定几个属性校验用 `@property`，多个属性共享校验逻辑用自定义描述符，动态代理用 `__getattr__`，全局审计用 `__getattribute__`。
