# 2.1 类与元类的基础

本章从 Python 对象模型的底层出发，回答三个问题：**类本身是什么**、**实例化到底发生了什么**、**方法中的 `self` 从哪里来**。

## 1.1 一切皆对象——包括类本身

Python 中"类"本身也是一个对象。这意味着类可以被赋值给变量、可以作为参数传递、可以在运行时动态创建。这个事实是整个元类机制的基础。

```python
class Dog:
    sound = "汪汪"

# 类就是对象——可以赋值、放进列表、传参
Animal = Dog
print(Animal.sound)        # 汪汪
print(type(Dog))           # <class 'type'> ← 类本身是 type 的实例
print(type(Dog()))         # <class '__main__.Dog'> ← 实例是 Dog 的实例
```

关键结论：**类也有类型**。普通实例的类型是它的类（`type(obj)` → 类），而类的类型是 `type`。`type` 是 Python 中最核心的元类（Metaclass）。

```python-run
# type 是所有类的"类的类"
print(type(int))           # <class 'type'>
print(type(str))           # <class 'type'>
print(type(type))          # <class 'type'> ← type 自己的类型还是 type！
print(isinstance(int, type))  # True
print(isinstance(object, type))  # True
```

> 📖 官方文档：[type](https://docs.python.org/3/library/functions.html#type), [metaclasses](https://docs.python.org/3/reference/datamodel.html#metaclasses)

## 1.2 `type(name, bases, namespace)` — 动态造类

`type()` 有两种用法：传入一个参数返回其类型（大家都熟），传入三个参数则**动态创建一个新类**。这也是 Python 在 `class` 关键字背后真正做的事情。

```python
# 这两种写法等价
class Dog:                          # 写法一：class 关键字
    sound = "汪汪"
    def bark(self):
        return self.sound

Dog2 = type('Dog2', (), {          # 写法二：type() 动态创建
    'sound': '汪汪',
    'bark': lambda self: self.sound,
})

print(Dog2().bark())  # 汪汪
```

**带继承的动态创建：**

```python-run
class Animal:
    def speak(self):
        return f"{self.sound}!!"

Dog = type('Dog', (Animal,), {'sound': '汪汪'})
print(Dog().speak())  # 汪汪!!
print(Dog.__bases__)  # (<class '__main__.Animal'>,)
```

`class` 关键字本质上是对 `type()` 的语法糖。Python 读到 `class X(A, B): body` 时，执行 body 构建命名空间字典，然后调用 `type('X', (A, B), namespace)` 创建类对象。

正因如此，你可以通过**替换 `type`** 来控制类的创建过程——这就是元类的核心思想。

```python-run
# 看看 class 关键字内部发生了什么
import builtins

class Meta(type):
    def __new__(mcs, name, bases, namespace):
        print(f"创建类: {name}, 基类: {bases}")
        return super().__new__(mcs, name, bases, namespace)

class Person(metaclass=Meta):   # 类定义本身触发 __new__
    name: str = "unknown"
# 输出: 创建类: Person, 基类: ()
```

> 📖 官方文档：[type — three-argument form](https://docs.python.org/3/library/functions.html#type), [Customizing class creation](https://docs.python.org/3/reference/datamodel.html#customizing-class-creation)

## 1.3 `__new__` vs `__init__`：创造 vs 初始化

`__new__` 负责**创建并返回实例**（分配内存、返回未初始化的对象），`__init__` 负责**初始化已经存在的实例**（为属性赋值）。这是 Python 对象创建中最容易混淆的一对方法。

```python-run
class Demo:
    def __new__(cls, *args, **kwargs):
        print(f"1. __new__ 被调用，创建实例")
        instance = super().__new__(cls)  # 调用 object.__new__ 分配内存
        print(f"   实例 id: {id(instance)}")
        return instance                  # 必须返回实例！

    def __init__(self, value):
        print(f"2. __init__ 被调用，id: {id(self)}")
        self.value = value

d = Demo(42)
# 输出:
# 1. __new__ 被调用，创建实例
#    实例 id: 12345678
# 2. __init__ 被调用，id: 12345678
```

关键规则：

| | `__new__` | `__init__` |
|---|---|---|
| 谁调用 | 由 `type.__call__` 调用 | 由 `type.__call__` 调用（在 `__new__` 之后） |
| 第一个参数 | `cls`（类本身） | `self`（已创建的实例） |
| 必须返回值？ | 必须返回实例 | 只能返回 `None` |
| 典型用途 | 不可变类型子类化、单例模式 | 常规属性初始化 |

### 什么时候必须写 `__new__`

**场景一：子类化不可变类型**（int、str、tuple 等），因为这些类型的值在 `__new__` 中确定，`__init__` 来不及。

```python-run
class UpperStr(str):
    """自动转为大写字符串"""
    def __new__(cls, value):
        value = value.upper()           # 在创建前转换
        return super().__new__(cls, value)  # str.__new__ 创建不可变对象

    # __init__ 不需要写——str 的 __init__ 会被自动调用

s = UpperStr("hello")
print(s)           # HELLO
print(type(s))     # <class '__main__.UpperStr'>
```

如果你尝试用 `__init__` 做这件事：

```python-run
class BrokenUpperStr(str):
    def __init__(self, value):
        self = value.upper()  # 没用！str 的值已经写死在 __new__ 里了
```

**场景二：单例模式（Singleton）**——用 `__new__` 控制实例只创建一次。

```python-run
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

a = Singleton()
b = Singleton()
print(a is b)  # True —— 同一个对象
```

> ⚠️ 单例在模块级变量面前几乎没有优势——模块本身就是天然的单例。只在需要**延迟初始化**或**控制实例个数**时才用。

> 📖 官方文档：[object.__new__](https://docs.python.org/3/reference/datamodel.html#object.__new__), [object.__init__](https://docs.python.org/3/reference/datamodel.html#object.__init__)

## 1.4 `type.__call__`：为什么 `MyClass()` 能工作

很多人天天写 `MyClass()` 但没想过**这个括号语法是谁提供的**。答案：是**元类的 `__call__` 方法**。

```python
# MyClass() 背后发生的事情:
# 1. type(MyClass) 是 type（或自定义元类）
# 2. type.__call__(MyClass, *args, **kwargs) 被触发
# 3. type.__call__ 内部调用 MyClass.__new__ → MyClass.__init__
```

可以用自定义元类截获这个过程：

```python-run
class InspectMeta(type):
    def __call__(cls, *args, **kwargs):
        print(f">>> {cls.__name__}.__call__ 被触发")
        print(f"    args={args}, kwargs={kwargs}")
        instance = super().__call__(*args, **kwargs)
        print(f"    返回实例: {instance}")
        return instance

class Person(metaclass=InspectMeta):
    def __init__(self, name):
        self.name = name

p = Person("Alice")
# 输出:
# >>> Person.__call__ 被触发
#     args=('Alice',), kwargs={}
#     返回实例: <__main__.Person object at 0x...>
```

**完整的调用链：**

```
Person("Alice")
  └─→ type(Person).__call__("Alice")     # 即 InspectMeta.__call__
        └─→ Person.__new__(Person)       # 创建实例
        └─→ Person.__init__(instance, "Alice")  # 初始化
        └─→ 返回 instance
```

这就是为什么"通过自定义元类可以实现单例、注册、自动装饰"——因为元类的 `__call__` 控制了**所有**实例化入口。

> 📖 官方文档：[object.__call__](https://docs.python.org/3/reference/datamodel.html#object.__call__), [metaclass.__call__](https://docs.python.org/3/reference/datamodel.html#metaclass.__call__)

## 1.5 属性查找顺序：`instance.__dict__` → `class.__dict__` → 父类

实例属性和类属性是两个不同的命名空间。Python 的属性查找遵循 MRO（Method Resolution Order）决定的顺序。

```python-run
class Base:
    x = "类属性 Base.x"

class Child(Base):
    x = "类属性 Child.x"   # 覆盖了 Base.x

obj = Child()
obj.x = "实例属性 obj.x"   # 覆盖了 Child.x（只在 obj 上）

print(obj.x)               # 实例属性 obj.x
del obj.x
print(obj.x)               # 类属性 Child.x ← 退回到类属性
del Child.x
print(obj.x)               # 类属性 Base.x  ← MRO 继续向上找
```

查找顺序可以用一句话概括：

```
instance.__dict__ → type(instance).__dict__ → type(instance).__mro__[1:].__dict__
```

用数据描述符（Descriptor）可以截获这个过程——见 1.6 和下一章。

```python-run
# 直观验证 __dict__ 是分离的
class A:
    shared = []  # 危险！所有实例共享同一个列表

a1 = A()
a2 = A()
a1.shared.append(1)
print(a2.shared)  # [1] ← a2 也被影响了！

# 正确做法：实例属性在 __init__ 中绑定
class B:
    def __init__(self):
        self.items = []  # 每个实例有自己的列表

b1 = B()
b2 = B()
b1.items.append(1)
print(b2.items)  # [] ← 互不影响
```

> ⚠️ 可变默认参数（如 `def __init__(self, items=[])`）和可变类属性（如上面的 `shared = []`）是同一个陷阱的两个变体。**永远不要在类体中定义可变对象作为"默认值"**——除非你就是想要共享状态。

> 📖 官方文档：[Class and Instance Attributes](https://docs.python.org/3/tutorial/classes.html#class-and-instance-variables), [MRO](https://www.python.org/download/releases/2.3/mro/)

## 1.6 描述符初窥：`@property` 就是一个描述符

当你写 `@property` 时，Python 内部创建了一个实现了 `__get__` 方法的描述符对象。描述符协议是 Python 属性访问机制的核心——`@property`、`@staticmethod`、`@classmethod`、`__slots__` 全都是描述符。

```python-run
class Celsius:
    """@property 内部原理：手写一个描述符"""
    def __init__(self):
        self._temperature = 0

    def get_temperature(self, instance, owner=None):
        print(f"[描述符.__get__] 被访问了")
        return instance._temperature

    def set_temperature(self, instance, value):
        print(f"[描述符.__set__] 设置值 {value}")
        if value < -273.15:
            raise ValueError("低于绝对零度！")
        instance._temperature = value

    # 把函数变成描述符
    temperature = property(get_temperature, set_temperature)

c = Celsius()
c.temperature = 100    # 触发 set_temperature → [描述符.__set__]
print(c.temperature)   # 触发 get_temperature → [描述符.__get__]
```

描述符三种方法的调用时机：

| 方法 | 触发场景 | 必须有 |
|------|---------|--------|
| `__get__(self, instance, owner)` | `instance.attr` 或 `Class.attr` | 是 |
| `__set__(self, instance, value)` | `instance.attr = value` | 否（只读描述符） |
| `__delete__(self, instance)` | `del instance.attr` | 否 |

**`@property` 就是描述符**的最大意义：当你理解了描述符协议，就理解了为什么 `property` 能拦截属性访问、为什么它能比 getter/setter 方法更自然。

> 📖 官方文档：[Descriptor HowTo Guide](https://docs.python.org/3/howto/descriptor.html), [property](https://docs.python.org/3/library/functions.html#property)

## 1.7 Bound Method vs Unbound Function：`self` 从哪里来

当你在类中定义一个函数后，通过实例访问它时得到的是一个**绑定方法（Bound Method）**——Python 自动把实例作为第一个参数绑定进去。这就是 `self` 的来源。

```python-run
class Cat:
    def meow(self, times=1):
        return "喵" * times

# 通过类访问：得到的是普通函数（unbound function）
print(Cat.meow)        # <function Cat.meow at 0x...>
# Cat.meow()           # TypeError: meow() missing 1 required argument

# 通过实例访问：得到的是绑定方法（bound method）
cat = Cat()
print(cat.meow)        # <bound method Cat.meow of <__main__.Cat object>>
print(cat.meow(2))     # 喵喵 —— self 已自动传入
```

**绑定方法的本质**是描述符：函数对象实现了 `__get__`，当通过实例访问时，描述符协议触发，返回一个 `MethodType(func, instance)` 包装对象。这就是为什么 `self` 不需要你手动传入。

```python-run
import types

class Dog:
    def bark(self):
        return "汪!"

dog = Dog()

# 绑定方法的本质：MethodType 包装了函数 + 实例
bound = types.MethodType(Dog.bark, dog)
print(bound())        # 汪!
print(bound.__func__) # <function Dog.bark>   ← 原始函数
print(bound.__self__) # <__main__.Dog object> ← 绑定的实例
```

| | 通过类访问 | 通过实例访问 |
|---|---|---|
| 返回类型 | `function` | `bound method` |
| 调用方式 | `Class.method(instance, arg)` | `instance.method(arg)` |
| 底层机制 | 原始函数对象 | `MethodType(func, instance)` |

> 📖 官方文档：[Method Objects](https://docs.python.org/3/tutorial/classes.html#method-objects), [types.MethodType](https://docs.python.org/3/library/types.html#types.MethodType)

## 1.8 `super()` 到底返回什么

`super()` 并不返回父类对象——它返回一个**代理对象**，该代理按照 MRO 顺序查找下一个类中的方法。这在多重继承中至关重要。

```python-run
class A:
    def method(self):
        print("A.method")
        return "A"

class B(A):
    def method(self):
        print("B.method → 调用 super()")
        result = super().method()
        print(f"B.method ← super() 返回 {result}")
        return f"B → {result}"

class C(A):
    def method(self):
        print("C.method")
        return "C"

class D(B, C):
    def method(self):
        print("D.method → 调用 super()")
        result = super().method()
        print(f"D.method ← super() 返回 {result}")
        return f"D → {result}"

print(f"MRO: {[c.__name__ for c in D.__mro__]}")
print(D().method())
# 输出顺序: D → B → C → A
# super() 按 MRO 顺序查找，不是按"父类"
```

`super()` 的两参数形式暴露了它的本质：`super(D, self)` 表示"从 D 之后的 MRO 位置开始查找"。

> 📖 官方文档：[super()](https://docs.python.org/3/library/functions.html#super), [MRO and super](https://www.python.org/download/releases/2.3/mro/)

## 本章小结

- 类本身也是对象，`type` 是所有类的元类——`class` 关键字本质是对 `type(name, bases, namespace)` 的语法糖。
- `__new__` 负责创建实例（必须返回实例），`__init__` 负责初始化；不可变类型子类化和单例模式必须重写 `__new__`。
- `type.__call__` 是实例化入口，内部依次调用 `__new__` 和 `__init__`；自定义元类的 `__call__` 可以截获所有实例化过程。
- 属性查找链路：数据描述符 > `instance.__dict__` > 非数据描述符 > `class.__dict__` > `__getattr__`。
- `self` 来自描述符协议——函数通过 `__get__` 被包装为 `MethodType(func, instance)`，自动绑定了实例。
- `super()` 返回的是沿 MRO 查找下一个类的代理对象，不是父类。
