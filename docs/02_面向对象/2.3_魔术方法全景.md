# 2.3 魔术方法全景

魔术方法（Magic Methods）是 Python 对象模型的脊梁骨——它们让你自定义的类能像内置类型一样自然地与语言协作。本章回答三个问题：每个魔术方法**管什么行为**、**为什么需要它**、**怎么实现才正确**。

> 魔术方法也叫"双下方法"（Dunder Methods），即前后各有两个下划线的方法，如 `__init__`。

## 3.1 对象的"身份证"：`__repr__` 与 `__str__`

### 是什么

```python
class Point:
    def __init__(self, x: float, y: float) -> None:
        self.x = x
        self.y = y

    def __repr__(self) -> str:
        return f"Point(x={self.x}, y={self.y})"   # 给开发者看：精确、可求值

    def __str__(self) -> str:
        return f"({self.x}, {self.y})"             # 给用户看：可读、简洁
```

`__repr__` 是给开发者看的，目标是在调试时**不产生歧义**——最好能 `eval(repr(obj))` 还原对象。`__str__` 是给终端用户看的，目标是**可读**。

### 为什么需要

没有 `__repr__`，打印对象会看到 `<__main__.Point object at 0x7f...>`——毫无信息量。没有 `__str__`，`print()` 会回退到 `__repr__`，虽然可用但不够友好。

```python
class BadPoint:
    def __init__(self, x, y): self.x, self.y = x, y

p = BadPoint(3, 4)
print(p)   # <__main__.BadPoint object at 0x103a2b...> ← 毫无价值
```

### 怎么用才对

**铁律：永远实现 `__repr__`，`__str__` 可选。** 当只有一个时，实现 `__repr__`。

| 场景 | 调用哪个 | 回退策略 |
|------|---------|---------|
| `repr(obj)` | `__repr__` | 不存在的回退，必须实现 |
| `str(obj)` | `__str__` | `__str__` 不存在 → 调用 `__repr__` |
| `print(obj)` | `__str__` | 同上 |
| `f"{obj}"` | `__str__` | 同上 |
| `f"{obj!r}"` | `__repr__` | — |
| 调试时在终端输入 `obj` | `__repr__` | — |

```python
import datetime
now = datetime.datetime.now()
print(now)       # 2026-05-26 10:30:00.123456  ← __str__：人能看懂
print(repr(now)) # datetime.datetime(2026, 5, 26, 10, 30, 0, 123456) ← __repr__：可直接求值
```

> 📖 官方文档：[`__repr__`](https://docs.python.org/3/reference/datamodel.html#object.__repr__), [`__str__`](https://docs.python.org/3/reference/datamodel.html#object.__str__)

## 3.2 相等判断与哈希契约：`__eq__` 与 `__hash__`

### 是什么

`__eq__` 控制 `==` 的行为。`__hash__` 为 `hash(obj)` 和 `set`/`dict` 键提供散列值。

### 为什么需要——两条铁律

**铁律 1：如果覆盖了 `__eq__` 而不覆盖 `__hash__`，Python 会把 `__hash__` 设为 `None`，对象变得不可哈希——无法放入 `set` 或作为 `dict` 的键。**

```python
class Person:
    def __init__(self, name: str): self.name = name
    def __eq__(self, other) -> bool:                 # 只覆盖了 __eq__
        return isinstance(other, Person) and self.name == other.name

p = Person("Alice")
s = {p}   # TypeError: unhashable type: 'Person' ← Python 阻止了潜在 bug
```

**铁律 2：相等（`==`）的对象必须返回相同的哈希值。反方向不要求：不同对象可以有相同哈希（哈希冲突）。**

```python
class Person:
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

    def __eq__(self, other) -> bool:
        return isinstance(other, Person) and self.name == other.name

    def __hash__(self) -> int:
        return hash(self.name)     # 用 name 做哈希，和 __eq__ 使用的字段一致

alice1 = Person("Alice", 30)
alice2 = Person("Alice", 25)
print(alice1 == alice2)           # True——只比 name
print(hash(alice1) == hash(alice2))  # True——哈希也一致，契约成立
```

### 为什么可变对象不该实现 `__hash__`

```python
# 危险示范
class BadSetItem:
    def __init__(self, items: list): self.data = items
    def __eq__(self, other): return self.data == other.data
    def __hash__(self): return hash(tuple(self.data))  # ← 危险！data 可变

obj = BadSetItem([1, 2, 3])
s = {obj}
obj.data.append(4)   # 哈希值变了！但 set 内部还在用旧哈希值找它
print(obj in s)      # False ← 对象在 set 里却"找不到"了
```

**Python 的设计哲学就是让可变对象不可哈希**——这是安全措施，不是 bug。`list`、`dict`、`set` 都不可哈希。如果你定义可变对象但覆盖了 `__hash__`，自己承担后果。

一言以蔽之：**值对象（Value Object）才实现 `__hash__`**——比如坐标、复数、日期，这些是不可变的。

> 📖 官方文档：[`__eq__`](https://docs.python.org/3/reference/datamodel.html#object.__eq__), [`__hash__`](https://docs.python.org/3/reference/datamodel.html#object.__hash__)

## 3.3 真值测试：`__bool__` 与 `__len__`

### 是什么

Python 判断对象的真值（Truthiness）时遵循固定的查找顺序：

1. 如果有 `__bool__`，调用它
2. 否则如果有 `__len__`，调用它，`0` 为 `False`，其他为 `True`
3. 都没有：永远是 `True`

```python
class Group:
    def __init__(self, members: list): self.members = members
    def __len__(self) -> int: return len(self.members)
    # 没定义 __bool__，Python 看 __len__

print(bool(Group([])))        # False——len 为 0
print(bool(Group(["A"])))     # True——len 为 1

class AlwaysFalse:
    def __bool__(self) -> bool: return False
    def __len__(self) -> int: return 999     # __bool__ 优先，这个被忽略

print(bool(AlwaysFalse()))     # False——__bool__ 说了算
```

### 为什么需要

让你定义的容器、集合、数据类自然地参与 `if obj:` 这类布尔上下文的判断，无需调用 `.is_empty()` 这种 Java 风格的方法。

```python
cards = Hand([])
if not cards:                    # Python 式写法
    print("空手牌")
# 而不是：
# if cards.is_empty():           # 非 Python 式写法
```

> 📖 官方文档：[`__bool__`](https://docs.python.org/3/reference/datamodel.html#object.__bool__), [`__len__`](https://docs.python.org/3/reference/datamodel.html#object.__len__)

## 3.4 运算符重载：`__add__`、`__radd__`、`__iadd__`

### 是什么

运算符重载（Operator Overloading）让你自定义对象能直接使用 `+`、`-`、`*` 等符号。Python 对每个二元运算符有三层查找：

```python
# a + b 的执行过程：
# 1. 先调 a.__add__(b)   → 如果返回 NotImplemented，进入第 2 步
# 2. 再调 b.__radd__(a)  → 如果也返回 NotImplemented，抛出 TypeError
```

### 为什么需要 `__radd__`（反射运算符）

当你写 `a + b` 而 `a` 不知道如何处理 `b` 时，Python 会反向问 `b`：你能处理 `a` 吗？

```python
class Vector:
    def __init__(self, x: float, y: float): self.x, self.y = x, y

    def __add__(self, other):
        if isinstance(other, Vector):
            return Vector(self.x + other.x, self.y + other.y)
        return NotImplemented

    def __radd__(self, other):
        if isinstance(other, (int, float)):
            return Vector(self.x + other, self.y + other)
        return NotImplemented

    def __repr__(self): return f"Vector({self.x}, {self.y})"

v = Vector(1, 2)
print(v + 5)     # Vector(6, 7) ← v.__add__(5) 不理解 int → 调 int.__radd__ → 不理解 → 调 v.__radd__(5)
# 切题一问：5 + v 呢？
print(5 + v)     # Vector(6, 7) ← int.__add__(v) 不理解 → 调 v.__radd__(5) 成功
```

### `__iadd__`：就地运算（`+=`）

`a += b` 先查找 `__iadd__`。如果没定义，退化为 `a = a + b`——创建新对象而非就地修改。

```python
class Counter:
    def __init__(self, n: int = 0): self.n = n
    def __iadd__(self, other: int):    # 就地运算
        self.n += other
        return self                     # 必须返回 self！
    def __repr__(self): return f"Counter({self.n})"

c = Counter(10)
c += 5              # 调用 __iadd__，就地修改
print(c)            # Counter(15)

# 注意：__iadd__ 必须返回对象本身。对不可变类型来说 += 等于重新赋值：
a = 1
print(id(a))        # 比如 140736...
a += 1
print(id(a))        # 不同地址——int 没有 __iadd__，a = a + 1 创建了新对象
```

> 📖 官方文档：[Emulating numeric types](https://docs.python.org/3/reference/datamodel.html#emulating-numeric-types)

## 3.5 `in` 运算符：`__contains__`

`x in obj` 查找 `obj.__contains__(x)`。如果没定义，Python 会用 `__iter__` 或 `__getitem__` 逐个遍历查找——效率低得多。

```python
class Range:
    def __init__(self, start: int, end: int):
        self.start, self.end = start, end

    def __contains__(self, item: int) -> bool:
        return self.start <= item < self.end    # O(1) 常数时间！

r = Range(0, 100)
print(50 in r)      # True——用了 __contains__，不遍历
print(150 in r)     # False——同样 O(1)
```

没有 `__contains__` 时，`in` 退化为遍历——对 `list` 是 `O(n)`。实现 `__contains__` 可以为自定义容器提供 `O(1)` 的成员判断。

> 📖 官方文档：[`__contains__`](https://docs.python.org/3/reference/datamodel.html#object.__contains__)

## 3.6 上下文管理器协议：`__enter__` 与 `__exit__`

Python 的 `with` 语句背后是一对魔术方法的精确调用：

```python
# with obj as var:
#     body
#
# 等价于：
# _manager = obj
# _enter = type(_manager).__enter__
# _exit  = type(_manager).__exit__
# var = _enter(_manager)
# try:
#     body
# except:
#     if not _exit(_manager, *sys.exc_info()):
#         raise
# else:
#     _exit(_manager, None, None, None)
```

用代码演示：

```python
class ManagedFile:
    def __init__(self, filename: str): self.filename = filename
    def __enter__(self):
        self.file = open(self.filename, 'w')
        print("文件已打开")
        return self.file
    def __exit__(self, exc_type, exc_val, exc_tb):
        self.file.close()
        print("文件已关闭")
        if exc_type:
            print(f"处理了异常: {exc_val}")
        return True   # 返回 True 表示"异常已处理，不往外抛"

with ManagedFile("test.txt") as f:
    f.write("hello")
# 输出：文件已打开 → 文件已关闭
```

`__enter__` 的返回值绑定给 `as` 后面的变量；`__exit__` 的三个参数在无异常时全是 `None`，有异常时分别存储异常类型、值、traceback。`__exit__` 返回 `True` 吞掉异常，返回 `False`（或 `None`）让异常继续传播。

## 3.7 下标访问：`__getitem__` 与 `__setitem__`

`obj[key]` 和 `obj[key] = value` 分别调用 `__getitem__` 和 `__setitem__`——让你的类像 `list`/`dict` 一样操作。

```python
class TaggedData:
    """一个支持 [] 语法的简单包装器"""
    def __init__(self, **kwargs):
        self._data = dict(kwargs)

    def __getitem__(self, key: str):
        return self._data[key]

    def __setitem__(self, key: str, value):
        self._data[key] = value

    def __repr__(self):
        return f"TaggedData({self._data})"

d = TaggedData(name="Alice", age=30)
print(d["name"])      # Alice
d["age"] = 31
print(d["age"])       # 31
```

```python
class SliceDemo:
    """展示切片参数是怎样传到 __getitem__ 的"""
    def __getitem__(self, index):
        if isinstance(index, slice):
            return f"切片: start={index.start}, stop={index.stop}, step={index.step}"
        return f"下标: {index}"

sd = SliceDemo()
print(sd[5])           # 下标: 5
print(sd[1:10:2])      # 切片: start=1, stop=10, step=2
```

> 📖 官方文档：[Emulating container types](https://docs.python.org/3/reference/datamodel.html#emulating-container-types)

## 3.8 综合示例：Vector2D

下面构建一个 `Vector2D` 类，将以上多种魔术方法有机整合在一个实际场景中——而不是零散的片段。

```python-run
import math

class Vector2D:
    """二维向量——一个魔术方法全景示例"""
    __slots__ = ('x', 'y')  # 省内存

    def __init__(self, x: float = 0.0, y: float = 0.0):
        self.x, self.y = float(x), float(y)

    # ── 3.1 标识 ──
    def __repr__(self) -> str:
        return f"Vector2D({self.x!r}, {self.y!r})"

    def __str__(self) -> str:
        return f"({self.x}, {self.y})"

    # ── 3.2 相等与哈希 ──
    def __eq__(self, other) -> bool:
        if not isinstance(other, Vector2D):
            return NotImplemented
        return math.isclose(self.x, other.x) and math.isclose(self.y, other.y)

    def __hash__(self) -> int:
        return hash((round(self.x, 10), round(self.y, 10)))

    # ── 3.3 真值 ──
    def __bool__(self) -> bool:
        return self.x != 0.0 or self.y != 0.0

    # ── 3.4 运算符 ──
    def __add__(self, other):
        if isinstance(other, Vector2D):
            return Vector2D(self.x + other.x, self.y + other.y)
        if isinstance(other, (int, float)):
            return Vector2D(self.x + other, self.y + other)
        return NotImplemented

    def __radd__(self, other):
        return self.__add__(other)

    def __iadd__(self, other):
        if isinstance(other, Vector2D):
            self.x += other.x; self.y += other.y
        elif isinstance(other, (int, float)):
            self.x += other; self.y += other
        else:
            return NotImplemented
        return self

    def __sub__(self, other):
        if isinstance(other, Vector2D):
            return Vector2D(self.x - other.x, self.y - other.y)
        return NotImplemented

    def __neg__(self):
        return Vector2D(-self.x, -self.y)

    def __abs__(self):
        return math.hypot(self.x, self.y)

    # ── 3.5 与 3.7 容器 ──
    def __len__(self) -> int:
        return 2  # 二维向量，始终有 2 个分量

    def __contains__(self, item: float) -> bool:
        return item == self.x or item == self.y

    def __getitem__(self, index: int) -> float:
        if index == 0: return self.x
        if index == 1: return self.y
        raise IndexError(f"下标 {index} 超出范围，Vector2D 只有 0 和 1")

    def __iter__(self):
        yield self.x
        yield self.y


# ── 使用演示 ──
v1 = Vector2D(3, 4)
v2 = Vector2D(1, 2)

print(v1)                        # (3.0, 4.0)          ← __str__
print(repr(v1))                  # Vector2D(3.0, 4.0)  ← __repr__
print(v1 == Vector2D(3.0, 4.0))  # True                ← __eq__
print(hash(v1))                  # 有哈希值             ← __hash__
print(bool(v1), bool(Vector2D(0, 0)))  # True False    ← __bool__
print(v1 + v2)                   # (4.0, 6.0)          ← __add__
print(abs(v1))                   # 5.0                 ← __abs__
print(4.0 in v1)                 # True                ← __contains__
print(v1[0], v1[1])              # 3.0 4.0             ← __getitem__
print(list(v1))                  # [3.0, 4.0]          ← __iter__
```

## 3.9 魔术方法速查表

| 类别 | 方法 | 触发方式 |
|------|------|---------|
| 字符串 | `__repr__` | `repr(obj)`, 终端裸输入 |
| 字符串 | `__str__` | `str(obj)`, `print(obj)` |
| 比较 | `__eq__` / `__ne__` / `__lt__` / `__gt__` / `__le__` / `__ge__` | `==` `!=` `<` `>` `<=` `>=` |
| 哈希 | `__hash__` | `hash(obj)`, `set()` `dict` 键 |
| 真值 | `__bool__` | `bool(obj)`, `if obj:` |
| 真值 | `__len__` | `len(obj)` + 真值回退 |
| 数值运算 | `__add__` / `__radd__` / `__iadd__` | `+` / 反射 `+` / `+=` |
| 数值运算 | `__sub__` / `__mul__` / `__truediv__` | `-` `*` `/` |
| 数值运算 | `__neg__` / `__abs__` | `-obj` / `abs(obj)` |
| 容器 | `__contains__` | `x in obj` |
| 容器 | `__getitem__` / `__setitem__` / `__delitem__` | `obj[k]` / `obj[k]=v` / `del obj[k]` |
| 容器 | `__iter__` / `__next__` | `for x in obj` / `next(obj)` |
| 上下文 | `__enter__` / `__exit__` | `with obj:` |
| 可调用 | `__call__` | `obj()` |

## 本章小结

- 永远实现 `__repr__`（给开发者看的精确表示），`__str__` 可选（给用户看的可读表示）；`print()` / `str()` 在 `__str__` 不存在时回退到 `__repr__`。
- 覆盖 `__eq__` 而不覆盖 `__hash__` 会导致对象不可哈希（无法放入 `set`/`dict` 键）；相等对象的哈希值必须一致，可变对象不该实现 `__hash__`。
- 运算符重载通过 `__add__`（左操作）、`__radd__`（反射）、`__iadd__`（就地）三层实现，`NotImplemented` 信号触发反射查找。
- `__contains__` 让 `in` 操作达到 O(1) 效率；`__getitem__` 同时处理下标索引和 `slice` 对象。
- 上下文管理器的 `__enter__` 返回值绑定给 `as` 变量，`__exit__` 返回 `True` 可吞掉异常；`__bool__` 和 `__len__` 共同决定对象的真值判断。

2. 一个类实现了 `__eq__` 但没有实现 `__hash__`。在什么场景下这会导致运行时错误？在什么场景下"恰好安全"？
3. `a + b` 调用了 `a.__add__(b)`。如果 `a` 是 `int` 而 `b` 是你自定义的 `Vector2D`，魔法是怎样发生的——一步步描述从 `5 + v` 到最终返回结果的完整过程。
