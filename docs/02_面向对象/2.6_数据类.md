# 2.6 数据类：告别样板代码

本章回答三个问题：`@dataclass` **消灭了什么**、**保留了什么控制权**、在 Python 的多种数据结构方案中**该如何选择**。

## 6.1 没有 @dataclass 的日子

先看看"传统写法"有多啰嗦：

```python-run
class Point:
    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"

    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y

    def __hash__(self):
        return hash((self.x, self.y))

p1 = Point(1.0, 2.0)
p2 = Point(1.0, 2.0)
print(p1)             # Point(x=1.0, y=2.0)
print(p1 == p2)       # True
# 写了 15+ 行，只为了"数据 + 比较 + 打印"——这就是样板代码
```

用 `@dataclass` 后：

```python-run
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

p1 = Point(1.0, 2.0)
p2 = Point(1.0, 2.0)
print(p1)             # Point(x=1.0, y=2.0)
print(p1 == p2)       # True
# 4 行。自动生成了 __init__, __repr__, __eq__
```

`@dataclass` 是一个**类装饰器（class decorator）**，它在类定义结束后自动为类添加方法。Python 3.7（PEP 557）引入，是标准库的一部分。

它自动生成的方法：

| 方法 | 作用 | 条件 |
|------|------|------|
| `__init__` | 构造器 | 总是生成（除非 `init=False`） |
| `__repr__` | 打印表示 | 除非 `repr=False` |
| `__eq__` | 值相等比较 | 除非 `eq=False` |
| `__hash__` | 哈希支持 | 当 `eq=True` 且 `frozen=True` 时添加 |
| `__lt__`, `__le__`, `__gt__`, `__ge__` | 排序比较 | 仅当 `order=True` |

> 📖 官方文档：[dataclasses — Data Classes](https://docs.python.org/3/library/dataclasses.html), [PEP 557](https://peps.python.org/pep-0557/)

## 6.2 field()：接管每个字段的行为

`@dataclass` 的默认行为很聪明，但遇到需要定制的字段时，你需要 `field()`：

```python-run
from dataclasses import dataclass, field

@dataclass
class User:
    name: str
    email: str
    # 可变默认值必须用 default_factory——这是 dataclass 的安全设计
    tags: list[str] = field(default_factory=list)
    # init=False：不在 __init__ 中出现，只能后续赋值
    _id: int = field(default=0, init=False, repr=False)
    # compare=False：不参与 __eq__ 比较
    login_count: int = field(default=0, compare=False)

# tags 的 default_factory 保证了每个实例有自己独立的 list
u1 = User("Alice", "alice@a.com")
u2 = User("Alice", "alice@a.com", login_count=999)
print(u1)          # User(name='Alice', email='alice@a.com', tags=[], login_count=0)
print(u1 == u2)    # True  ← login_count 不参与比较
u1.tags.append("vip")
print(u2.tags)     # []  ← u2 的 tags 不受影响（default_factory 起作用了！）
```

**为什么不能写 `tags: list[str] = []`？**

```python-run
from dataclasses import dataclass

try:
    @dataclass
    class BadUser:
        name: str
        tags: list[str] = []    # ← Python 会阻止你
except ValueError as e:
    print(f"ValueError: {e}")
    # "mutable default <class 'list'> for field tags is not allowed"
    # 原因：Python 的默认参数在函数定义时只求值一次，
    # 所有实例会共享同一个 list 对象——这是 Python 最经典的陷阱
```

`field()` 的完整参数：

| 参数 | 作用 | 默认值 |
|------|------|--------|
| `default` | 字段默认值 | `MISSING`（必填） |
| `default_factory` | 零参函数，每次创建时调用 | `MISSING` |
| `init` | 是否纳入 `__init__` | `True` |
| `repr` | 是否纳入 `__repr__` | `True` |
| `hash` | 是否纳入 `__hash__` | `None`（自动推断） |
| `compare` | 是否参与 `__eq__` 等比较 | `True` |
| `metadata` | 用户自定义元数据（给库/框架用） | `None` |
| `kw_only` | 仅限关键字参数（3.10+） | `False` |

> 📖 官方文档：[field()](https://docs.python.org/3/library/dataclasses.html#dataclasses.field)

## 6.3 __post_init__：构造完成后的钩子

`__init__` 是自动生成的，你不能再写一个。但如果需要在构造完成后做验证或计算，`__post_init__` 就是你的钩子：

```python-run
from dataclasses import dataclass, field
import re

@dataclass
class Customer:
    name: str
    email: str
    age: int
    # 不传入 __init__，由 __post_init__ 计算
    is_adult: bool = field(init=False)
    # 派生字段，不在 init 中
    created_from: str = field(default="", init=False, repr=False)

    def __post_init__(self):
        # 验证
        if self.age < 0:
            raise ValueError(f"年龄不能为负: {self.age}")
        if not re.match(r'^[^@]+@[^@]+\.[^@]+$', self.email):
            raise ValueError(f"邮箱格式无效: {self.email}")
        # 派生计算
        self.is_adult = self.age >= 18
        self.created_from = "manual"

try:
    Customer("Bad", "not-an-email", -5)
except ValueError as e:
    print(f"校验失败: {e}")

c = Customer("Alice", "alice@example.com", 25)
print(f"name={c.name}, adult={c.is_adult}")
# name=Alice, adult=True
```

**`__post_init__` 的典型用途**：
- 字段值验证（替代传统 `__init__` 中的 `if...raise`）
- 从已有字段计算派生字段
- 数据格式标准化（如 `self.email = self.email.strip().lower()`）

> 📖 官方文档：[__post_init__](https://docs.python.org/3/library/dataclasses.html#post-init-processing)

## 6.4 slots=True：内存敏感的利器

Python 3.10+ 支持 `@dataclass(slots=True)`——自动为类生成 `__slots__`：

```python-run
from dataclasses import dataclass
import sys

@dataclass
class Normal:
    x: float
    y: float
    name: str

@dataclass(slots=True)
class Slotted:
    x: float
    y: float
    name: str

n = Normal(1.0, 2.0, "hello")
s = Slotted(1.0, 2.0, "hello")

# __slots__ 移除了 __dict__，导致：
try:
    n.new_attr = "surprise"  # OK：Normal 有 __dict__
    s.new_attr = "surprise"  # AttributeError：Slotted 没有 __dict__！
except AttributeError as e:
    print(f"Slotted 不允许动态属性: {e}")

# 内存差异（大致值，依赖于具体环境）：
print(f"Normal:  {sys.getsizeof(n) + sys.getsizeof(n.__dict__)} 字节（含 __dict__）")
print(f"Slotted: {sys.getsizeof(s)} 字节")
```

**`slots=True` 的权衡**：

| | slots=False（默认） | slots=True |
|---|---|---|
| 动态属性 | 支持（有 `__dict__`） | 不支持（没有 `__dict__`） |
| 多继承 | 无限制 | 所有父类也必须有 `__slots__` |
| 内存 | 每个实例有 `__dict__` 开销 | 大幅减少内存 |
| 速度 | 属性访问需要字典查找 | 属性访问更快（偏移量直接访问） |
| 适用场景 | 少量实例 / 需要动态属性 | 大量实例（百万级） / 无需动态属性 |

数据科学 / 金融计算中创建大量小对象时，`slots=True` 可以把内存从几 GB 砍到几百 MB。

> 📖 官方文档：[__slots__](https://docs.python.org/3/reference/datamodel.html#slots), [dataclass slots](https://docs.python.org/3/library/dataclasses.html#dataclasses.dataclass)

## 6.5 kw_only 与 KW_ONLY：强制关键字参数

### 全场关键字：kw_only=True（3.10+）

```python-run
from dataclasses import dataclass

@dataclass(kw_only=True)
class Book:
    title: str
    author: str
    pages: int = 0

# 所有参数必须用关键字传入
b = Book(title="Python编程", author="Guido", pages=500)
# b = Book("Python编程", "Guido", 500)  # TypeError!
```

这是**防止传参顺序错误**的最简单手段。当你发现自己写 `Point(x, y)` 时经常搞反 x 和 y，`kw_only=True` 是你的朋友。

### 混合模式：KW_ONLY 占位符（3.10+）

有时你希望前几个字段是位置型的（因为它们的含义足够明显），后面的强制关键字：

```python-run
from dataclasses import dataclass, KW_ONLY

@dataclass
class SearchQuery:
    keyword: str              # 位置参数（含义太明显了）
    _: KW_ONLY                # ← 从这里开始，后面全是 keyword-only
    page: int = 1
    page_size: int = 20
    sort_by: str = "relevance"

# keyword 可以位置传入，其余必须关键字
q = SearchQuery("python教程", page=1, sort_by="date")
print(q)
# SearchQuery(keyword='python教程', page=1, page_size=20, sort_by='date')
# SearchQuery("python教程", 1, 20, "date")  # TypeError!
```

> 📖 官方文档：[KW_ONLY](https://docs.python.org/3/library/dataclasses.html#dataclasses.KW_ONLY)

## 6.6 五种数据容器的选择地图

Python 有太多方式定义"数据结构"。选择困惑是正常的——下表帮你理清：

| | dataclass | NamedTuple | TypedDict | plain class | dict |
|---|---|---|---|---|---|
| **可变** | ✓ 默认 | ✗ 不可变 | ✗（仅标注） | ✓ | ✓ |
| **默认值** | ✓ | ✓ | ✗ | 手动 | ✗ |
| **运行时校验** | ✗（需手动） | ✗ | ✗ | ✗ | ✗ |
| **方法** | ✓ | ✓ | ✗ | ✓ | ✗ |
| **内存效率** | 普通/`slots=True` | 较好 | 无实例 | 普通 | 较差 |
| **类型安全** | mypy/pyright | mypy/pyright | mypy/pyright | mypy/pyright | ✗ |
| **迭代/解包** | ✗ | ✓（像 tuple） | ✗ | ✗ | ✓ |

### 代码对比

```python-run
from dataclasses import dataclass
from typing import NamedTuple, TypedDict

# ── NamedTuple：不可变、可解包、轻量 ──
class PointNT(NamedTuple):
    x: float
    y: float

p = PointNT(1.0, 2.0)
x, y = p                   # ← 可以解包！
# p.x = 3.0                # AttributeError: can't set attribute

# ── TypedDict：只是类型提示，零运行时开销 ──
class PointTD(TypedDict):
    x: float
    y: float

d: PointTD = {"x": 1.0, "y": 2.0}  # 就是一个普通 dict
print(type(d))                     # <class 'dict'>

# ── dataclass：可变、有方法、有默认值 ──
@dataclass
class PointDC:
    x: float
    y: float
    label: str = "point"

    def distance_from_origin(self) -> float:
        return (self.x ** 2 + self.y ** 2) ** 0.5

pdc = PointDC(3.0, 4.0)
print(pdc.distance_from_origin())  # 5.0
```

### 选择决策树

```
需要运行时校验？
├── 是 → Pydantic BaseModel（见第 2.7 章）
└── 否
    ├── 需要可变 + 方法？→ @dataclass
    ├── 需要不可变 + 可解包？→ NamedTuple
    ├── 只是描述外部 JSON / API 的类型？→ TypedDict
    ├── 需要动态属性 / 任意控制？→ 普通类
    └── 数据无固定结构？→ 普通 dict
```

> 📖 官方文档：[NamedTuple](https://docs.python.org/3/library/typing.html#typing.NamedTuple), [TypedDict](https://docs.python.org/3/library/typing.html#typing.TypedDict)

## 6.7 工具函数：fields()、asdict()、astuple()

标准库提供了遍历、转换 dataclass 的工具：

```python-run
from dataclasses import dataclass, fields, asdict, astuple, field

@dataclass
class Person:
    name: str
    age: int
    email: str = field(default="", repr=False)
    _secret: str = field(default="hidden", repr=False)

# fields()：遍历/反射字段定义
for f in fields(Person):
    print(f"  {f.name}: type={f.type}, default={f.default}, "
          f"repr={f.repr}, init={f.init}")

# asdict()：递归转 dict（适合 JSON 序列化）
p = Person("Alice", 30, email="alice@example.com")
print(f"\nasdict: {asdict(p)}")
# {'name': 'Alice', 'age': 30, 'email': 'alice@example.com', '_secret': 'hidden'}

# astuple()：递归转 tuple
print(f"astuple: {astuple(p)}")
# ('Alice', 30, 'alice@example.com', 'hidden')

# 可以手动 import json 后序列化
import json
print(json.dumps(asdict(p), ensure_ascii=False))
```

**注意**：`asdict()` 和 `astuple()` 是**深拷贝**——修改返回值不影响原始 `dataclass` 实例。

> 📖 官方文档：[fields()](https://docs.python.org/3/library/dataclasses.html#dataclasses.fields), [asdict()](https://docs.python.org/3/library/dataclasses.html#dataclasses.asdict)

## 6.8 实战：一个配置类

结合上面所有知识，写一个真实的实战示例：应用配置类。

```python-run
from dataclasses import dataclass, field, fields, asdict
from typing import Literal
import json
import os

EnvType = Literal["dev", "staging", "production"]

@dataclass(kw_only=True)
class AppConfig:
    # ── 核心配置 ──
    env: EnvType
    app_name: str = "my-app"
    version: str = "1.0.0"

    # ── 网络 ──
    host: str = "0.0.0.0"
    port: int = 8000

    # ── 数据库 ──
    database_url: str = ""

    # ── 运行时状态：不参与 init，序列化时隐藏 ──
    _started_at: float = field(default=0.0, init=False, repr=False)
    # 计算字段：不暴露给 JSON
    _is_secure: bool = field(init=False, repr=False)

    def __post_init__(self):
        # 校验
        if self.port < 1 or self.port > 65535:
            raise ValueError(f"无效端口: {self.port}")
        # 环境感知默认值
        if not self.database_url:
            env_db = {
                "dev": "sqlite:///dev.db",
                "staging": "postgresql://localhost:5432/staging",
                "production": "postgresql://prod-db:5432/main",
            }
            self.database_url = env_db[self.env]
        # 派生字段
        self._is_secure = "postgresql" in self.database_url

    def to_json(self) -> str:
        """导出配置（跳过下划线开头的内部字段）"""
        public = {k: v for k, v in asdict(self).items()
                  if not k.startswith('_')}
        return json.dumps(public, indent=2, ensure_ascii=False)

    @property
    def connection_info(self) -> str:
        return f"{self.host}:{self.port} ({self.env})"

# ── 使用 ──
config = AppConfig(env="dev")
print(f"连接: {config.connection_info}")
print(f"DB:   {config.database_url}")
print(f"JSON:\n{config.to_json()}")
```

这个配置类展示了：`field()` 定制、`__post_init__` 验证与默认值计算、`kw_only=True` 强制显式传参、`@property` 提供只读派生值、`asdict()` + `json.dumps()` 导出。

> 📖 延伸：这个模式在真实项目中非常实用——Pydantic 的 `BaseSettings` 就是基于类似思想构建的（[pydantic-settings](https://docs.pydantic.dev/latest/concepts/pydantic_settings/)）。

## 本章小结

- `@dataclass` 自动生成 `__init__`、`__repr__`、`__eq__`，用 4 行替代传统 15+ 行的样板代码；它是类装饰器，在类定义结束后自动添加方法。
- 可变默认值必须用 `field(default_factory=list)` 而非 `= []`——这是 Python 默认参数在函数定义时只求值一次的经典陷阱的 dataclass 版本。
- `__post_init__` 是自动生成 `__init__` 后的钩子，用于字段验证、派生字段计算；`slots=True` 大幅度减少内存，`kw_only=True` 强制关键字传参。
- 五种数据容器各有侧重：需要运行时校验 → Pydantic；需要可变+方法 → dataclass；需要不可变+解包 → NamedTuple；描述 JSON 形状 → TypedDict；无固定结构 → dict。
- `fields()`、`asdict()`、`astuple()` 提供遍历和转换能力；`asdict()` 进行深拷贝，适合 JSON 序列化。

## 思考题

1. 为什么 `@dataclass` 禁止用可变对象作为默认值（`tags: list = []`），但允许 `field(default_factory=list)`？这两者在 Python 执行语义上的区别是什么？

2. 你想让一个 dataclass 既可变又希望它可哈希（能放进 `set` / 做 `dict` 的 key）。这能实现吗？需要怎么配置？

3. 一个类上同时写了 `@dataclass` 和自己定义的 `__init__`。哪个 `__init__` 生效？如果你需要自定义 `__init__`，但希望保留 dataclass 生成的其他方法（`__repr__`, `__eq__` 等），应该怎么做？

4. 你在写一个处理百万条日志行的程序。每条日志是一个 dataclass。用 `slots=True` vs 不用，内存差异有多大？写代码用 `sys.getsizeof()` 验证你的估算。

5. `@dataclass(order=True)` 生成的比较方法（`__lt__` 等）按照什么顺序比较字段？如果你的类有 5 个字段但只想按其中 2 个排序，该怎么设计？
