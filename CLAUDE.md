# CLAUDE.md — Python 教程项目

## 项目定位

编写一份面向 **Python 3.12+** 的中文进阶教程，目标读者是有一定 Python 基础、想要**深入理解 Python 内部机制、掌握现代用法、提升代码品味**的开发者。

回答的时候, 必须在最开始加上称呼, `Lord Dean` 
## 核心理念

- **不是面面俱到的参考手册**，而是"核心 + 思想"——每个主题回答三个问题：它是什么、为什么需要它、怎么用才对。
- **用代码说话**：每个概念配最小可运行示例，避免长篇文字描述。
- **对比出真知**：关键概念用"错误 vs 正确"、"Python 方式 vs 其他语言方式"对比展示。
- **新特性融入主线**：不单独开一章罗列新特性，而是在讲相关主题时自然带出（如讲类型时引入 `match case`、讲并发时引入 `asyncio` 的新用法）。
- **官方溯源**：每个核心概念附带 Python 官方文档（docs.python.org）的原文引用链接，引导读者建立看官方文档的习惯。
- **简化实现讲思想**：讲解框架/库时，不光讲用法，还要提供 50-100 行的简化版核心实现，让读者看清背后的设计模式（如 FastAPI 依赖注入的简化版、Flask 路由装饰器的简化版、SQLAlchemy Session 管理的简化版等）。

## 技术约束

- 基于 **Python 3.12/3.13**，语法和标准库用法以此为准。
- 代码示例必须可以在 Python 3.12+ 下直接运行。
- 示例文件名用英文（如 `decorator_demo.py`），教程正文用中文。概念提供英文原版定义。

## 质量要求

- 代码示例优先展示"最佳实践"和"常见陷阱"，而非穷举所有语法。
- 每章结尾有 3-5 个"思考题"帮助读者检验理解，而非大量习题。
- 术语首次出现时附英文原名（如"装饰器（Decorator）"）。
- 核心概念必须标注 Python 官方文档对应章节的链接（如 **[Official Docs: Data Model](https://docs.python.org/3/reference/datamodel.html)**）。
- 框架/库讲解中，设计思想部分提供简化版实现代码（标注为"简化实现"），用法部分正常讲解 API。
- 引用外部资源时标注出处。

### 00 快速基础（6 章）

| 章节 | 一句话摘要 |
|------|-----------|
| 0.1 变量与基本类型 | 变量是贴在对象上的"名字标签"，`=` 的本质是名字绑定而非赋值 |
| 0.2 字符串与编码 | f-string 在 `{}` 内可写任意表达式，Unicode/UTF-8 编解码在 IO 边界处理 |
| 0.3 list 与 tuple | list 可变序列支持增删改，tuple 浅层不可变但有结构语义 |
| 0.4 条件判断与模式匹配 | `match case`（3.10+）是解构+匹配+绑定三合一，`_` 匹配任意值 |
| 0.5 循环 | `for` 遍历可迭代对象，`enumerate`/`zip`/`else` 三大搭档，推导式一行搞定循环+过滤 |
| 0.6 dict 与 set | dict 是 Python 内部基石（3.7+ 保序），set 无重复且 O(1) 成员检查 |

### 01 基础（11 章）

| 章节 | 一句话摘要 |
|------|-----------|
| 1.1 Python 对象模型 | 一切皆对象，`is` 比身份 `==` 比值，引用计数+分代 GC 管理内存，可变默认参数陷阱用 None 哨兵 |
| 1.2 名字与作用域 | LEGB 四层查找规则，`global`/`nonlocal` 写入外层作用域，`del` 删名字不删对象 |
| 命名空间深度解析 | 变量=裸名字走 LEGB，属性=`obj.name` 走描述符+`__dict__`+MRO——两种完全不同的查找机制 |
| 闭包深度解析 | 闭包=函数+捕获的自由变量（存在 `__closure__` 的 cell 中），循环中延迟绑定需用默认参数"冻结" |
| 1.3 容器深度解析 | list 动态数组均摊 O(1) append，dict 紧凑哈希 3.6+ 节省内存，slice 是真实类型对象 |
| 1.4 字符串与字节 | str→bytes=encode，bytes→str=decode，f-string 3.12 支持引号混用，memoryview 零拷贝 |
| 1.5 模式匹配 | `match case` 声明式结构拆解，序列/映射/类模式，守卫弥补范围匹配，OR 模式共享分支 |
| 装饰器深度解析 | `@deco` 是语法糖=定义时 `func=deco(func)`，本质是闭包应用，`@wraps` 必须加，参数化需三层 |
| 1.6 函数是一等公民 | 函数是对象带完整魔术属性，`__call__` 让函数可调用，`__get__` 让 `self` 自动绑定，`/` 和 `*` 控制传参方式 |
| 1.7 异常体系 | 异常即对象带 `__traceback__`/`__cause__`，`raise...from` 翻译异常，`except*` 处理 ExceptionGroup（3.11+） |
| 1.8 模块系统详解 | Finder→Loader→Module 三步导入，`sys.modules` 缓存，相对导入限包内，循环导入三种修复 |

### 02 面向对象（8 章）

| 章节 | 一句话摘要 |
|------|-----------|
| 2.1 类与元类的基础 | `class` 即 `type(name,bases,namespace)`，`__new__` 创建 `__init__` 初始化，属性查找链有固定优先级 |
| 2.2 属性控制 | `@property` 伪装属性适用校验计算，`__getattr__` 兜底 `__getattribute__` 全拦截，`__slots__` 省内存 |
| 2.3 魔术方法全景 | 必实现 `__repr__`，`__eq__`+`__hash__` 有契约约束，运算符重载三层（`__add__`/`__radd__`/`__iadd__`） |
| 2.4 可迭代与可调用对象 | Iterable 返回 Iterator，生成器是创建迭代器最简方式，`__call__` 让实例像函数 |
| 2.5 继承与 MRO | C3 线性化保证子类优先，`super()` 沿 MRO 找下一个类，Mixin 是多重继承的正确用法 |
| 2.6 数据类 | `@dataclass` 消灭样板代码，可变默认值用 `field(default_factory=...)`，五种容器各有侧重 |
| 2.7 类型系统进阶 | 类型标注不约束运行时，`TypeVar`/`ParamSpec`/`TypeVarTuple` 三大泛型，Protocol 是鸭子类型静态表达 |
| 2.8 元类实战 | `__init_subclass__` 覆盖 90% 场景，`__set_name__` 自动获取属性名，元类独有能力是修改类创建前的命名空间 |

### 03 高级主题（8 章）

| 章节 | 一句话摘要 |
|------|-----------|
| 3.1 迭代器与生成器深度 | `for` 脱糖为 `iter()+next()+StopIteration`，`yield from` 完全委托，`send()`/`throw()`/`close()` 双向通信 |
| 3.2 装饰器完全指南 | 可选参数装饰器检测首参是否 callable，类装饰器批量改类行为，`lru_cache`/`cached_property` 标准库应用 |
| 3.3 上下文管理器深度 | `__enter__`/`__exit__` 协议，`@contextmanager` 用生成器简化，`ExitStack` 动态管理，异步 `async with` |
| 3.4 描述符协议 | 数据描述符优先级>实例 `__dict__`，`@property`/`@staticmethod`/`@classmethod` 都是描述符，`__set_name__` 自动获取属性名 |
| 3.5 并发：线程与 GIL | GIL 保护引用计数但限制 CPU 并行，I/O 密集用线程 CPU 密集用 multiprocessing，`concurrent.futures` 统一接口 |
| 3.6 并发：asyncio 深入 | 协程协作式调度纳秒级切换，`TaskGroup`(3.11+)结构化并发，`asyncio.Queue`/`Semaphore`/`to_thread` 实战模式 |
| 并发编程全景 | 进程/线程/协程 API 对照，经典案例，推荐 API 及选择指南，决策树与混合使用 |
| 3.7 内存管理与性能 | 引用计数+分代 GC，`weakref` 不增加计数，`__slots__` 省 65% 内存，`tracemalloc`/`dis` 性能分析 |
| 3.8 元编程 | `exec`/`eval`/`compile` 动态执行，`type()` 三参数动态创建类，`inspect` 运行时内省，`ast` 语法树操作 |
